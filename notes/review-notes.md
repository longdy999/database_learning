## Key Rules to Never Forget
1. Every table needs a PRIMARY KEY
2. Always use WHERE with UPDATE and DELETE
3. Use IS NULL not = NULL
4. Name tables in lowercase_with_underscores
5. SELECT first, then UPDATE/DELETE (safety pattern)
6. Use named columns in INSERT (not positional)
7. Test constraints with bad data to verify they work
