# 180-Day Database Learning Roadmap - Complete Overview

## Philosophy

This program is designed like a professional training system:
- **Progressive:** Each day builds on the previous day
- **Practical:** Every concept includes real SQL you can run
- **Bilingual:** English + Khmer for deep understanding
- **Professional:** Builds habits used by real database professionals
- **Exam-ready:** Prepares you for database certification exams

---

## Monthly Breakdown

### Month 1: Database Fundamentals (Days 1-30)
**Theme:** Understanding what databases are and writing your first SQL

**Weekly progression:**
- Week 1 (Days 1-7): What is a database? Tables, rows, columns
- Week 2 (Days 8-14): Primary keys, data types, CREATE TABLE
- Week 3 (Days 15-21): INSERT, SELECT basics, WHERE clause
- Week 4 (Days 22-28): UPDATE, DELETE, basic filtering
- Days 29-30: Review and Month 1 mini-project

**Key skills:** Create tables, insert data, basic SELECT queries

---

### Month 2: SQL Querying (Days 31-60)
**Theme:** Becoming comfortable reading and writing SQL queries

**Weekly progression:**
- Week 5 (Days 31-37): ORDER BY, LIMIT, LIKE, wildcards
- Week 6 (Days 38-44): JOIN basics - INNER JOIN, LEFT JOIN
- Week 7 (Days 45-51): GROUP BY, COUNT, SUM, AVG, aggregate functions
- Week 8 (Days 52-58): Multiple joins, HAVING, complex queries
- Days 59-60: Review and Month 2 mini-project

**Key skills:** Joins, grouping, filtering, multi-table queries


---

### Month 3: Database Design (Days 61-90)
**Theme:** Designing databases that are clean, efficient, and professional

**Weekly progression:**
- Week 9 (Days 61-67): Relationships - one-to-one, one-to-many, many-to-many
- Week 10 (Days 68-74): Normalization - 1NF, 2NF, 3NF
- Week 11 (Days 75-81): ERD diagrams, foreign keys, constraints
- Week 12 (Days 82-88): Design patterns, naming conventions, real examples
- Days 89-90: Review and Month 3 mini-project

**Key skills:** Design normalized databases, draw ERD diagrams, define relationships

---

### Month 4: Advanced SQL & Performance (Days 91-120)
**Theme:** Writing powerful queries and understanding database performance

**Weekly progression:**
- Week 13 (Days 91-97): Subqueries, nested SELECT, EXISTS
- Week 14 (Days 98-104): Views, temporary tables, CTEs
- Week 15 (Days 105-111): Indexes, query performance, EXPLAIN
- Week 16 (Days 112-118): Transactions, ACID, stored procedures basics
- Days 119-120: Review and Month 4 mini-project

**Key skills:** Subqueries, views, indexes, transactions, performance thinking

---

### Month 5: Real-World Database Work (Days 121-150)
**Theme:** Using databases like a professional in real projects

**Weekly progression:**
- Week 17 (Days 121-127): PostgreSQL/MySQL setup and differences
- Week 18 (Days 128-134): Backup, restore, import/export, data cleaning
- Week 19 (Days 135-141): Database security, users, permissions, injection prevention
- Week 20 (Days 142-148): Reporting queries, dashboards, API database patterns
- Days 149-150: Review and Month 5 mini-project

**Key skills:** Production database management, security, reporting

---

### Month 6: Professional Readiness (Days 151-180)
**Theme:** Exam preparation, portfolio building, client communication

**Weekly progression:**
- Week 21 (Days 151-157): Exam review - fundamentals and SQL syntax
- Week 22 (Days 158-164): Exam review - design, normalization, advanced SQL
- Week 23 (Days 165-171): Portfolio project - full database design + implementation
- Week 24 (Days 172-178): Client communication, technical writing, professional habits
- Days 179-180: Final review and exam simulation

**Key skills:** Exam readiness, professional communication, portfolio quality work

---

## Daily Time Allocation (30 minutes)

| Activity | Time | Purpose |
|----------|------|---------|
| Read topic + explanation | 5 min | Understanding |
| Study SQL example | 5 min | Pattern recognition |
| Hands-on practice | 10-15 min | Skill building |
| Vocabulary + Speaking | 5 min | Communication |
| Review task | 5 min | Retention |

## Success Metrics

By Day 180, you should be able to:
- [ ] Write any basic-to-intermediate SQL query without help
- [ ] Design a normalized database for a real business problem
- [ ] Explain database concepts to non-technical clients
- [ ] Pass a database certification exam
- [ ] Read and understand existing database schemas
- [ ] Troubleshoot basic performance issues
- [ ] Work with PostgreSQL or MySQL professionally
- [ ] Present your portfolio project confidently
