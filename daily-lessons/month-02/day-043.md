# Day 43: RIGHT JOIN and FULL JOIN Concepts

## Topic
RIGHT JOIN as the mirror of LEFT JOIN, and FULL OUTER JOIN for keeping all rows from both tables. Understanding when to use each and SQLite limitations.

## English Explanation
A RIGHT JOIN is the exact mirror of LEFT JOIN — it keeps all rows from the RIGHT table and fills NULL for unmatched left-table columns. In practice, most developers simply rewrite RIGHT JOIN as LEFT JOIN by swapping the table order. They produce identical results.

A FULL OUTER JOIN keeps all rows from BOTH tables. If a row from the left has no match on the right, the right columns are NULL. If a row from the right has no match on the left, the left columns are NULL. This gives you the complete picture of both tables combined.

Important note: SQLite does not support FULL OUTER JOIN natively. You can simulate it with UNION of LEFT JOIN and RIGHT JOIN results. PostgreSQL and MySQL 8+ support it directly.

## SQL Example

### RIGHT JOIN (Mirror of LEFT JOIN)
```sql
-- RIGHT JOIN: keep all orders, show customer if exists
SELECT c.name, o.order_id, o.product
FROM customers c
RIGHT JOIN orders o ON c.customer_id = o.customer_id;

-- Equivalent LEFT JOIN (just swap table order):
SELECT c.name, o.order_id, o.product
FROM orders o
LEFT JOIN customers c ON c.customer_id = o.customer_id;
```

### FULL OUTER JOIN (PostgreSQL syntax)
```sql
SELECT c.name, o.order_id, o.product
FROM customers c
FULL OUTER JOIN orders o ON c.customer_id = o.customer_id;
```


### FULL JOIN Result Example
| name   | order_id | product  |
|--------|----------|----------|
| Sokha  | 101      | Laptop   |
| Sokha  | 102      | Mouse    |
| Dara   | 103      | Keyboard |
| Chenda | NULL     | NULL     |
| Bopha  | NULL     | NULL     |
| NULL   | 104      | Monitor  |

The last row shows an order (104) with no matching customer.

### Simulating FULL JOIN in SQLite
```sql
-- SQLite workaround using UNION
SELECT c.name, o.order_id, o.product
FROM customers c
LEFT JOIN orders o ON c.customer_id = o.customer_id
UNION
SELECT c.name, o.order_id, o.product
FROM orders o
LEFT JOIN customers c ON c.customer_id = o.customer_id;
```

## Reading Practice
Compare these two queries and confirm they produce the same result:
```sql
-- Query A
SELECT * FROM orders o RIGHT JOIN customers c ON o.customer_id = c.customer_id;
-- Query B
SELECT * FROM customers c LEFT JOIN orders o ON o.customer_id = c.customer_id;
```
Question: Are Query A and Query B equivalent? Answer: Yes.
