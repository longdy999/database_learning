## SQL Example

```sql
-- Scenario 1: Employees + Departments
-- departments: id, name, location
-- employees: id, name, salary, department_id

SELECT e.name AS employee, d.name AS department, e.salary
FROM employees e
INNER JOIN departments d ON e.department_id = d.id;

-- Find employees in the "Engineering" department
SELECT e.name, e.salary
FROM employees e
INNER JOIN departments d ON e.department_id = d.id
WHERE d.name = 'Engineering';

-- Count employees per department
SELECT d.name, COUNT(e.id) AS employee_count
FROM departments d
INNER JOIN employees e ON d.id = e.department_id
GROUP BY d.name
ORDER BY employee_count DESC;
```


```sql
-- Scenario 2: Orders + Products
-- products: id, name, price, category
-- orders: id, product_id, customer_id, quantity, order_date

SELECT o.id AS order_id, p.name AS product, p.price, o.quantity
FROM orders o
INNER JOIN products p ON o.product_id = p.id;

-- Total revenue per product
SELECT p.name, SUM(p.price * o.quantity) AS revenue
FROM orders o
INNER JOIN products p ON o.product_id = p.id
GROUP BY p.name
ORDER BY revenue DESC;

-- Scenario 3: Students + Courses
-- courses: id, title, instructor, credits
-- enrollments: id, student_id, course_id, grade
-- students: id, name, email, major

SELECT s.name AS student, c.title AS course, e.grade
FROM students s
INNER JOIN enrollments e ON s.id = e.student_id
INNER JOIN courses c ON e.course_id = c.id;
```

## Reading Practice

Read and understand this multi-table JOIN:

```sql
SELECT s.name, c.title, c.instructor, e.grade
FROM students s
INNER JOIN enrollments e ON s.id = e.student_id
INNER JOIN courses c ON e.course_id = c.id
WHERE e.grade IS NOT NULL
ORDER BY s.name, c.title;
```

Questions:
- How many tables are joined here?
- What is the role of the enrollments table?
- Why do we need enrollments as a middle table?
- What does the WHERE clause filter out?
