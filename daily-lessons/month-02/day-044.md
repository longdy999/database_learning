## SQL Example

### 15 JOIN Queries to Write from Memory

**Basic INNER JOIN (queries 1-5):**
```sql
-- 1. Show customers with their orders
SELECT c.name, o.product FROM customers c
INNER JOIN orders o ON c.customer_id = o.customer_id;

-- 2. Show employees with department names
SELECT e.name, d.dept_name FROM employees e
INNER JOIN departments d ON e.dept_id = d.dept_id;

-- 3. Show orders with product details
SELECT o.order_id, p.product_name, p.price FROM orders o
INNER JOIN products p ON o.product_id = p.product_id;

-- 4. Show students with their enrolled courses
SELECT s.name, c.course_name FROM students s
INNER JOIN enrollments e ON s.student_id = e.student_id
INNER JOIN courses c ON e.course_id = c.course_id;

-- 5. Show invoices with customer and amount
SELECT c.name, i.invoice_date, i.total FROM invoices i
INNER JOIN customers c ON i.customer_id = c.customer_id;
```


**LEFT JOIN (queries 6-10):**
```sql
-- 6. All customers, with orders if any
SELECT c.name, o.product FROM customers c
LEFT JOIN orders o ON c.customer_id = o.customer_id;

-- 7. Customers WITHOUT any orders
SELECT c.name FROM customers c
LEFT JOIN orders o ON c.customer_id = o.customer_id
WHERE o.order_id IS NULL;

-- 8. All products, show sales count (0 if never sold)
SELECT p.product_name, COUNT(oi.item_id) AS times_sold
FROM products p
LEFT JOIN order_items oi ON p.product_id = oi.product_id
GROUP BY p.product_name;

-- 9. All employees with COALESCE for missing department
SELECT e.name, COALESCE(d.dept_name, 'Unassigned') AS department
FROM employees e
LEFT JOIN departments d ON e.dept_id = d.dept_id;

-- 10. All categories, show products if any
SELECT cat.name, p.product_name FROM categories cat
LEFT JOIN products p ON cat.category_id = p.category_id;
```

**RIGHT JOIN and FULL JOIN (queries 11-15):**
```sql
-- 11. RIGHT JOIN equivalent of query 6
SELECT c.name, o.product FROM orders o
RIGHT JOIN customers c ON c.customer_id = o.customer_id;

-- 12. All departments, even those with no employees
SELECT d.dept_name, e.name FROM employees e
RIGHT JOIN departments d ON e.dept_id = d.dept_id;

-- 13. FULL JOIN: all customers and all orders
SELECT c.name, o.order_id FROM customers c
FULL OUTER JOIN orders o ON c.customer_id = o.customer_id;

-- 14. Find orphan orders (no matching customer)
SELECT o.order_id, o.product FROM customers c
RIGHT JOIN orders o ON c.customer_id = o.customer_id
WHERE c.customer_id IS NULL;

-- 15. SQLite FULL JOIN workaround
SELECT c.name, o.order_id FROM customers c
LEFT JOIN orders o ON c.customer_id = o.customer_id
UNION
SELECT c.name, o.order_id FROM orders o
LEFT JOIN customers c ON c.customer_id = o.customer_id;
```

| LEFT JOIN | All left + matching right | ខាងឆ org្វេងទាំងអស org់ + ស org្ដាំត org្រូវគ org្នា |

| Anti-join | Find unmatched rows | រកជួរដេកគ org្មានដូគូ |

## Speaking Practice
**Explain to a client:**
"There are four main ways to combine tables. INNER JOIN shows only matching data. LEFT JOIN shows all items from one side plus matches. RIGHT JOIN does the reverse. FULL JOIN shows everything. The choice depends on whether you need to see missing data or only confirmed connections."

## Reading Practice
Look at this query and identify the JOIN type and purpose:
```sql
SELECT d.name, COUNT(e.emp_id) AS headcount
FROM departments d
LEFT JOIN employees e ON d.dept_id = e.dept_id
GROUP BY d.name;
```
Answer: LEFT JOIN ensures all departments appear, even with 0 employees.
