# Day 50: HAVING - Filtering After Grouping

## Topic
Understanding HAVING clause to filter groups after GROUP BY. HAVING filters aggregated results (groups), while WHERE filters individual rows before grouping. You cannot use aggregate functions in WHERE — use HAVING instead.

## English Explanation
HAVING is like WHERE but for groups:

- **WHERE** filters individual rows BEFORE grouping happens
- **HAVING** filters groups AFTER GROUP BY and aggregation

Key rule: You cannot write `WHERE COUNT(*) > 5` — that is invalid. You must write `HAVING COUNT(*) > 5` because COUNT is calculated after grouping.

Query execution order: FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY

### HAVING with COUNT - Departments with More Than 2 Employees
```sql
-- Find departments with more than 2 employees
SELECT department, COUNT(*) AS emp_count
FROM employees
GROUP BY department
HAVING COUNT(*) > 2;
```
| department | emp_count |
|------------|-----------|
| IT         | 5         |
| Sales      | 3         |

(HR has only 2, so it is filtered out by HAVING)

### HAVING with SUM - Categories with High Total Salary
```sql
-- Departments where total salary exceeds 4000
SELECT department, SUM(salary) AS total_salary
FROM employees
GROUP BY department
HAVING SUM(salary) > 4000;
```
| department | total_salary |
|------------|-------------|
| IT         | 7050        |

### WHERE vs HAVING Together
```sql
-- Active employees only (WHERE), then departments with >2 active (HAVING)
SELECT department, COUNT(*) AS active_count
FROM employees
WHERE status = 'active'
GROUP BY department
HAVING COUNT(*) > 2;
```
| department | active_count |
|------------|-------------|
| IT         | 4           |

Explanation: WHERE removes inactive first (rows 7,10 gone). Then GROUP BY groups remaining. Then HAVING keeps only groups with count > 2.

### HAVING with AVG
```sql
-- Departments where average salary is above 1200
SELECT department, AVG(salary) AS avg_salary
FROM employees
GROUP BY department
HAVING AVG(salary) > 1200;
```
| department | avg_salary |
|------------|-----------|
| IT         | 1410      |

## Speaking Practice
**Explain to a client:**
"HAVING lets us filter summary results. For example, if you want to see only departments with more than 5 employees or product categories with total sales above $1000, we use HAVING. WHERE filters individual records before summarizing, HAVING filters the summaries themselves."

## Reading Practice
Read this query and predict the result:
```sql
SELECT department, COUNT(*) AS total, SUM(salary) AS payroll
FROM employees
WHERE status = 'active'
GROUP BY department
HAVING SUM(salary) > 2000
ORDER BY payroll DESC;
```
Answer: IT: total=4, payroll=6100 (top). Sales: total=2, payroll=1850 — but wait, 1850 < 2000 so Sales is filtered out. HR: total=2, payroll=2400 passes. Final result: IT (6100), HR (2400).
