# Day 53: Joins with WHERE and ORDER BY

## Topic
Combining JOIN queries with complex WHERE filtering and ORDER BY sorting to write complete, real-world queries.

## English Explanation
A JOIN alone gives you combined data from multiple tables, but in practice you almost always need to filter and sort the results. Today you learn to write full queries that use JOIN to connect tables, WHERE to filter specific rows, and ORDER BY to sort the output. The correct clause order in SQL is: SELECT, FROM, JOIN, WHERE, GROUP BY, HAVING, ORDER BY, LIMIT. Getting this order right is essential for writing correct queries.

## SQL Example
```sql
-- Full query: JOIN + WHERE + ORDER BY
SELECT
    c.first_name,
    c.last_name,
    o.order_date,
    o.total_amount
FROM customers c
INNER JOIN orders o ON c.customer_id = o.customer_id
WHERE o.order_date >= '2024-01-01'
  AND o.total_amount > 100
ORDER BY o.total_amount DESC, o.order_date ASC;

-- Multiple conditions across both tables
SELECT
    e.employee_name,
    d.department_name,
    e.salary
FROM employees e
INNER JOIN departments d ON e.dept_id = d.dept_id
WHERE d.department_name IN ('Sales', 'Marketing')
  AND e.salary > 50000
ORDER BY d.department_name, e.salary DESC;
```

3. Employees hired after 2023 in the 'Engineering' department, sorted by name

## Speaking Practice
Explain to a client:
"When I write a database query, I first connect the tables that have the data you need, then I filter out the rows you do not want, and finally I sort the results in the order that makes the most sense for your report. Each step narrows down the data so you only see exactly what is relevant."

## Reading Practice
Read this query and explain what it does step by step:
```sql
SELECT p.product_name, c.category_name, p.price
FROM products p
INNER JOIN categories c ON p.category_id = c.category_id
WHERE p.price BETWEEN 10 AND 100
  AND c.category_name != 'Discontinued'
ORDER BY c.category_name, p.price;
```
Identify: What tables are joined? What is filtered out? How is it sorted?
