# Day 46: SUM and AVG - Totals and Averages

## Topic
Understanding SUM(column) and AVG(column) aggregate functions. SUM adds all values in a column to produce a total. AVG calculates the arithmetic mean (average). Both functions automatically skip NULL values.

## English Explanation
SUM and AVG are aggregate functions that work with numeric columns:

- **SUM(column)** adds all non-NULL values together. It answers "what is the total?"
- **AVG(column)** calculates the mean by dividing the sum by the count of non-NULL values. It answers "what is the average?"

Key behavior: Both skip NULL values entirely. If you have values 100, 200, NULL, 300 then SUM = 600 and AVG = 200 (600 ÷ 3, not 600 ÷ 4). NULL rows are ignored, not treated as zero.

## SQL Example

### Setup Data
```sql
CREATE TABLE products (
    product_id INTEGER PRIMARY KEY,
    product_name TEXT NOT NULL,
    category TEXT,
    price REAL,
    quantity_sold INTEGER
);

INSERT INTO products VALUES (1, 'Laptop', 'Electronics', 800, 50);
INSERT INTO products VALUES (2, 'Mouse', 'Electronics', 25, 200);
INSERT INTO products VALUES (3, 'Desk', 'Furniture', 300, 30);
INSERT INTO products VALUES (4, 'Chair', 'Furniture', 150, NULL);
INSERT INTO products VALUES (5, 'Keyboard', 'Electronics', NULL, 120);
INSERT INTO products VALUES (6, 'Bookshelf', 'Furniture', 200, 15);
```


### Basic SUM Examples
```sql
-- Total revenue (price × quantity for all products)
SELECT SUM(price) AS total_price FROM products;
-- Result: 1475 (NULL is skipped, 800+25+300+150+200)

-- Total items sold
SELECT SUM(quantity_sold) AS total_sold FROM products;
-- Result: 415 (NULL is skipped, 50+200+30+120+15)

-- Total revenue per category
SELECT category, SUM(price) AS category_total
FROM products
WHERE category IS NOT NULL
GROUP BY category;
```
| category    | category_total |
|-------------|---------------|
| Electronics | 825           |
| Furniture   | 650           |

### Basic AVG Examples
```sql
-- Average price across all products
SELECT AVG(price) AS avg_price FROM products;
-- Result: 295 (1475 ÷ 5, NULL row excluded)

-- Average quantity sold
SELECT AVG(quantity_sold) AS avg_sold FROM products;
-- Result: 83 (415 ÷ 5, NULL row excluded)

-- Average price per category
SELECT category, AVG(price) AS avg_price
FROM products
WHERE category IS NOT NULL
GROUP BY category;
```
| category    | avg_price |
|-------------|-----------|
| Electronics | 412.5     |
| Furniture   | 216.67    |

### SUM and AVG Together
```sql
SELECT
    category,
    SUM(price) AS total_price,
    AVG(price) AS average_price,
    COUNT(price) AS products_with_price
FROM products
WHERE category IS NOT NULL
GROUP BY category;
```

## Speaking Practice
**Explain to a client:**
"SUM gives you the total of a column — for example, total revenue or total hours worked. AVG gives you the average — for example, average order value or average salary. Both ignore missing data, so your calculations are based only on actual values, not gaps in the data."

## Reading Practice
Read this query and predict the result:
```sql
SELECT
    SUM(price) AS total,
    AVG(price) AS average,
    COUNT(price) AS counted,
    COUNT(*) AS all_rows
FROM products;
```
Answer: total=1475, average=295, counted=5, all_rows=6. Notice COUNT(price)=5 because one price is NULL, but COUNT(*)=6 counts all rows.
