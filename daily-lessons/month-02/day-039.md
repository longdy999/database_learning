# Day 39: INNER JOIN - Matching Rows from Two Tables

## Topic
Learning the full INNER JOIN syntax to combine matching rows from two tables.

## English Explanation
INNER JOIN returns only the rows where there is a match in BOTH tables. If a customer has no orders, they will NOT appear in the results. If an order has no matching customer, it will NOT appear either. Think of INNER JOIN as finding the overlap — only rows that have a connection in both tables are included.

## SQL Example

```sql
-- Full INNER JOIN syntax:
SELECT columns
FROM table1
INNER JOIN table2 ON table1.id = table2.foreign_key;

-- Practical example: Get customer names with their orders
SELECT customers.name, orders.product, orders.amount
FROM customers
INNER JOIN orders ON customers.id = orders.customer_id;

-- Using table aliases for cleaner code
SELECT c.name, o.product, o.amount, o.order_date
FROM customers c
INNER JOIN orders o ON c.id = o.customer_id;


```sql
-- Adding WHERE clause to a JOIN
SELECT c.name, o.product, o.amount
FROM customers c
INNER JOIN orders o ON c.id = o.customer_id
WHERE o.amount > 100;

-- Adding ORDER BY to a JOIN
SELECT c.name, o.product, o.amount
FROM customers c
INNER JOIN orders o ON c.id = o.customer_id
ORDER BY o.amount DESC;

-- JOIN with aggregate function
SELECT c.name, COUNT(o.id) AS total_orders, SUM(o.amount) AS total_spent
FROM customers c
INNER JOIN orders o ON c.id = o.customer_id
GROUP BY c.name
ORDER BY total_spent DESC;
```

## Technical Vocabulary

| English Term | Khmer Term | Definition |
|---|---|---|
| INNER JOIN | ភ្ជាប់ខាងក្នុង | Returns only matching rows from both tables |
| ON clause | ឃ្លា ON | Specifies the join condition |
| Table alias | ឈ្មោះកាត់តារាង | Short name for a table (e.g., c for customers) |
| Join condition | លក្ខខណ្ឌភ្ជាប់ | The rule for matching rows between tables |
| Matching rows | ជួរផ្គូផ្គង | Rows that satisfy the join condition |

when you want a report showing customer names alongside their orders, we

## Reading Practice

```sql
SELECT c.name, c.email, o.product, o.amount, o.order_date
FROM customers c
INNER JOIN orders o ON c.id = o.customer_id
