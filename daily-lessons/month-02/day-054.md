# Day 54: Joins with GROUP BY and Aggregates

## Topic
Combining JOINs with GROUP BY and aggregate functions to create summary reports across multiple tables.

## English Explanation
Today you combine two powerful skills: joining tables and aggregating data. In real work, most reports require data from multiple tables summarized into groups. For example: "total spending per customer" requires joining customers with orders, then grouping by customer and summing the amounts. Or "number of products sold per category" requires joining products, order_items, and categories, then counting. The pattern is: JOIN tables first, then GROUP BY to summarize.

## SQL Example
```sql
-- Total spending per customer
SELECT
    c.first_name,
    c.last_name,
    COUNT(o.order_id) AS total_orders,
    SUM(o.total_amount) AS total_spent,
    AVG(o.total_amount) AS avg_order_value
FROM customers c
INNER JOIN orders o ON c.customer_id = o.customer_id
GROUP BY c.customer_id, c.first_name, c.last_name
HAVING SUM(o.total_amount) > 500
ORDER BY total_spent DESC;

-- Products sold per category
SELECT
    cat.category_name,
    COUNT(DISTINCT p.product_id) AS unique_products_sold,
    SUM(oi.quantity) AS total_units_sold,
    SUM(oi.quantity * p.price) AS total_revenue
FROM categories cat
INNER JOIN products p ON cat.category_id = p.category_id
INNER JOIN order_items oi ON p.product_id = oi.product_id
GROUP BY cat.category_id, cat.category_name
ORDER BY total_revenue DESC;
```

## Reading Practice
Read this query and explain the business question it answers:
```sql
SELECT d.department_name, COUNT(e.employee_id) AS headcount,
       AVG(e.salary) AS avg_salary, MAX(e.salary) AS top_salary
FROM departments d
INNER JOIN employees e ON d.dept_id = e.dept_id
GROUP BY d.dept_id, d.department_name
HAVING COUNT(e.employee_id) >= 5
ORDER BY avg_salary DESC;
```
