# Day 45: COUNT(*) and COUNT(column) - Counting Rows

## Topic
Understanding the difference between COUNT(*) and COUNT(column). COUNT(*) counts all rows including those with NULL. COUNT(column) skips rows where that column is NULL.

## English Explanation
COUNT is an aggregate function that counts rows. There are two main forms:

- **COUNT(*)** counts ALL rows in the result set, regardless of NULL values. It answers "how many rows exist?"
- **COUNT(column_name)** counts only rows where that specific column is NOT NULL. It answers "how many rows have a value in this column?"

This difference matters when your data has missing values. If a column contains 10 rows but 3 are NULL, COUNT(*) returns 10 but COUNT(column) returns 7.

## SQL Example

### Setup Data
```sql
CREATE TABLE employees (
    emp_id INTEGER PRIMARY KEY,
    name TEXT NOT NULL,
    email TEXT,
    department TEXT,
    salary REAL
);

INSERT INTO employees VALUES (1, 'Sokha', 'sokha@mail.com', 'IT', 1200);
INSERT INTO employees VALUES (2, 'Dara', NULL, 'IT', 1100);
INSERT INTO employees VALUES (3, 'Chenda', 'chenda@mail.com', NULL, 1000);
INSERT INTO employees VALUES (4, 'Bopha', 'bopha@mail.com', 'HR', NULL);
INSERT INTO employees VALUES (5, 'Vanna', NULL, NULL, 900);
```


### Comparing COUNT(*) vs COUNT(column)
```sql
-- Count ALL rows
SELECT COUNT(*) AS total_employees FROM employees;
-- Result: 5

-- Count rows where email is NOT NULL
SELECT COUNT(email) AS employees_with_email FROM employees;
-- Result: 3 (Dara and Vanna have NULL email)

-- Count rows where department is NOT NULL
SELECT COUNT(department) AS employees_with_dept FROM employees;
-- Result: 3 (Chenda and Vanna have NULL department)

-- Count rows where salary is NOT NULL
SELECT COUNT(salary) AS employees_with_salary FROM employees;
-- Result: 4 (Bopha has NULL salary)
```

### COUNT with GROUP BY
```sql
-- Count employees per department
SELECT department, COUNT(*) AS emp_count
FROM employees
WHERE department IS NOT NULL
GROUP BY department;
```
| department | emp_count |
|------------|-----------|
| IT         | 2         |
| HR         | 1         |

### COUNT DISTINCT
```sql
-- Count unique departments (excluding NULL)
SELECT COUNT(DISTINCT department) AS unique_depts FROM employees;
-- Result: 2 (IT and HR, NULL is not counted)
```

## Speaking Practice
**Explain to a client:**
"COUNT(*) tells you the total number of records in a table. COUNT(column) tells you how many records actually have data in that specific field. For example, if you have 100 customers but only 80 have email addresses, COUNT(*) gives 100 and COUNT(email) gives 80. This helps identify incomplete data."

## Reading Practice
Read this query and predict the result:
```sql
SELECT
    COUNT(*) AS total,
    COUNT(email) AS has_email,
    COUNT(*) - COUNT(email) AS missing_email
FROM employees;
```
Answer: total=5, has_email=3, missing_email=2
