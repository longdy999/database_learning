# Day 55: Query Planning — Thinking Before Writing

## Topic
A systematic step-by-step approach to planning complex SQL queries before writing them.

## English Explanation
Many beginners start writing SQL immediately and get confused. Professional developers plan first. Today you learn a 5-step method: (1) Identify what information the output needs, (2) Identify which tables contain that data, (3) Identify how those tables connect (foreign keys), (4) Write FROM and JOIN clauses first, (5) Then add SELECT, WHERE, GROUP BY, and ORDER BY. This "plan first, write second" approach prevents errors and helps you tackle complex queries with confidence.

## SQL Example
```sql
-- Business question: "Show me the top 5 customers by total spending
-- in 2024, but only those who ordered more than 3 times"

-- STEP 1: What do I need to show?
--   customer name, order count, total spending

-- STEP 2: Which tables have this data?
--   customers (name), orders (dates, amounts)

-- STEP 3: How do they connect?
--   customers.customer_id = orders.customer_id

-- STEP 4: Write FROM/JOIN first
-- FROM customers c
-- INNER JOIN orders o ON c.customer_id = o.customer_id

-- STEP 5: Add the rest
SELECT
    c.first_name,
    c.last_name,
    COUNT(o.order_id) AS order_count,
    SUM(o.total_amount) AS total_spent
FROM customers c
INNER JOIN orders o ON c.customer_id = o.customer_id
WHERE o.order_date >= '2024-01-01'

GROUP BY c.customer_id, c.first_name, c.last_name
HAVING COUNT(o.order_id) > 3
ORDER BY total_spent DESC
LIMIT 5;
```

## Hands-on Practice
Plan and write queries for these business questions. For each one, write out your 5-step plan BEFORE writing the SQL:

1. "Which departments have average salaries above $60,000?"
   - Tables needed: employees, departments
   - Connection: employees.dept_id = departments.dept_id

2. "Show all products that have never been ordered"
   - Tables needed: products, order_items
   - Connection: products.product_id = order_items.product_id
   - Hint: Use LEFT JOIN and check for NULL

3. "What is the monthly revenue for each product category in 2024?"
   - Tables needed: categories, products, order_items, orders
   - Connection: chain of foreign keys

4. "List customers who spent more than the average customer"
   - Tables needed: customers, orders
   - Hint: This may need a subquery

5. "Top 3 cities by number of active customers (with orders in last 90 days)"
   - Tables needed: customers, orders
   - Filter: order_date within last 90 days

## Speaking Practice
Explain to a client:
"Before I write any complex query, I follow a planning process. First I identify exactly what data you need to see, then I figure out which tables hold that data and how they connect. This planning step takes one minute but saves significant debugging time. It is the same as an architect drawing blueprints before building — the plan comes first."
