# Day 41: LEFT JOIN - Keeping All Rows from the Left Table

## Topic
LEFT JOIN — returning all rows from the left table even when there is no matching row in the right table.

## English Explanation
A LEFT JOIN returns every row from the left table (the first table listed). If a matching row exists in the right table, the columns are filled with data. If no match exists, the right-side columns appear as NULL. This is different from INNER JOIN, which only returns rows that have matches on both sides.

Think of it like a class roster: every student appears on the list, but only students who submitted homework have a grade next to their name. Students without homework still show up — their grade column is simply empty (NULL).

## SQL Example

### Sample Tables
```sql
-- customers table
CREATE TABLE customers (
    customer_id INTEGER PRIMARY KEY,
    name TEXT NOT NULL
);

INSERT INTO customers VALUES (1, 'Sokha');
INSERT INTO customers VALUES (2, 'Dara');
INSERT INTO customers VALUES (3, 'Chenda');
INSERT INTO customers VALUES (4, 'Bopha');

-- orders table
CREATE TABLE orders (
    order_id INTEGER PRIMARY KEY,
    customer_id INTEGER,
    product TEXT,
    amount REAL
);

INSERT INTO orders VALUES (101, 1, 'Laptop', 850.00);
INSERT INTO orders VALUES (102, 1, 'Mouse', 25.00);
INSERT INTO orders VALUES (103, 2, 'Keyboard', 45.00);
```


### LEFT JOIN Query
```sql
SELECT c.name, o.product, o.amount
FROM customers c
LEFT JOIN orders o ON c.customer_id = o.customer_id;
```

### Result
| name   | product  | amount |
|--------|----------|--------|
| Sokha  | Laptop   | 850.00 |
| Sokha  | Mouse    | 25.00  |
| Dara   | Keyboard | 45.00  |
| Chenda | NULL     | NULL   |
| Bopha  | NULL     | NULL   |

### Compare with INNER JOIN
```sql
SELECT c.name, o.product, o.amount
FROM customers c
INNER JOIN orders o ON c.customer_id = o.customer_id;
```
-- INNER JOIN result: only Sokha and Dara appear (3 rows)
-- LEFT JOIN result: all 4 customers appear (5 rows)

## Reading Practice
Read this query and predict the output:
```sql
SELECT e.name, d.department_name
FROM employees e
LEFT JOIN departments d ON e.dept_id = d.dept_id;
```
Question: If an employee has no department assigned (dept_id is NULL), what appears in department_name column?
Answer: NULL — because LEFT JOIN keeps all employees regardless of match.
