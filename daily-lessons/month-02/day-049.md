# Day 49: GROUP BY with Multiple Columns

## Topic
Using GROUP BY with multiple columns to create sub-groups. GROUP BY col1, col2 groups rows that share the same combination of values in both columns. This creates more detailed breakdowns.

## English Explanation
When you GROUP BY one column, you get one row per unique value. When you GROUP BY two or more columns, you get one row per unique combination.

For example:
- GROUP BY category gives: Electronics, Furniture
- GROUP BY category, month gives: Electronics-Jan, Electronics-Feb, Furniture-Jan, Furniture-Feb

This is powerful for detailed reports like "revenue by category per month" or "employee count by department and status."

## SQL Example

### Setup Data
```sql
CREATE TABLE orders (
    order_id INTEGER PRIMARY KEY,
    category TEXT NOT NULL,
    status TEXT NOT NULL,
    amount REAL NOT NULL,
    order_month TEXT NOT NULL
);

INSERT INTO orders VALUES (1, 'Electronics', 'completed', 800, '2024-01');
INSERT INTO orders VALUES (2, 'Electronics', 'completed', 200, '2024-01');
INSERT INTO orders VALUES (3, 'Electronics', 'pending', 150, '2024-02');
INSERT INTO orders VALUES (4, 'Furniture', 'completed', 500, '2024-01');
INSERT INTO orders VALUES (5, 'Furniture', 'cancelled', 300, '2024-02');
INSERT INTO orders VALUES (6, 'Furniture', 'completed', 400, '2024-02');
INSERT INTO orders VALUES (7, 'Electronics', 'completed', 600, '2024-02');
INSERT INTO orders VALUES (8, 'Furniture', 'pending', 250, '2024-01');
```


### GROUP BY Two Columns
```sql
-- Revenue by category AND month
SELECT category, order_month, SUM(amount) AS revenue, COUNT(*) AS orders
FROM orders
GROUP BY category, order_month
ORDER BY category, order_month;
```
| category    | order_month | revenue | orders |
|-------------|-------------|---------|--------|
| Electronics | 2024-01     | 1000    | 2      |
| Electronics | 2024-02     | 750     | 2      |
| Furniture   | 2024-01     | 750     | 2      |
| Furniture   | 2024-02     | 700     | 2      |

### GROUP BY Category and Status
```sql
-- Count orders by category and status
SELECT category, status, COUNT(*) AS order_count, SUM(amount) AS total
FROM orders
GROUP BY category, status
ORDER BY category, status;
```
| category    | status    | order_count | total |
|-------------|-----------|-------------|-------|
| Electronics | completed | 3           | 1600  |
| Electronics | pending   | 1           | 150   |
| Furniture   | cancelled | 1           | 300   |
| Furniture   | completed | 2           | 900   |
| Furniture   | pending   | 1           | 250   |

### Comparing Single vs Multiple GROUP BY
```sql
-- Single column: total per category
SELECT category, SUM(amount) AS total
FROM orders
GROUP BY category;
-- Result: Electronics=1750, Furniture=1450

-- Two columns: total per category per month
SELECT category, order_month, SUM(amount) AS total
FROM orders
GROUP BY category, order_month;
-- More detailed breakdown (4 rows instead of 2)
```

## Reading Practice
Read this query and predict the result:
```sql
SELECT category, status, COUNT(*) AS cnt
FROM orders
GROUP BY category, status
ORDER BY category, cnt DESC;
```
Answer: Electronics-completed: 3, Electronics-pending: 1, Furniture-completed: 2, Furniture-cancelled: 1, Furniture-pending: 1. Each unique combination of category+status gets its own row.
