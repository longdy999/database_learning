# Day 42: LEFT JOIN Practice and NULL Handling

## Topic
Using LEFT JOIN with WHERE clause to find unmatched rows, and using COALESCE to replace NULL values with meaningful defaults.

## English Explanation
One of the most powerful uses of LEFT JOIN is finding rows that do NOT have a match. After the LEFT JOIN, the unmatched rows have NULL in the right-table columns. By adding WHERE right_table.column IS NULL, we filter to show ONLY the unmatched rows. This pattern answers questions like "which customers have never ordered?" or "which products have never been sold?"

COALESCE is a function that returns the first non-NULL value from a list. It is useful for replacing NULL with a readable default like 'No order' or 0.

## SQL Example

### Finding Customers Without Orders
```sql
-- Pattern: LEFT JOIN + WHERE IS NULL
SELECT c.customer_id, c.name
FROM customers c
LEFT JOIN orders o ON c.customer_id = o.customer_id
WHERE o.order_id IS NULL;
```

### Result
| customer_id | name   |
|-------------|--------|
| 3           | Chenda |
| 4           | Bopha  |

### Using COALESCE to Replace NULL
```sql
SELECT
    c.name,
    COALESCE(o.product, 'No order') AS product,
    COALESCE(o.amount, 0) AS amount
FROM customers c
LEFT JOIN orders o ON c.customer_id = o.customer_id;
```


### Result with COALESCE
| name   | product  | amount |
|--------|----------|--------|
| Sokha  | Laptop   | 850.00 |
| Sokha  | Mouse    | 25.00  |
| Dara   | Keyboard | 45.00  |
| Chenda | No order | 0      |
| Bopha  | No order | 0      |

### More COALESCE Examples
```sql
-- Replace NULL email with 'not provided'
SELECT name, COALESCE(email, 'not provided') AS email
FROM customers;

-- Use first available contact
SELECT name, COALESCE(phone, email, 'no contact') AS contact
FROM customers;
```

## Reading Practice
Read and understand this real-world query:
```sql
SELECT p.product_name, p.price
FROM products p
LEFT JOIN order_items oi ON p.product_id = oi.product_id
WHERE oi.item_id IS NULL
ORDER BY p.price DESC;
```
Question: What does this query find?
Answer: Products that have never been ordered, sorted by price (most expensive first).
