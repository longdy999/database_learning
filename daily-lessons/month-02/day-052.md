# Day 52: Multiple Joins вҖ” 3 or More Tables

## Topic
Joining three or more tables in a single query using chained INNER JOINs.

## English Explanation
In real databases, data is spread across many tables. Often you need information from 3 or more tables at once. For example, to see what products a customer ordered, you need the customers table, the orders table, and the products table. You chain JOINs by writing INNER JOIN...ON for each relationship. The key is identifying which columns connect the tables вҖ” usually foreign keys pointing to primary keys.

## SQL Example
```sql
-- Joining 3 tables: customers + orders + products
SELECT
    c.first_name,
    c.last_name,
    o.order_date,
    p.product_name,
    p.price
FROM customers c
INNER JOIN orders o ON c.customer_id = o.customer_id
INNER JOIN order_items oi ON o.order_id = oi.order_id
INNER JOIN products p ON oi.product_id = p.product_id
ORDER BY c.last_name, o.order_date;
```

## Hands-on Practice
Create and query these three tables:
```sql
-- Table setup
CREATE TABLE categories (
    category_id INT PRIMARY KEY,
    category_name VARCHAR(50)
);

CREATE TABLE products (
    product_id INT PRIMARY KEY,
    product_name VARCHAR(100),
    category_id INT,

    price DECIMAL(10,2),
    FOREIGN KEY (category_id) REFERENCES categories(category_id)
);

CREATE TABLE order_items (
    item_id INT PRIMARY KEY,
    order_id INT,
    product_id INT,
    quantity INT,
    FOREIGN KEY (product_id) REFERENCES products(product_id)
);

-- Practice query: Join all three
SELECT
    cat.category_name,
    p.product_name,
    oi.quantity,
    p.price * oi.quantity AS line_total
FROM categories cat
INNER JOIN products p ON cat.category_id = p.category_id
INNER JOIN order_items oi ON p.product_id = oi.product_id;
```

## Speaking Practice
Explain to a client:
"Your data is organized in separate tables for good reasons вҖ” it avoids duplication and keeps things clean. When I need a report that combines customer names with their orders and product details, I join these tables together in one query. Think of it like connecting puzzle pieces вҖ” each table has a key that links to the next."
