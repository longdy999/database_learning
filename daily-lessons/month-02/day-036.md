# Day 36: DISTINCT - Removing Duplicate Values from Results

## Topic
Using DISTINCT to eliminate duplicate rows and find unique values in query results.

## English Explanation
When you query a database, you often get repeated values. For example, if you ask "What categories do my products belong to?" you might see "Electronics" appear 50 times. DISTINCT removes these duplicates and shows each value only once. Think of it like removing duplicate names from a guest list — you only want each name to appear once.

## SQL Example

```sql
-- Without DISTINCT (shows duplicates)
SELECT category FROM products;
-- Result: Electronics, Clothing, Electronics, Food, Clothing, Electronics...

-- With DISTINCT (unique values only)
SELECT DISTINCT category FROM products;
-- Result: Electronics, Clothing, Food, Books, Sports

-- Count how many unique categories exist
SELECT COUNT(DISTINCT category) AS unique_categories
FROM products;
-- Result: 5

-- DISTINCT with multiple columns
SELECT DISTINCT category, brand FROM products;

-- Find unique cities where customers live
SELECT DISTINCT city FROM customers
ORDER BY city;

-- Count unique customers who placed orders
SELECT COUNT(DISTINCT customer_id) AS active_customers
FROM orders;
```

## Speaking Practice

Explain to a client:
"When we look at your product database, DISTINCT helps us answer questions like
'How many different categories do we sell?' Without DISTINCT, we would see every
single product row. With DISTINCT, we get a clean list of unique categories only.
This is very useful for reports and dropdown menus in your application."


## Reading Practice

Read and understand this query:

```sql
SELECT COUNT(DISTINCT category) AS total_categories,
       COUNT(DISTINCT brand) AS total_brands,
       COUNT(*) AS total_products
FROM products
WHERE status = 'active';
```
