# Day 48: GROUP BY - Grouping Rows by Category

## Topic
Understanding GROUP BY to organize rows into groups based on column values. GROUP BY is used with aggregate functions (COUNT, SUM, AVG, MIN, MAX) to calculate summaries per group. Every non-aggregated column in SELECT must appear in GROUP BY.

## English Explanation
GROUP BY divides your result set into groups based on one or more columns. Then aggregate functions calculate a value for each group separately.

Key rules:
- Every column in SELECT that is NOT inside an aggregate function MUST be in GROUP BY
- GROUP BY comes after WHERE but before ORDER BY
- You can GROUP BY columns not in SELECT (but it is less common)

Example: If you GROUP BY department, then COUNT(*) counts per department, SUM(salary) totals per department, and AVG(salary) averages per department.

## SQL Example

### Setup Data
```sql
CREATE TABLE sales (
    sale_id INTEGER PRIMARY KEY,
    product_name TEXT NOT NULL,
    category TEXT NOT NULL,
    amount REAL NOT NULL,
    sale_date TEXT NOT NULL
);

INSERT INTO sales VALUES (1, 'Laptop', 'Electronics', 800, '2024-01-15');
INSERT INTO sales VALUES (2, 'Mouse', 'Electronics', 25, '2024-01-16');
INSERT INTO sales VALUES (3, 'Desk', 'Furniture', 300, '2024-01-17');
INSERT INTO sales VALUES (4, 'Chair', 'Furniture', 150, '2024-02-01');
INSERT INTO sales VALUES (5, 'Monitor', 'Electronics', 400, '2024-02-05');
INSERT INTO sales VALUES (6, 'Lamp', 'Furniture', 50, '2024-02-10');
INSERT INTO sales VALUES (7, 'Keyboard', 'Electronics', 60, '2024-03-01');
INSERT INTO sales VALUES (8, 'Sofa', 'Furniture', 700, '2024-03-15');
```

### GROUP BY with COUNT
```sql
-- Count sales per category
SELECT category, COUNT(*) AS sale_count
FROM sales
GROUP BY category;
```
| category    | sale_count |
|-------------|-----------|
| Electronics | 4         |
| Furniture   | 4         |

### GROUP BY with SUM and AVG
```sql
-- Total and average sale amount per category
SELECT
    category,
    SUM(amount) AS total_sales,
    AVG(amount) AS avg_sale,
    COUNT(*) AS num_sales
FROM sales
GROUP BY category;
```
| category    | total_sales | avg_sale | num_sales |
|-------------|-------------|----------|-----------|
| Electronics | 1285        | 321.25   | 4         |
| Furniture   | 1200        | 300      | 4         |


### GROUP BY with WHERE (filter before grouping)
```sql
-- Total sales per category, only for 2024-02
SELECT category, SUM(amount) AS feb_total
FROM sales
WHERE sale_date LIKE '2024-02%'
GROUP BY category;
```
| category    | feb_total |
|-------------|-----------|
| Electronics | 400       |
| Furniture   | 200       |

## Reading Practice
Read this query and predict the result:
```sql
SELECT category, COUNT(*) AS cnt, MAX(amount) AS biggest_sale
FROM sales
GROUP BY category
ORDER BY biggest_sale DESC;
```
Answer: Electronics: cnt=4, biggest_sale=800 (Laptop). Furniture: cnt=4, biggest_sale=700 (Sofa). Electronics appears first because 800 > 700.
