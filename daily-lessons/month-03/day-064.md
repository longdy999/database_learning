# Day 64: Junction/Bridge Tables for Many-to-Many

## Topic
Designing Junction Tables (Bridge Tables) for Many-to-Many Relationships

## English Explanation
A junction table (also called bridge table, linking table, or associative table) is the middle table that connects two tables in a many-to-many relationship. It contains foreign keys pointing to both parent tables and can also hold extra data about the relationship itself.

**Key design decisions for junction tables:**
- Use a composite primary key (both FKs together) OR a surrogate auto-increment PK
- Add extra columns for relationship metadata (date, quantity, status)
- Always add proper indexes for performance
- Consider what happens on DELETE (CASCADE is common)

## SQL Example

```sql
-- Parent table 1: orders
CREATE TABLE orders (
    order_id INT PRIMARY KEY AUTO_INCREMENT,
    customer_id INT NOT NULL,
    order_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    status VARCHAR(20) DEFAULT 'pending'
);

-- Parent table 2: products
CREATE TABLE products (
    product_id INT PRIMARY KEY AUTO_INCREMENT,
    product_name VARCHAR(100) NOT NULL,
    unit_price DECIMAL(10, 2) NOT NULL
);


-- Junction table: order_items (with extra columns!)
CREATE TABLE order_items (
    item_id INT PRIMARY KEY AUTO_INCREMENT,
    order_id INT NOT NULL,
    product_id INT NOT NULL,
    quantity INT NOT NULL DEFAULT 1,
    unit_price DECIMAL(10, 2) NOT NULL,
    discount_percent DECIMAL(5, 2) DEFAULT 0.00,
    FOREIGN KEY (order_id) REFERENCES orders(order_id)
        ON DELETE CASCADE,
    FOREIGN KEY (product_id) REFERENCES products(product_id)
        ON DELETE RESTRICT,
    UNIQUE (order_id, product_id)
);
```

**Notice:** `order_items` has extra columns (`quantity`, `unit_price`, `discount_percent`) that describe the relationship itself — not just which order contains which product, but HOW MANY and at WHAT PRICE.

## Hands-on Practice

Build an `actors ↔ movies` junction table:

```sql
CREATE TABLE actors (
    actor_id INT PRIMARY KEY AUTO_INCREMENT,
    full_name VARCHAR(100) NOT NULL,
    birth_year INT,
    nationality VARCHAR(50)
);

CREATE TABLE movies (
    movie_id INT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(200) NOT NULL,
    release_year INT,
    genre VARCHAR(50),
    director VARCHAR(100)
);

-- Junction table with relationship metadata
CREATE TABLE movie_cast (
    cast_id INT PRIMARY KEY AUTO_INCREMENT,
    actor_id INT NOT NULL,
    movie_id INT NOT NULL,
    role_name VARCHAR(100) NOT NULL,
    is_lead_role BOOLEAN DEFAULT FALSE,
    FOREIGN KEY (actor_id) REFERENCES actors(actor_id)
        ON DELETE CASCADE,
    FOREIGN KEY (movie_id) REFERENCES movies(movie_id)
        ON DELETE CASCADE,
    UNIQUE (actor_id, movie_id, role_name)
);


-- Insert sample data

```

## Technical Vocabulary

| English Term | Khmer Translation | Definition |
|---|---|---|
| Junction Table | តារាងភ្ជាប់ | Middle table connecting two tables in M:N |
| Bridge Table | តារាងស្ពាន | Synonym for junction table |
| Composite PK | សោចម្បងផ្សំ | PK made of two+ columns |
| Surrogate Key | សោជំនួស | Auto-generated ID |
| Relationship Metadata | ទិន្នន័យមេតានៃទំនាក់ទំនង | Extra data about connection |
| Linking Table | តារាងភ្ជាប់ | Synonym for junction table |


## Speaking Practice

Explain to a client:
> "When two things have a many-to-many relationship — like orders and products — we create a middle table to track the connections. This middle table can also store extra information, like how many of each product was ordered and at what price. It's like an invoice line item that connects the order to each product."

## Reading Practice

Read and understand this complex junction table query:

```sql
SELECT m.title, m.release_year, a.full_name, mc.role_name,
       CASE WHEN mc.is_lead_role THEN 'Lead' ELSE 'Supporting' END AS role_type
FROM movies m
INNER JOIN movie_cast mc ON m.movie_id = mc.movie_id
INNER JOIN actors a ON mc.actor_id = a.actor_id
WHERE m.title = 'The Matrix'
ORDER BY mc.is_lead_role DESC, a.full_name;
```

**Question:** What extra information does `movie_cast` provide beyond just linking actors to movies?
**Answer:** It stores the character name (`role_name`) and whether it's a lead role (`is_lead_role`).
