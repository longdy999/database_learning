# Day 72: Normalization Practice — Step by Step

## Topic
Complete Normalization Walkthrough: Denormalized Table → 1NF → 2NF → 3NF

## English Explanation
Today we take a completely flat, denormalized table and normalize it step by step through all three normal forms. This is the most important practical skill for database design.

**The Process:**
1. Start with a flat table (like a paper form with all data in one row)
2. Apply 1NF: Remove repeating groups, ensure atomic values, add a primary key
3. Apply 2NF: Remove partial dependencies (columns depending on only part of composite key)
4. Apply 3NF: Remove transitive dependencies (non-key depending on non-key)

**Worked Example:** A flat "Order Form" table:
- order_id, order_date, customer_id, customer_name, customer_city, product1_id, product1_name, product1_price, product1_qty, product2_id, product2_name, product2_price, product2_qty

After full normalization, this becomes 4 tables: customers, products, orders, order_items.

**ដំណើរការ:**
1. ចាប់ផ្ដើមពីតារាង flat (ដូចទម្រង់ក្រដាសដែលមានទិន្នន័យទាំងអស់ក្នុងជួរមួយ)
2. អនុវត្ត 1NF: លុប repeating groups ធានាតម្លៃ atomic បន្ថែម primary key
3. អនុវត្ត 2NF: លុប partial dependencies
4. អនុវត្ត 3NF: លុប transitive dependencies

**លទ្ធផល:** តារាង flat មួយក្លាយជា 4 តារាង: customers, products, orders, order_items។

## SQL Example

### Step 0 — Original Denormalized Table:
```sql
CREATE TABLE order_form_flat (
    order_id INT,
    order_date DATE,
    customer_id INT,
    customer_name VARCHAR(100),
    customer_city VARCHAR(100),
    product_id INT,
    product_name VARCHAR(100),
    product_price DECIMAL(10,2),
    quantity INT
);
```


### Step 1 — 1NF (atomic values, no repeating groups):
```sql
-- Already atomic in this version. Each row = one product per order.
-- Primary key: (order_id, product_id)
```

### Step 2 — 2NF (remove partial dependencies):
```sql
CREATE TABLE orders (
    order_id INT PRIMARY KEY,
    order_date DATE,
    customer_id INT,
    customer_name VARCHAR(100),
    customer_city VARCHAR(100)
);

CREATE TABLE order_items (
    order_id INT,
    product_id INT,
    product_name VARCHAR(100),
    product_price DECIMAL(10,2),
    quantity INT,
    PRIMARY KEY (order_id, product_id)
);
```

### Step 3 — 3NF (remove transitive dependencies):
```sql
CREATE TABLE customers (
    customer_id INT PRIMARY KEY,
    customer_name VARCHAR(100),
    customer_city VARCHAR(100)
);

CREATE TABLE products (
    product_id INT PRIMARY KEY,
    product_name VARCHAR(100),
    product_price DECIMAL(10,2)
);

CREATE TABLE orders (
    order_id INT PRIMARY KEY,
    order_date DATE,
    customer_id INT,
    FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
);

CREATE TABLE order_items (
    order_id INT,
    product_id INT,
    quantity INT,
    PRIMARY KEY (order_id, product_id),
    FOREIGN KEY (order_id) REFERENCES orders(order_id),
    FOREIGN KEY (product_id) REFERENCES products(product_id)
);
```

## Technical Vocabulary

| Term | Definition | Khmer |
|------|-----------|-------|
| Denormalized | All data in one flat table | ទិន្នន័យមិនបានរៀបចំ |
| Normalization Process | Step-by-step table decomposition | ដំណើរការ normalization |
| Composite Key | Primary key made of multiple columns | សោចម្រុះ |
| Partial Dependency | Column depends on part of composite key | ការពឹងផ្អែកផ្នែក |
| Decompose | Split one table into multiple related tables | បំបែកតារាង |

## Speaking Practice
"Let me walk you through the normalization process. We start with this flat order form table. First, we ensure 1NF — every cell has one value, no repeating groups. Then for 2NF, we separate columns that only depend on part of the key. Finally for 3NF, we remove any column that depends on a non-key column. The result is four clean, related tables."
