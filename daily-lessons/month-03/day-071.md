# Day 71: Third Normal Form (3NF)

## Topic
Third Normal Form (3NF) — Eliminating Transitive Dependencies

## English Explanation
Third Normal Form (3NF) builds on top of Second Normal Form (2NF). A table is in 3NF when:
1. It is already in 2NF (no partial dependencies)
2. There are NO transitive dependencies — meaning no non-key column depends on another non-key column

A **transitive dependency** happens when column A determines column B, and column B determines column C. So column C indirectly depends on column A through column B.

**Example:** In an employee table, `department_name` depends on `department_id`, not directly on `employee_id`. The `department_name` is transitively dependent through `department_id`.

**Fix:** Move the transitively dependent columns into their own table. Create a separate `departments` table with `department_id` and `department_name`.

## Khmer Explanation
Third Normal Form (3NF) គឺកសាងនៅលើ Second Normal Form (2NF)។ តារាងមួយស្ថិតក្នុង 3NF នៅពេលដែល:
1. វាស្ថិតក្នុង 2NF រួចហើយ (គ្មាន partial dependencies)
2. គ្មាន transitive dependencies — មានន័យថា គ្មាន non-key column ណាមួយពឹងផ្អែកលើ non-key column ផ្សេងទៀត

**Transitive dependency** កើតឡើងពេល column A កំណត់ column B ហើយ column B កំណត់ column C។ ដូច្នេះ column C ពឹងផ្អែកលើ column A តាមរយៈ column B ដោយប្រយោល។

**ឧទាហរណ៍:** ក្នុងតារាង employee, `department_name` ពឹងផ្អែកលើ `department_id` មិនមែនផ្ទាល់លើ `employee_id` ទេ។

**ដំណោះស្រាយ:** ផ្លាស់ទី columns ដែលពឹងផ្អែកដោយប្រយោលទៅតារាងផ្ទាល់ខ្លួន។ បង្កើតតារាង `departments` ដាច់ដោយឡែក។

## SQL Example

### Before 3NF (has transitive dependency):
```sql
CREATE TABLE employees_bad (
    employee_id INT PRIMARY KEY,
    employee_name VARCHAR(100),
    department_id INT,
    department_name VARCHAR(100),  -- depends on department_id, NOT employee_id
    department_location VARCHAR(100)  -- also depends on department_id
);
```

### After 3NF (transitive dependency removed):
```sql
CREATE TABLE departments (
    department_id INT PRIMARY KEY,
    department_name VARCHAR(100),
    department_location VARCHAR(100)
);


CREATE TABLE employees (
    employee_id INT PRIMARY KEY,
    employee_name VARCHAR(100),
    department_id INT,
    FOREIGN KEY (department_id) REFERENCES departments(department_id)
);
```

## Technical Vocabulary

| Term | Definition | Khmer |
|------|-----------|-------|
| Third Normal Form (3NF) | Table with no transitive dependencies | ទម្រង់ធម្មតាទីបី |
| Transitive Dependency | Non-key column depends on another non-key column | ការពឹងផ្អែកដោយប្រយោល |
| Non-key Column | A column that is not part of the primary key | Column ដែលមិនមែនជា key |
| Decomposition | Splitting a table into smaller tables | ការបំបែកតារាង |
| Data Redundancy | Same data stored in multiple places | ទិន្នន័យស្ទួន |
