# Day 63: Many-to-Many Relationships

## Topic
Many-to-Many Relationships in Database Design

## English Explanation
A many-to-many (M:N) relationship means many rows in Table A can relate to many rows in Table B. This relationship cannot be represented directly with just two tables — it requires a third table called a junction table (also called bridge table, linking table, or associative table).

For example: one student can enroll in many courses, and one course can have many students. One product can belong to many categories, and one category can contain many products.

**Why a junction table is needed:** A single foreign key column cannot hold multiple values. The junction table creates two 1:N relationships that together form the M:N relationship.

## SQL Example

```sql
-- Table A: students
CREATE TABLE students (
    student_id INT PRIMARY KEY AUTO_INCREMENT,
    first_name VARCHAR(50) NOT NULL,
    last_name VARCHAR(50) NOT NULL,
    email VARCHAR(100) UNIQUE
);

-- Table B: courses
CREATE TABLE courses (
    course_id INT PRIMARY KEY AUTO_INCREMENT,
    course_name VARCHAR(100) NOT NULL,
    credits INT DEFAULT 3,
    instructor VARCHAR(100)
);


-- Junction table: enrollments (connects students ↔ courses)
CREATE TABLE enrollments (
    enrollment_id INT PRIMARY KEY AUTO_INCREMENT,
    student_id INT NOT NULL,
    course_id INT NOT NULL,
    enrollment_date DATE DEFAULT (CURRENT_DATE),
    grade VARCHAR(2),
    FOREIGN KEY (student_id) REFERENCES students(student_id)
        ON DELETE CASCADE,
    FOREIGN KEY (course_id) REFERENCES courses(course_id)
        ON DELETE CASCADE,
    UNIQUE (student_id, course_id)  -- Prevent duplicate enrollments
);
```

**Structure:** students (1) → (N) enrollments (N) ← (1) courses

## Hands-on Practice

Design a M:N relationship between `products` and `categories`:

```sql
CREATE TABLE products (
    product_id INT PRIMARY KEY AUTO_INCREMENT,
    product_name VARCHAR(100) NOT NULL,
    price DECIMAL(10, 2) NOT NULL,
    stock_quantity INT DEFAULT 0
);

CREATE TABLE categories (
    category_id INT PRIMARY KEY AUTO_INCREMENT,
    category_name VARCHAR(50) NOT NULL UNIQUE,
    description TEXT
);

-- Junction table
CREATE TABLE product_categories (
    product_id INT NOT NULL,
    category_id INT NOT NULL,
    PRIMARY KEY (product_id, category_id),
    FOREIGN KEY (product_id) REFERENCES products(product_id)
        ON DELETE CASCADE,
    FOREIGN KEY (category_id) REFERENCES categories(category_id)
        ON DELETE CASCADE
);

## Technical Vocabulary

| English Term | Khmer Translation | Definition |
|---|---|---|
| Many-to-Many | ច្រើនទៅច្រើន | Multiple rows relate to multiple rows in both directions |
| Junction Table | តារាងភ្ជាប់ | Third table that connects two tables in M:N |
| Bridge Table | តារាងស្ពាន | Another name for junction table |
| Composite Key | សោផ្សំ | Primary key made of two or more columns |
| Associative Entity | អង្គភាពភ្ជាប់ | ER diagram term for junction table |
| UNIQUE Constraint | កម្រិតតែមួយគត់ | Prevents duplicate combinations |

In the database, we create a middle table that records each connection — like a signup sheet that links students to their courses."

## Reading Practice

Read and understand this query that retrieves M:N data:

```sql
SELECT s.first_name, s.last_name, c.course_name, e.grade
FROM students s
INNER JOIN enrollments e ON s.student_id = e.student_id
INNER JOIN courses c ON e.course_id = c.course_id
WHERE s.student_id = 1
ORDER BY c.course_name;
```

**Question:** How many JOINs are needed to connect students to courses?
**Answer:** Two JOINs — one from students to enrollments, another from enrollments to courses.
