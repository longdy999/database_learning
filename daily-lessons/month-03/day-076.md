# Day 76: Drawing ERDs from Business Requirements

## Topic
Drawing Entity-Relationship Diagrams (ERDs) from Business Requirements

## English Explanation
An ERD (Entity-Relationship Diagram) is a visual blueprint of a database. To draw an ERD from business requirements, you extract entities (nouns), relationships (verbs), and attributes (adjectives/details) from business text. This skill is essential for translating what clients say into a database structure.

**Step-by-step process:**
1. Read the business requirement carefully
2. Identify **nouns** → these become entities (tables)
3. Identify **verbs** → these become relationships between entities
4. Identify **adjectives/details** → these become attributes (columns)
5. Determine cardinality (one-to-one, one-to-many, many-to-many)
6. Draw the diagram with boxes (entities) and lines (relationships)

## SQL Example

**Business Requirement:** "A library has books. Members can borrow books."

```sql
-- Step 1: Identify entities: Library, Books, Members, Borrowing
-- Step 2: Identify relationships: "has" (Library-Books), "borrow" (Members-Books)
-- Step 3: Create tables from ERD

CREATE TABLE libraries (
    library_id SERIAL PRIMARY KEY,
    library_name VARCHAR(100) NOT NULL,
    address VARCHAR(200)
);

CREATE TABLE members (
    member_id SERIAL PRIMARY KEY,
    full_name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE,
    join_date DATE DEFAULT CURRENT_DATE
);

CREATE TABLE books (
    book_id SERIAL PRIMARY KEY,
    title VARCHAR(200) NOT NULL,
    author VARCHAR(100),
    isbn VARCHAR(13) UNIQUE,
    library_id INT REFERENCES libraries(library_id)
);

CREATE TABLE borrowings (
    borrowing_id SERIAL PRIMARY KEY,
    member_id INT REFERENCES members(member_id),
    book_id INT REFERENCES books(book_id),
    borrow_date DATE DEFAULT CURRENT_DATE,
    return_date DATE
);
```
