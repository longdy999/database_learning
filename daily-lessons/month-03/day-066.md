# Day 66: Referential Integrity and CASCADE

## Topic
Referential integrity rules and CASCADE effects on UPDATE and DELETE operations.

## English Explanation
Referential integrity means that every foreign key value must match a valid primary key in the referenced table. If you try to insert a FK value that does not exist as a PK, the database rejects it. CASCADE tells the database what to do automatically when you update or delete a referenced row — it can propagate the change to all related rows.

## SQL Example

```sql
-- Create parent table
CREATE TABLE departments (
    dept_id INT PRIMARY KEY,
    dept_name VARCHAR(50) NOT NULL
);

-- Create child table with CASCADE rules
CREATE TABLE employees (
    emp_id INT PRIMARY KEY,
    emp_name VARCHAR(100) NOT NULL,
    dept_id INT,
    FOREIGN KEY (dept_id) REFERENCES departments(dept_id)
        ON DELETE CASCADE
        ON UPDATE CASCADE
);

-- Insert valid data
INSERT INTO departments VALUES (1, 'Engineering'), (2, 'Marketing');
INSERT INTO employees VALUES (101, 'Sokha', 1), (102, 'Dara', 2);


-- This FAILS: referential integrity violation
INSERT INTO employees VALUES (103, 'Vanna', 99);
-- ERROR: dept_id 99 does not exist in departments

-- CASCADE DELETE: deleting department removes all its employees
DELETE FROM departments WHERE dept_id = 1;
-- Sokha is automatically deleted from employees

-- CASCADE UPDATE: updating PK propagates to FK
UPDATE departments SET dept_id = 20 WHERE dept_id = 2;
-- Dara's dept_id automatically changes to 20
```

## Technical Vocabulary

| Term | Definition |
|------|-----------|
| Referential Integrity | Rule ensuring FK values always match valid PKs |
| CASCADE | Automatically propagate changes to related rows |
| ON DELETE CASCADE | Delete child rows when parent row is deleted |
| ON UPDATE CASCADE | Update child FK when parent PK changes |
| ON DELETE SET NULL | Set FK to NULL when parent is deleted |
| Violation | Breaking a constraint rule |
| Orphan Record | Child row with FK pointing to non-existent parent |


## Speaking Practice
"Referential integrity ensures data consistency between related tables. When we use CASCADE on delete, removing a parent record automatically removes all child records. This prevents orphan data but must be used carefully — you could accidentally delete many rows."
