# Day 83: Designing a User/Authentication System

## Topic
Tables for users, roles, permissions, user_roles, and login_attempts with security considerations.

## English Explanation
An authentication system is one of the most critical database designs. It includes a `users` table storing credentials, a `roles` table defining access levels (admin, editor, viewer), a `permissions` table listing specific actions, a `user_roles` junction table for many-to-many role assignment, and a `login_attempts` table for security monitoring. Critical rule: NEVER store plain-text passwords. Always store hashed passwords using bcrypt or similar algorithms.

## Khmer Explanation
ប្រព័ន្ធផ្ទៀងផ្ទាត់អត្តសញ្ញាណ (authentication) គឺជាការរចនា database សំខាន់បំផុតមួយ។ វារួមបញ្ចូល `users` table សម្រាប់ credentials, `roles` table កំណត់កម្រិតចូលប្រើ, `permissions` table រាយសកម្មភាពជាក់លាក់, `user_roles` junction table សម្រាប់ many-to-many, និង `login_attempts` table សម្រាប់តាមដានសុវត្ថិភាព។ ច្បាប់សំខាន់: កុំរក្សាទុក password ជាអក្សរធម្មតាឡើយ។

## SQL Example
```sql
-- Users table (NEVER store plain passwords!)
CREATE TABLE users (
    user_id SERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    is_active BOOLEAN DEFAULT TRUE,
    is_verified BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW(),
    last_login TIMESTAMP
);

-- Roles table
CREATE TABLE roles (
    role_id SERIAL PRIMARY KEY,
    role_name VARCHAR(50) UNIQUE NOT NULL,
    description TEXT
);

-- Permissions table
CREATE TABLE permissions (
    permission_id SERIAL PRIMARY KEY,
    permission_name VARCHAR(100) UNIQUE NOT NULL,
    description TEXT
);

-- User-Roles junction (many-to-many)
CREATE TABLE user_roles (
    user_id INT REFERENCES users(user_id) ON DELETE CASCADE,
    role_id INT REFERENCES roles(role_id) ON DELETE CASCADE,
    assigned_at TIMESTAMP DEFAULT NOW(),
    PRIMARY KEY (user_id, role_id)
);


-- Role-Permissions junction
CREATE TABLE role_permissions (
    role_id INT REFERENCES roles(role_id) ON DELETE CASCADE,
    permission_id INT REFERENCES permissions(permission_id) ON DELETE CASCADE,
    PRIMARY KEY (role_id, permission_id)
);

-- Login attempts for security monitoring
CREATE TABLE login_attempts (
    attempt_id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(user_id),
    ip_address INET NOT NULL,
    attempted_at TIMESTAMP DEFAULT NOW(),
    success BOOLEAN NOT NULL,
    failure_reason VARCHAR(100)
);

-- Sample data
INSERT INTO roles (role_name, description) VALUES
('admin', 'Full system access'),
('editor', 'Can create and edit content'),
('viewer', 'Read-only access');

INSERT INTO permissions (permission_name, description) VALUES
('create_post', 'Can create new posts'),
('edit_post', 'Can edit existing posts'),
('delete_post', 'Can delete posts'),
('manage_users', 'Can manage user accounts');
```
