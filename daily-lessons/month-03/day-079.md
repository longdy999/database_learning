# Day 79: UNIQUE Constraints and Composite Keys

## Topic
Understanding UNIQUE Constraints on Single and Multiple Columns

## English Explanation
A UNIQUE constraint ensures no two rows have the same value in a column (or combination of columns). Unlike PRIMARY KEY, a table can have multiple UNIQUE constraints, and UNIQUE columns can contain NULL values.

**Key differences: UNIQUE vs PRIMARY KEY:**
- PRIMARY KEY: only one per table, cannot be NULL, identifies the row
- UNIQUE: multiple per table, can allow NULL, prevents duplicates

**Composite UNIQUE:** A unique constraint across multiple columns means the *combination* must be unique, not each column individually.

## Khmer Explanation
UNIQUE constraint ធានាថាមិនមានពីរជួរដេកមានតម្លៃដូចគ្នាក្នុងជួរឈរ (ឬបន្សំជួរឈរ)។ មិនដូច PRIMARY KEY ទេ តារាងអាចមាន UNIQUE constraints ច្រើន ហើយជួរឈរ UNIQUE អាចមាន NULL។

**ភាពខុសគ្នាសំខាន់: UNIQUE vs PRIMARY KEY:**
- PRIMARY KEY: មានតែមួយក្នុងតារាង មិនអាច NULL កំណត់ជួរដេក
- UNIQUE: មានច្រើនក្នុងតារាង អាចអនុញ្ញាត NULL ការពារស្ទួន

**Composite UNIQUE:** unique constraint លើជួរឈរច្រើនមានន័យថា *បន្សំ* ត្រូវតែ unique មិនមែនជួរឈរនីមួយៗទេ។

## SQL Example
```sql
-- Single column UNIQUE
CREATE TABLE users (
    user_id SERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    phone VARCHAR(20) UNIQUE
);


-- Composite UNIQUE (combination must be unique)
CREATE TABLE course_enrollments (
    enrollment_id SERIAL PRIMARY KEY,
    student_id INT NOT NULL,
    course_id INT NOT NULL,
    semester VARCHAR(20) NOT NULL,
    grade CHAR(1),
    UNIQUE(student_id, course_id, semester)
);

-- A student can enroll in the same course in DIFFERENT semesters
-- But NOT the same course in the same semester twice
INSERT INTO course_enrollments (student_id, course_id, semester)
VALUES (1, 101, 'Fall 2024');  -- OK

INSERT INTO course_enrollments (student_id, course_id, semester)
VALUES (1, 101, 'Spring 2025');  -- OK (different semester)

INSERT INTO course_enrollments (student_id, course_id, semester)
VALUES (1, 101, 'Fall 2024');  -- FAILS: duplicate combination

-- Named UNIQUE constraint
CREATE TABLE products (
    product_id SERIAL PRIMARY KEY,
    product_name VARCHAR(100) NOT NULL,
    sku VARCHAR(30),
    barcode VARCHAR(50),
    CONSTRAINT uq_product_sku UNIQUE(sku),
    CONSTRAINT uq_product_barcode UNIQUE(barcode)
);

-- Adding UNIQUE to existing table
ALTER TABLE products
    ADD CONSTRAINT uq_product_name_category
    UNIQUE(product_name, category_id);
```

## Speaking Practice
"UNIQUE constraints prevent duplicate data in our database. For example, two users cannot have the same email address. We can also make combinations unique — like ensuring a student cannot enroll in the same course twice in one semester, even though they can take it again in a different semester."
