# Day 75: ERD Notation and Symbols

## Topic
Entity-Relationship Diagram (ERD) — Reading and Understanding Diagram Notation

## English Explanation
An ERD (Entity-Relationship Diagram) is a visual representation of your database structure. It shows tables, their columns, and how they relate to each other.

**Core ERD Symbols:**
- **Entities** = Rectangles → represent tables (e.g., Customer, Order, Product)
- **Attributes** = Ovals or listed columns → represent fields in a table
- **Relationships** = Lines connecting entities → show how tables relate
- **Diamonds** = Relationship labels (in Chen notation)

**Crow's Foot Notation (most common in industry):**
- One: `|` (single vertical line)
- Many: `<<<` (crow's foot / fork shape)
- Zero or one: `o|`
- Zero or many: `o<<<`

**Cardinality Types:**
- **1:1** (One-to-One) — One person has one passport
- **1:N** (One-to-Many) — One customer has many orders
- **M:N** (Many-to-Many) — Students take many courses, courses have many students

## Khmer Explanation
ERD (Entity-Relationship Diagram) គឺជាតំណាងដែលមើលឃើញនៃរចនាសម្ព័ន្ធ database របស់អ្នក។ វាបង្ហាញតារាង columns របស់ពួកគេ និងរបៀបដែលពួកគេទាក់ទងគ្នា។

**និមិត្តសញ្ញា ERD សំខាន់:**
- **Entities** = ចតុកោណកែង → តំណាងតារាង
- **Attributes** = ពងក្រពើ ឬ columns រាយ → តំណាង fields ក្នុងតារាង
- **Relationships** = បន្ទាត់ភ្ជាប់ entities → បង្ហាញទំនាក់ទំនង
- **Diamonds** = ស្លាកទំនាក់ទំនង (ក្នុង Chen notation)

**Crow's Foot Notation (ប្រើច្រើនបំផុតក្នុងឧស្សាហកម្ម):**
- មួយ: `|` (បន្ទាត់បញ្ឈរតែមួយ)
- ច្រើន: `<<<` (រាងជើងក្អែក)

**ប្រភេទ Cardinality:**
- **1:1** — មនុស្សម្នាក់មាន passport មួយ
- **1:N** — អតិថិជនម្នាក់មានការបញ្ជាទិញច្រើន
- **M:N** — សិស្សរៀនមុខវិជ្ជាច្រើន មុខវិជ្ជាមានសិស្សច្រើន


## SQL Example

### 1:1 Relationship — Person and Passport:
```sql
CREATE TABLE persons (
    person_id INT PRIMARY KEY,
    person_name VARCHAR(100)
);

CREATE TABLE passports (
    passport_id INT PRIMARY KEY,
    passport_number VARCHAR(20) UNIQUE,
    person_id INT UNIQUE,  -- UNIQUE enforces 1:1
    FOREIGN KEY (person_id) REFERENCES persons(person_id)
);
```

### 1:N Relationship — Customer and Orders:
```sql
CREATE TABLE customers (
    customer_id INT PRIMARY KEY,
    customer_name VARCHAR(100)
);

CREATE TABLE orders (
    order_id INT PRIMARY KEY,
    order_date DATE,
    customer_id INT,  -- Many orders point to one customer
    FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
);
```

### M:N Relationship — Students and Courses (needs junction table):
```sql
CREATE TABLE students (
    student_id INT PRIMARY KEY,
    student_name VARCHAR(100)
);

CREATE TABLE courses (
    course_id INT PRIMARY KEY,
    course_name VARCHAR(100)
);

CREATE TABLE student_courses (
    student_id INT,
    course_id INT,
    enrollment_date DATE,
    PRIMARY KEY (student_id, course_id),
    FOREIGN KEY (student_id) REFERENCES students(student_id),
    FOREIGN KEY (course_id) REFERENCES courses(course_id)
);
```

## Technical Vocabulary

| Term | Definition | Khmer |
|------|-----------|-------|
| ERD | Visual diagram showing database structure | ដ្យាក្រាមទំនាក់ទំនង |
| Entity | A real-world object represented as a table | អង្គភាព (តារាង) |
| Cardinality | The number relationship between entities (1:1, 1:N, M:N) | ចំនួនទំនាក់ទំនង |
| Crow's Foot | Notation style using fork symbol for "many" | សញ្ញាជើងក្អែក |
| Junction Table | Table that resolves M:N relationships | តារាងភ្ជាប់ |
