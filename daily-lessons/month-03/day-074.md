# Day 74: Week 10 Review and Checkpoint

## Topic
Comprehensive Review — 1NF, 2NF, 3NF, Normalization Process, and Denormalization

## English Explanation
This is your Week 10 checkpoint. You have now learned all three normal forms, how to normalize from scratch, and when to break the rules with denormalization. Today is about testing yourself WITHOUT looking at your notes.

**What you should know by now:**
- 1NF: Atomic values, no repeating groups, every row unique
- 2NF: No partial dependencies (all non-key columns depend on the WHOLE primary key)
- 3NF: No transitive dependencies (no non-key depends on another non-key)
- Normalization process: Flat table → 1NF → 2NF → 3NF step by step
- Denormalization: When and why to intentionally break normal form rules

**Self-assessment:** Try to normalize a table from scratch without any notes. If you struggle, review Days 66-73 before moving forward.

## Hands-on Practice
**10 Practice Questions — Identify the Normal Form Violation:**

1. A table has a column "phone_numbers" containing "012-345, 098-765". Which NF is violated?
2. A table with composite key (student_id, course_id) has student_name depending only on student_id. Which NF?
3. A table has zip_code determining city_name, but zip_code is not the primary key. Which NF?
4. A table stores each product in its own column: product1, product2, product3. Which NF?
5. order_items table has product_price that depends on product_id (not on the full key). Which NF?
6. An employee table stores department_manager_name based on department_id. Which NF?
7. A table has one value per cell, a clear primary key, no repeating groups. Minimum which NF?
8. A 2NF table where country_name depends on country_code (not PK). What violation?
9. Should a real-time sales dashboard use normalized or denormalized data? Why?
10. A table is in 3NF but reports are slow due to 6 JOINs. What should you consider?


**Answer Key:**
1. 1NF — not atomic values
2. 2NF — partial dependency
3. 3NF — transitive dependency
4. 1NF — repeating groups
5. 2NF — partial dependency
6. 3NF — transitive dependency
7. 1NF (at minimum)
8. 3NF violation — transitive dependency
9. Denormalized — read-heavy, needs speed over integrity
10. Denormalization (summary tables or materialized views)

## Speaking Practice
"Let me explain the normalization levels. First Normal Form ensures every cell holds one value. Second Normal Form removes columns that only depend on part of the key. Third Normal Form removes columns that depend on other non-key columns. We always normalize first for data quality, but sometimes denormalize specific tables when performance is critical."
