# Day 69: First Normal Form (1NF)

## Topic
Rules of First Normal Form: atomic values, no repeating groups, and each row uniquely identifiable.

## English Explanation
A table is in First Normal Form (1NF) when: (1) every column contains only atomic (single, indivisible) values — no lists or comma-separated data, (2) there are no repeating groups of columns like phone1, phone2, phone3, and (3) each row can be uniquely identified. 1NF is the foundation — you must achieve it before moving to higher normal forms.

## Khmer Explanation
តារាងស្ថិតក្នុង First Normal Form (1NF) នៅពេល: (១) រាល់ column មានតែតម្លៃ atomic (តែមួយ មិនអាចបែងចែកបាន) — គ្មានបញ្ជី ឬទិន្នន័យបំបែកដោយសញ្ញាក្បៀស (២) គ្មានក្រុម column ដែលធ្វើម្ដងទៀតដូចជា phone1, phone2, phone3 និង (៣) រាល់ជួរដេកអាចកំណត់អត្តសញ្ញាណបានតែមួយគត់។

## SQL Example

```sql
-- NOT in 1NF: comma-separated values in one column
CREATE TABLE contacts_bad (
    contact_id INT PRIMARY KEY,
    name VARCHAR(100),
    phone_numbers VARCHAR(200)  -- stores "012345678, 098765432"
);

INSERT INTO contacts_bad VALUES
(1, 'Sokha', '012-111-111, 077-222-222'),
(2, 'Dara', '012-333-333, 088-444-444, 096-555-555');

-- NOT in 1NF: repeating column groups
CREATE TABLE contacts_bad2 (
    contact_id INT PRIMARY KEY,
    name VARCHAR(100),
    phone1 VARCHAR(20),
    phone2 VARCHAR(20),
    phone3 VARCHAR(20)  -- What if someone has 4 phones?
);


-- CORRECT 1NF: separate table for phone numbers
CREATE TABLE contacts (
    contact_id INT PRIMARY KEY,
    name VARCHAR(100)
);

CREATE TABLE phone_numbers (
    phone_id INT PRIMARY KEY,
    contact_id INT,
    phone_number VARCHAR(20),
    phone_type VARCHAR(20),
    FOREIGN KEY (contact_id) REFERENCES contacts(contact_id)
);

INSERT INTO contacts VALUES (1, 'Sokha'), (2, 'Dara');
INSERT INTO phone_numbers VALUES
(1, 1, '012-111-111', 'mobile'),
(2, 1, '077-222-222', 'work'),
(3, 2, '012-333-333', 'mobile'),
(4, 2, '088-444-444', 'home'),
(5, 2, '096-555-555', 'work');

-- Now each value is atomic, no repeating groups, each row is unique
```

## Technical Vocabulary

| Term | Definition |
|------|-----------|
| First Normal Form (1NF) | Table with atomic values, no repeating groups |
| Atomic Value | Single, indivisible piece of data in a cell |
| Repeating Group | Multiple columns storing the same type of data |
| Composite Attribute | Multi-valued field that should be split |
| Normal Form | A level of database table organization |
| Decomposition | Splitting a table into multiple related tables |
| Multi-valued | A cell containing more than one value |

## Speaking Practice
"This table violates First Normal Form because the phone_numbers column stores multiple values separated by commas. This makes it impossible to search for a specific phone number efficiently. The fix is to create a separate phone_numbers table where each phone number gets its own row."

## Reading Practice
Look at any CSV file or spreadsheet and check: Are there any cells with comma-separated lists? Any columns like item1, item2, item3? These are 1NF violations that need to be fixed before the data can be stored properly in a relational database.
