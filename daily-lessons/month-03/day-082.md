# Day 82: Common Database Design Patterns

## Topic
Soft delete, audit trail, status workflow, hierarchical data, and polymorphic relationships.

## English Explanation
Professional databases use common patterns to solve recurring problems. **Soft delete** uses an is_deleted flag instead of actually removing rows, preserving data history. **Audit trail** tracks who changed what and when using created_at, updated_at, and created_by columns. **Status workflow** manages state transitions (e.g., pending -> approved -> completed). **Hierarchical data** uses parent_id to create tree structures within a single table. **Polymorphic relationships** allow one table to relate to multiple other tables using a type column.

## SQL Example
```sql
-- Pattern 1: Soft Delete
CREATE TABLE products (
    product_id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    price DECIMAL(10,2) NOT NULL,
    is_deleted BOOLEAN DEFAULT FALSE,
    deleted_at TIMESTAMP NULL
);

-- Query active products only
SELECT * FROM products WHERE is_deleted = FALSE;

-- Soft delete a product
UPDATE products SET is_deleted = TRUE, deleted_at = NOW() WHERE product_id = 5;

-- Pattern 2: Audit Trail
CREATE TABLE orders (
    order_id SERIAL PRIMARY KEY,
    customer_id INT NOT NULL,
    total DECIMAL(10,2) NOT NULL,
    status VARCHAR(20) DEFAULT 'pending',
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW(),
    created_by INT,
    updated_by INT
);

-- Pattern 3: Status Workflow
CREATE TABLE tickets (
    ticket_id SERIAL PRIMARY KEY,
    title VARCHAR(200) NOT NULL,
    status VARCHAR(20) DEFAULT 'open'
        CHECK (status IN ('open','in_progress','resolved','closed')),
    priority VARCHAR(10) DEFAULT 'medium'
        CHECK (priority IN ('low','medium','high','critical'))
);


-- Pattern 4: Hierarchical Data (Self-referencing)
CREATE TABLE categories (
    category_id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    parent_id INT REFERENCES categories(category_id),
    level INT DEFAULT 0
);

-- Pattern 5: Polymorphic Relationships
CREATE TABLE comments (
    comment_id SERIAL PRIMARY KEY,
    body TEXT NOT NULL,
    commentable_type VARCHAR(50) NOT NULL,
    commentable_id INT NOT NULL,
    created_at TIMESTAMP DEFAULT NOW()
);
-- commentable_type can be 'post', 'photo', 'video'
```

## Technical Vocabulary

| Term | Definition | Khmer |
|------|-----------|-------|
| Soft delete | Marking a record as deleted without removing it | ការលុបទន់ |
| Audit trail | Record of all changes made to data | កំណត់ត្រាការផ্লាorg់ |
| Status workflow | Defined states and transitions for records | ដំណើរការស្ថានភាព |
| Hierarchical data | Tree-structured data with parent-child relations | ទិន្នន័យឋានានុក្រម |
| Polymorphic | One structure relating to many different types | ពហុរូបភាព |
