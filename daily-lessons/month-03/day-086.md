# Day 86: Naming Conventions and Documentation

## Topic
Professional naming conventions for tables, columns, and keys, plus database documentation best practices.

## English Explanation
Good naming conventions make databases readable and maintainable. Use plural lowercase with underscores for table names (users, order_items). Use singular descriptive names for columns (first_name, created_at). Foreign keys should reference the singular table name plus _id (customer_id). Documentation through SQL comments and README files helps teams understand your database.

## SQL Example
```sql
-- GOOD naming conventions
CREATE TABLE customers (
    customer_id SERIAL PRIMARY KEY,
    first_name VARCHAR(50) NOT NULL,
    last_name VARCHAR(50) NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE orders (
    order_id SERIAL PRIMARY KEY,
    customer_id INT REFERENCES customers(customer_id),
    order_date DATE NOT NULL,
    total_amount DECIMAL(10,2),
    status VARCHAR(20) DEFAULT 'pending'
);

-- BAD naming (avoid these)
-- CREATE TABLE tbl_Cust (CID int, FN varchar(50), dt datetime);
```

## Technical Vocabulary

| Term | Definition |
|------|-----------|
| Naming Convention | A set of rules for choosing names in code |
| Snake Case | Writing words with underscores: first_name |
| Plural Table Names | Tables named as collections: users, products |
| Foreign Key Naming | Pattern: referenced_table_singular_id |
| SQL Comment | Notes in code using -- or /* */ |
| README | Documentation file explaining a project |
| Self-documenting | Code that explains itself through clear names |
| Schema Documentation | Written description of database structure |

## Speaking Practice
"In our database, we follow strict naming conventions. Table names are plural and lowercase with underscores, like order_items. Column names are singular and descriptive, like first_name. Foreign keys follow the pattern of the referenced table in singular form plus _id, such as customer_id. We also maintain a README file that documents the purpose of each table."
