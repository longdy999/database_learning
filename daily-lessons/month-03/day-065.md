# Day 65: Foreign Keys - Syntax and Behavior

## Topic
Foreign Key Constraints: Full Syntax, ON DELETE, and ON UPDATE Behaviors

## English Explanation
A foreign key (FK) links two tables by referencing the primary key of another table. FK syntax includes options controlling what happens when the referenced row is modified or deleted:
- **RESTRICT:** Block delete/update if child rows exist
- **CASCADE:** Automatically delete/update child rows
- **SET NULL:** Set FK column to NULL in child rows
- **SET DEFAULT:** Set FK column to its default value

## Khmer Explanation (ការពន្យល់ជាភាសាខ្មែរ)
Foreign Key ភ្ជាប់តារាងពីរដោយយោងទៅ Primary Key នៃតារាងមេ។
- **RESTRICT:** ហាមលុប row មេ បើមាន row កូនភ្ជាប់នៅ
- **CASCADE:** លុប row កូនដោយស្វ័យប្រវត្តិ នៅពេល row មេត្រូវបានលុប
- **SET NULL:** កំណត់ column FK ទៅជា NULL នៅពេល row មេត្រូវបានលុប
- **SET DEFAULT:** កំណត់ column FK ទៅតម្លៃ default (កម្រប្រើ)

## SQL Example
```sql
CREATE TABLE departments (dept_id INT PRIMARY KEY, dept_name VARCHAR(50) NOT NULL);

CREATE TABLE emp_restrict (
    emp_id INT PRIMARY KEY, name VARCHAR(100), dept_id INT,
    FOREIGN KEY (dept_id) REFERENCES departments(dept_id) ON DELETE RESTRICT
);
CREATE TABLE emp_cascade (
    emp_id INT PRIMARY KEY, name VARCHAR(100), dept_id INT,
    FOREIGN KEY (dept_id) REFERENCES departments(dept_id) ON DELETE CASCADE
);
CREATE TABLE emp_setnull (
    emp_id INT PRIMARY KEY, name VARCHAR(100), dept_id INT,
    FOREIGN KEY (dept_id) REFERENCES departments(dept_id) ON DELETE SET NULL
);
```

## Hands-on Practice
```sql
CREATE TABLE categories (cat_id INT PRIMARY KEY, cat_name VARCHAR(50));
INSERT INTO categories VALUES (1,'Electronics'),(2,'Books');
CREATE TABLE products (
    id INT PRIMARY KEY, name VARCHAR(100), cat_id INT,
    FOREIGN KEY (cat_id) REFERENCES categories(cat_id) ON DELETE CASCADE
);
INSERT INTO products VALUES (1,'Laptop',1),(2,'Novel',2);
DELETE FROM categories WHERE cat_id = 1;  -- Products with cat_id=1 auto-deleted
SELECT * FROM products;                    -- Only 'Novel' remains
```

## Technical Vocabulary

| English Term | Khmer Translation | Definition |
|---|---|---|
| Foreign Key | សោបរទេស | Column referencing another table's primary key |
| RESTRICT | ហាមឃាត់ | Blocks delete/update if child rows exist |
| CASCADE | ឆ្លងបន្ត | Propagates delete/update to child rows |
| SET NULL | កំណត់ទទេ | Sets FK to NULL when parent is removed |
| Referential Integrity | សុចរិតភាពយោង | FK values must match a valid parent key |
| ON DELETE | នៅពេលលុប | Behavior when parent row is deleted |
| ON UPDATE | នៅពេលកែប្រែ | Behavior when parent key is updated |

## Speaking Practice
**Client asks:** "Why can't I delete this department?"
**Your answer:** "The system uses RESTRICT to protect data. You cannot delete a department while employees belong to it. Reassign employees first, or we can change it to SET NULL so employees stay but lose the department link."

## Reading Practice
```sql
CREATE TABLE order_items (
    item_id INT PRIMARY KEY, order_id INT NOT NULL, product_id INT,
    FOREIGN KEY (order_id) REFERENCES orders(order_id) ON DELETE CASCADE,
    FOREIGN KEY (product_id) REFERENCES products(product_id) ON DELETE SET NULL
);
```
**Identify:** Why CASCADE for order_id but SET NULL for product_id?
