# Day 147: Database Monitoring and Alerts

## Topic
Monitoring database health metrics and setting up alerts for critical thresholds.

## English Explanation
Database monitoring means continuously tracking key metrics like query execution time, active connections, disk space usage, lock waits, and deadlocks. When a metric exceeds a critical threshold, an alert notifies the team so they can respond before users are affected. PostgreSQL provides **pg_stat_statements** for query performance tracking, **pg_stat_activity** for connection monitoring, and system catalog views for disk usage. Good monitoring prevents outages and helps identify slow queries before they become problems.

## SQL Example

```sql
-- Check active connections and their states
SELECT state, COUNT(*) as connection_count
FROM pg_stat_activity
GROUP BY state
ORDER BY connection_count DESC;

-- Find long-running queries (over 5 minutes)
SELECT pid, now() - pg_stat_activity.query_start AS duration,
       query, state
FROM pg_stat_activity
WHERE (now() - pg_stat_activity.query_start) > INTERVAL '5 minutes'
AND state != 'idle'
ORDER BY duration DESC;

-- Check database size and growth
SELECT pg_database.datname AS database_name,
       pg_size_pretty(pg_database_size(pg_database.datname)) AS size
FROM pg_database
ORDER BY pg_database_size(pg_database.datname) DESC;

-- Check table sizes
SELECT schemaname, tablename,
       pg_size_pretty(pg_total_relation_size(schemaname || '.' || tablename)) AS total_size
FROM pg_tables
WHERE schemaname = 'public'
ORDER BY pg_total_relation_size(schemaname || '.' || tablename) DESC
LIMIT 10;
```
