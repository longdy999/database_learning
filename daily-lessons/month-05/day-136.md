# Day 136: Users, Roles, and Permissions

## Topic
Database user management, role-based access control, and permission assignment.

## English Explanation
Database security starts with controlling who can access what. Users are individual accounts that connect to the database. Roles are named groups of permissions that can be assigned to users. The principle of least privilege means giving each user only the minimum permissions they need to do their job — nothing more. A role hierarchy like admin → editor → viewer ensures clear separation of responsibilities.

## SQL Example
```sql
-- Create users
CREATE USER 'admin_user'@'localhost' IDENTIFIED BY 'StrongPass123!';
CREATE USER 'editor_user'@'localhost' IDENTIFIED BY 'EditPass456!';
CREATE USER 'viewer_user'@'localhost' IDENTIFIED BY 'ViewPass789!';

-- Create roles
CREATE ROLE admin_role;
CREATE ROLE editor_role;
CREATE ROLE viewer_role;

-- Grant permissions to roles
GRANT ALL PRIVILEGES ON company_db.* TO admin_role;
GRANT SELECT, INSERT, UPDATE ON company_db.* TO editor_role;
GRANT SELECT ON company_db.* TO viewer_role;

-- Assign roles to users
GRANT admin_role TO 'admin_user'@'localhost';
GRANT editor_role TO 'editor_user'@'localhost';
GRANT viewer_role TO 'viewer_user'@'localhost';

-- Revoke a permission
REVOKE INSERT ON company_db.* FROM editor_role;
```

## Technical Vocabulary

| Term | Definition |
|------|-----------|
| User | An individual database account with login credentials |
| Role | A named collection of permissions assignable to users |
| GRANT | SQL command to give permissions to a user or role |
| REVOKE | SQL command to remove permissions from a user or role |
| Privilege | A specific permission (SELECT, INSERT, UPDATE, DELETE) |
| Least Privilege | Security principle: give minimum necessary access |
| Role Hierarchy | Layered permission structure (admin > editor > viewer) |
| Authentication | Verifying identity (who are you?) |
| Authorization | Verifying permissions (what can you do?) |
| SHOW GRANTS | Command to display a user's current permissions |

## Speaking Practice
"In our database, we follow the principle of least privilege. Each team member has a role matching their responsibilities. Administrators have full access, editors can modify data but not structure, and viewers can only read. This protects our data from accidental or unauthorized changes."

## Reading Practice
Read the MySQL or PostgreSQL documentation on CREATE USER and GRANT statements. Study how role inheritance works — when a parent role has permissions, child roles can inherit them automatically.
