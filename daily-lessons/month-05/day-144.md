# Day 144: Database and APIs - REST Patterns

## Topic
How APIs interact with databases: mapping CRUD operations to HTTP verbs, writing pagination queries, and designing queries that power REST API endpoints.

## English Explanation
REST APIs are the bridge between frontend applications and databases. Each HTTP verb maps to a database operation: GET → SELECT (read data), POST → INSERT (create data), PUT → UPDATE (modify data), DELETE → DELETE (remove data). API responses often need pagination (showing 20 results at a time using LIMIT and OFFSET). Understanding this mapping helps you write efficient queries that serve API endpoints well, including filtering, sorting, and partial responses.

## SQL Example
```sql
-- GET /api/products?page=2&limit=20&sort=price&order=desc
-- Maps to: SELECT with pagination and sorting
SELECT id, name, price, category, created_at
FROM products
WHERE is_active = true
ORDER BY price DESC
LIMIT 20 OFFSET 20;  -- page 2: skip first 20 rows

-- GET /api/products/42
-- Maps to: SELECT single record
SELECT id, name, description, price, category, stock_quantity, created_at
FROM products
WHERE id = 42 AND is_active = true;

-- POST /api/products  (request body: {name, price, category})
-- Maps to: INSERT with RETURNING
INSERT INTO products (name, price, category, created_at)
VALUES ('New Widget', 29.99, 'electronics', NOW())
RETURNING id, name, price, category, created_at;

-- PUT /api/products/42  (request body: {name, price})
-- Maps to: UPDATE with RETURNING
UPDATE products
SET name = 'Updated Widget', price = 34.99, updated_at = NOW()
WHERE id = 42 AND is_active = true
RETURNING id, name, price, updated_at;

-- DELETE /api/products/42
-- Maps to: Soft delete (best practice)
UPDATE products
SET is_active = false, deleted_at = NOW()
WHERE id = 42
RETURNING id;

-- GET /api/products?category=electronics&min_price=10&max_price=100
-- Maps to: Filtered SELECT (dynamic WHERE)
SELECT id, name, price, category
FROM products
WHERE is_active = true
  AND category = 'electronics'
  AND price BETWEEN 10 AND 100
ORDER BY created_at DESC
LIMIT 20 OFFSET 0;
```

## Technical Vocabulary

| English Term | Khmer Translation | Definition |
|---|---|---|
| REST API | REST API | Architectural style for web services using HTTP verbs |
| CRUD | CRUD | Create, Read, Update, Delete operations |
| Pagination | ការបែងចែកទំព័រ | Splitting results into pages (LIMIT + OFFSET) |
| OFFSET | ចំណុចចាប់ផ្តើម | Number of rows to skip before returning results |
| LIMIT | កំណត់ចំនួន | Maximum rows to return |
| RETURNING | ត្រឡប់តម្លៃ | PostgreSQL clause that returns affected rows |
| Soft Delete | លុបទន់ | Marking record inactive instead of removing it |
| Endpoint | ចំណុចបញ្ចប់ | A specific URL path that handles API requests |

## Speaking Practice
"Our REST API maps directly to database operations. GET requests run SELECT queries with pagination using LIMIT and OFFSET. POST creates new records with INSERT RETURNING so the API can send back the new ID. We use soft deletes instead of hard deletes so we can recover data and maintain referential integrity."
