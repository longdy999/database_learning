# Day 126: Database Administration Basics

## Topic
Creating and managing databases, schemas, ownership, and authentication concepts.

## English Explanation
Database administration (DBA) involves creating databases, managing who owns them, controlling access, and configuring how users authenticate. In PostgreSQL, the `CREATE DATABASE` command makes a new database, and `DROP DATABASE` removes one permanently. Schemas organize objects within a database. The `pg_hba.conf` file controls client authentication вҖ” it defines which users can connect from which hosts using which methods.

## SQL Example

```sql
-- Create a new database
CREATE DATABASE company_hr;

-- Create a database with specific owner
CREATE DATABASE sales_db OWNER sales_admin;

-- List all databases (psql command)
-- \l

-- Switch to a database (psql command)
-- \c company_hr

-- Create a schema inside a database
CREATE SCHEMA hr_schema;

-- Create a table inside a specific schema
CREATE TABLE hr_schema.employees (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100),
    department VARCHAR(50)
);

-- Change database owner
ALTER DATABASE sales_db OWNER TO new_admin;

-- Drop a database (CAUTION: permanent!)
-- DROP DATABASE company_hr;
```
