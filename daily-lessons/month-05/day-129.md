# Day 129: Database Restore and Point-in-Time Recovery

## Topic
Restoring databases from backups and verifying data integrity.

## English Explanation
Restoring a database means loading data from a backup file back into a database system. In PostgreSQL, you use `psql` to restore plain SQL backups or `pg_restore` for custom-format backups. In MySQL, you use the `mysql` command to import SQL files. Always test your backups by restoring them to a separate database. Point-in-time recovery (PITR) allows restoring a database to a specific moment using WAL (Write-Ahead Log) files in PostgreSQL or binary logs in MySQL.

## SQL Example

```sql
-- PostgreSQL: Restore from plain SQL backup
-- psql -U username -d target_database < backup.sql

-- PostgreSQL: Restore to a NEW database
-- createdb -U username new_database
-- psql -U username -d new_database < backup.sql

-- PostgreSQL: Restore custom format backup
-- pg_restore -U username -d target_database backup.dump

-- PostgreSQL: Restore with clean (drop existing objects first)
-- pg_restore -U username -d target_database --clean backup.dump

-- MySQL: Restore from SQL backup
-- mysql -u username -p target_database < backup.sql

-- MySQL: Restore to a new database
-- mysql -u username -p -e "CREATE DATABASE new_database;"
-- mysql -u username -p new_database < backup.sql

-- Verify restoration: count rows
SELECT COUNT(*) FROM employees;
SELECT COUNT(*) FROM departments;

-- Verify restoration: check sample data
SELECT * FROM employees LIMIT 5;
```
