# Day 123: PostgreSQL CLI (psql) Essential Commands

## Topic
PostgreSQL CLI (psql) Essential Commands - Mastering the psql interactive terminal for efficient database navigation and management.

## English Explanation
The psql command-line tool is PostgreSQL's interactive terminal. It allows you to connect to databases, run SQL queries, and perform administrative tasks. psql has special backslash commands (meta-commands) that are not SQL but provide quick access to database information. Key commands include: `\l` to list all databases, `\dt` to list tables in current database, `\d tablename` to describe a table's structure, `\c dbname` to switch databases, `\i filename` to execute SQL from a file, and `\copy` to import/export CSV data. Learning psql is essential because it gives you direct control over your PostgreSQL server without needing a graphical tool.

## SQL Example
```sql
-- psql Meta-Commands Reference

-- Connect to PostgreSQL
psql -U postgres -h localhost -d mydb

-- List all databases
\l

-- Connect to a different database
\c another_database

-- List all tables in current database
\dt

-- List all tables with more detail (size, description)
\dt+

-- Describe a specific table structure
\d employees

-- Describe with more detail (storage, stats)
\d+ employees

-- List all schemas
\dn

-- List all indexes
\di

-- List all views
\dv
```

```sql
-- Execute SQL from a file
\i /path/to/setup.sql

-- Import CSV data into a table
\copy employees(name, email, salary) FROM 'data.csv' WITH CSV HEADER;

-- Export query results to CSV
\copy (SELECT * FROM employees WHERE salary > 2000) TO 'high_salary.csv' WITH CSV HEADER;

-- Show current connection info
\conninfo

-- Turn on/off expanded display (vertical format)
\x

-- Show query execution time
\timing

-- Get help on SQL commands
\h SELECT

-- Get help on psql commands
\?

-- Quit psql
\q
```

## Hands-on Practice
1. Connect to your PostgreSQL server using psql
2. Use `\l` to list all databases and note their sizes
3. Create a new database and switch to it with `\c`
4. Create 3 tables, then use `\dt` and `\d tablename` to explore them
5. Create a SQL file with INSERT statements and run it with `\i`
6. Export one table to CSV using `\copy`, then import it into a new table
7. Enable `\timing` and run several queries to see execution time
