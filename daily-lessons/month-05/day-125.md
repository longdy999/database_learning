# Day 125: Database Connection from Applications

## Topic
Database Connection from Applications - Understanding how software applications connect to databases using connection strings, drivers, ORMs, and connection pooling.

## English Explanation
Applications do not use psql or the command line to talk to databases. Instead, they use database drivers (libraries) that know how to communicate with the database server. A connection string contains all the information needed to connect: host address, port number, database name, username, and password. Connection pooling is a technique where the application maintains a set of pre-opened connections that can be reused, avoiding the overhead of opening a new connection for every query. ORMs (Object-Relational Mappers) like SQLAlchemy or Prisma provide a higher-level interface so developers can work with database tables as if they were programming objects. Critically, credentials should never be hardcoded in source code - always use environment variables.

## SQL Example
```sql
-- Connection String Anatomy
-- Format: protocol://username:password@host:port/database?options

-- PostgreSQL connection strings:
-- postgresql://postgres:mypassword@localhost:5432/mydb
-- postgresql://admin:secret@db.server.com:5432/production?sslmode=require

-- MySQL connection strings:
-- mysql://root:password@localhost:3306/mydb
-- mysql://user:pass@db.host.com:3306/app_db?charset=utf8mb4

-- SQLite connection (just a file path):
-- sqlite:///path/to/database.db

-- Connection string parts explained:
-- postgresql://  = protocol (which database type)
-- postgres       = username
-- mypassword     = password
-- localhost      = host (server address)
-- 5432           = port number (PostgreSQL default)
-- mydb           = database name
-- ?sslmode=require = connection options
```

```python
# Python example with psycopg2 (PostgreSQL driver)
import os
import psycopg2

# GOOD: credentials from environment variables
conn = psycopg2.connect(
    host=os.environ['DB_HOST'],
    port=os.environ['DB_PORT'],
    dbname=os.environ['DB_NAME'],
    user=os.environ['DB_USER'],
    password=os.environ['DB_PASSWORD']
)

# BAD: never hardcode credentials!
# conn = psycopg2.connect(host='localhost', password='secret123')

# Using connection pooling
from psycopg2 import pool
connection_pool = pool.SimpleConnectionPool(
    minconn=2,    # minimum connections kept open
    maxconn=10,   # maximum connections allowed
    host=os.environ['DB_HOST'],
    dbname=os.environ['DB_NAME'],
    user=os.environ['DB_USER'],
    password=os.environ['DB_PASSWORD']
)

# Get a connection from pool, use it, return it
conn = connection_pool.getconn()
cursor = conn.cursor()
cursor.execute("SELECT * FROM users WHERE active = TRUE")
rows = cursor.fetchall()
connection_pool.putconn(conn)  # return to pool
```

## Speaking Practice
"When an application connects to a database, it uses a connection string that includes the server address, port, database name, and credentials. We never put passwords directly in our code. Instead, we use environment variables. For high-traffic applications, we use connection pooling which keeps several connections ready to use, so we don't waste time opening new connections for every request."
