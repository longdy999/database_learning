# Day 130: Import/Export Data - CSV and SQL Files

## Topic
Using COPY, LOAD DATA INFILE, and export techniques for CSV and SQL files.

## English Explanation
Importing and exporting data is essential for moving data between systems, loading bulk data, and generating reports. PostgreSQL uses the COPY command to import from or export to CSV files. MySQL uses LOAD DATA INFILE for imports. Both systems can export query results to CSV format for use in spreadsheets or other applications.

## SQL Example

```sql
-- PostgreSQL: Import CSV into a table
COPY employees(name, email, department, salary)
FROM '/tmp/employees.csv'
DELIMITER ','
CSV HEADER;

-- PostgreSQL: Export table to CSV
COPY employees TO '/tmp/employees_export.csv'
DELIMITER ','
CSV HEADER;

-- PostgreSQL: Export query results to CSV
COPY (SELECT * FROM employees WHERE salary > 50000)
TO '/tmp/high_salary.csv'
DELIMITER ','
CSV HEADER;

-- MySQL: Import CSV into a table
LOAD DATA INFILE '/tmp/employees.csv'
INTO TABLE employees
FIELDS TERMINATED BY ','
ENCLOSED BY '"'
LINES TERMINATED BY '\n'
IGNORE 1 ROWS;

-- MySQL: Export to CSV
SELECT * FROM employees
INTO OUTFILE '/tmp/employees_export.csv'
FIELDS TERMINATED BY ','
ENCLOSED BY '"'
LINES TERMINATED BY '\n';
```
