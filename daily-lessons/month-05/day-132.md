# Day 132: Data Migration Between Databases

## Topic
Data Migration Between Databases — Moving data from SQLite to PostgreSQL, handling schema differences, and testing migration strategies.

## English Explanation
Data migration is the process of moving data from one database system to another. This commonly happens when a project outgrows SQLite and needs PostgreSQL or MySQL for production. Migration involves exporting data, converting schema definitions (because each database has different data types and syntax), transforming data formats, importing into the target system, and verifying everything transferred correctly. A good migration plan includes testing on a copy first, comparing row counts, and validating data integrity after the move.

## SQL Example

```sql
-- SQLite schema (source)
CREATE TABLE products (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    price REAL DEFAULT 0,
    created_at TEXT DEFAULT (datetime('now'))
);

-- PostgreSQL equivalent (target)
CREATE TABLE products (
    id SERIAL PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    price DECIMAL(10,2) DEFAULT 0.00,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Key differences to handle:
-- SQLite: INTEGER PRIMARY KEY AUTOINCREMENT
-- PostgreSQL: SERIAL PRIMARY KEY

-- SQLite: TEXT (no length limit)
-- PostgreSQL: VARCHAR(n) or TEXT

-- SQLite: REAL
-- PostgreSQL: DECIMAL(precision, scale)

-- SQLite: datetime('now')
-- PostgreSQL: CURRENT_TIMESTAMP
```

Plan a migration from SQLite to PostgreSQL for a sample application:

```sql
-- Step 1: Export from SQLite to CSV
-- Command: sqlite3 mydb.sqlite
-- .mode csv

-- Step 2: Create target table in PostgreSQL
CREATE TABLE products (
    id SERIAL PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    price DECIMAL(10,2) DEFAULT 0.00,
    category VARCHAR(100),
    stock_qty INTEGER DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Step 3: Import CSV into PostgreSQL
-- COPY products(name, price, category, stock_qty, created_at)
-- FROM '/tmp/products.csv'
-- DELIMITER ','
-- CSV HEADER;

-- Step 4: Verify migration
SELECT COUNT(*) AS total_rows FROM products;

-- Step 5: Check data integrity
SELECT
    MIN(price) AS min_price,
    MAX(price) AS max_price,
    COUNT(CASE WHEN name IS NULL THEN 1 END) AS null_names
FROM products;

-- Step 6: Reset sequence after import
SELECT setval('products_id_seq', (SELECT MAX(id) FROM products));
```

## Speaking Practice

**Explain to a client:**
"We need to move our database from SQLite to PostgreSQL because our application has grown beyond what SQLite can handle efficiently. I will create a migration plan that includes exporting all data, converting the schema to PostgreSQL format, importing the data, and verifying everything is correct. We will test on a copy first so there is no risk to our live data."
