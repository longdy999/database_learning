# Day 139: Audit Logging and Compliance

## Topic
Implementing audit logs for security monitoring, legal compliance, and tracking database changes.

## English Explanation
Audit logging records who did what, when, and from where in your database. This is critical for legal compliance (many industries require it), security monitoring (detecting unauthorized access), and debugging (understanding what changed and when). A good audit system captures the user, the action performed, the table affected, the old and new values, a timestamp, and the source IP address. Triggers are commonly used to automatically log changes without modifying application code.

## SQL Example
```sql
-- Create audit log table
CREATE TABLE audit_log (
    audit_id SERIAL PRIMARY KEY,
    table_name VARCHAR(100) NOT NULL,
    action_type VARCHAR(10) NOT NULL,  -- INSERT, UPDATE, DELETE
    record_id INTEGER NOT NULL,
    old_values JSONB,
    new_values JSONB,
    changed_by VARCHAR(100) NOT NULL,
    changed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    ip_address VARCHAR(45)
);

-- Create trigger function for automatic logging
CREATE OR REPLACE FUNCTION log_changes()
RETURNS TRIGGER AS $$
BEGIN
    IF TG_OP = 'INSERT' THEN
        INSERT INTO audit_log (table_name, action_type, record_id, new_values, changed_by)
        VALUES (TG_TABLE_NAME, 'INSERT', NEW.id, row_to_json(NEW)::jsonb, current_user);
    ELSIF TG_OP = 'UPDATE' THEN
        INSERT INTO audit_log (table_name, action_type, record_id, old_values, new_values, changed_by)
        VALUES (TG_TABLE_NAME, 'UPDATE', NEW.id, row_to_json(OLD)::jsonb, row_to_json(NEW)::jsonb, current_user);
    ELSIF TG_OP = 'DELETE' THEN
        INSERT INTO audit_log (table_name, action_type, record_id, old_values, changed_by)
        VALUES (TG_TABLE_NAME, 'DELETE', OLD.id, row_to_json(OLD)::jsonb, current_user);
    END IF;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;


-- Attach trigger to a table
CREATE TRIGGER employees_audit
    AFTER INSERT OR UPDATE OR DELETE ON employees
    FOR EACH ROW EXECUTE FUNCTION log_changes();

-- Query audit log: who changed what?
SELECT action_type, record_id, changed_by, changed_at, old_values, new_values
FROM audit_log
WHERE table_name = 'employees'
ORDER BY changed_at DESC
LIMIT 20;
```

## Technical Vocabulary

| Term | Definition |
|------|-----------|
| Audit Log | A record of all actions performed on the database |
| Compliance | Meeting legal or regulatory requirements |
| Trigger | A function that runs automatically on table events |
| JSONB | PostgreSQL binary JSON type for storing structured data |
| Data Retention | How long audit records must be kept |
| Immutable | Cannot be changed after creation (audit logs should be) |
| Chain of Custody | Tracking who accessed evidence/data and when |
| SOX Compliance | Financial reporting regulations requiring audit trails |
| GDPR | European data privacy regulation |
| Forensics | Investigating security incidents using logs |
