# Day 146: Scheduling and Automation (Cron Jobs, pg_cron)

## Topic
Database scheduling and automation using cron jobs and pg_cron for routine maintenance tasks.

## English Explanation
Databases need regular maintenance — vacuuming tables, cleaning old data, generating reports, and creating backups. Instead of doing these manually, we automate them using scheduling tools. In PostgreSQL, **pg_cron** is an extension that lets you schedule SQL commands to run at specific intervals, similar to Linux cron jobs. MySQL uses the **Event Scheduler** for the same purpose. Automation ensures consistency, reduces human error, and frees up time for other work.

## SQL Example

```sql
-- Install pg_cron extension (PostgreSQL)
CREATE EXTENSION pg_cron;

-- Schedule daily VACUUM at 3:00 AM
SELECT cron.schedule('nightly-vacuum', '0 3 * * *',
  'VACUUM ANALYZE customers;'
);

-- Schedule weekly cleanup of old logs every Sunday at 2:00 AM
SELECT cron.schedule('weekly-log-cleanup', '0 2 * * 0',
  $$DELETE FROM system_logs WHERE created_at < NOW() - INTERVAL '90 days';$$
);

-- Schedule monthly report generation on 1st of each month
SELECT cron.schedule('monthly-report', '0 6 1 * *',
  'INSERT INTO monthly_reports (report_date, total_orders, revenue)
   SELECT CURRENT_DATE, COUNT(*), SUM(total)
   FROM orders
   WHERE order_date >= DATE_TRUNC(''month'', CURRENT_DATE - INTERVAL ''1 month'')
   AND order_date < DATE_TRUNC(''month'', CURRENT_DATE);'
);

-- View all scheduled jobs
SELECT * FROM cron.job;

-- Unschedule a job
SELECT cron.unschedule('nightly-vacuum');
```
