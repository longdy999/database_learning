# Day 140: Network Security and Access Control

## Topic
Securing database network access with firewalls, SSL/TLS, IP whitelisting, and VPN/private networks.

## English Explanation
Even with strong user permissions, your database is vulnerable if anyone on the internet can try to connect to it. Network security creates layers of protection: firewalls block unauthorized ports, SSL/TLS encrypts all data traveling between your application and database, IP whitelisting allows only known servers to connect, and VPNs or private networks isolate your database from public internet entirely. Think of it like a building: user permissions are the door locks, but network security is the fence, security guard, and surveillance cameras around the building.

## SQL Example
```sql
-- PostgreSQL: Configure pg_hba.conf for IP whitelisting
-- (This is a configuration file, not SQL, but critical for security)
-- Example pg_hba.conf entries:
-- TYPE  DATABASE  USER        ADDRESS           METHOD
-- host  all       all         192.168.1.0/24    scram-sha-256
-- host  all       all         10.0.0.5/32       scram-sha-256
-- host  all       all         0.0.0.0/0         reject

-- Check current connections and their sources
SELECT pid, usename, client_addr, client_port, state, query_start
FROM pg_stat_activity
WHERE state = 'active'
ORDER BY query_start DESC;

-- Force SSL connections only (in postgresql.conf)
-- ssl = on
-- ssl_cert_file = '/path/to/server.crt'
-- ssl_key_file = '/path/to/server.key'

-- Verify SSL is being used for current connection
SELECT ssl, version, cipher FROM pg_stat_ssl
WHERE pid = pg_backend_pid();

-- Check failed connection attempts (if logging configured)
-- In postgresql.conf: log_connections = on, log_disconnections = on
```

## Technical Vocabulary

| Term | Definition |
|------|-----------|
| Firewall | Network barrier that blocks unauthorized traffic |
| Port | A numbered endpoint for network communication (e.g., 5432) |
| SSL/TLS | Protocols for encrypting network connections |
| IP Whitelisting | Only allowing specific IP addresses to connect |
| VPN | Virtual Private Network; creates encrypted tunnel |
| Private Network | Isolated network not accessible from public internet |
| pg_hba.conf | PostgreSQL Host-Based Authentication configuration file |
| CIDR | Notation for IP ranges (e.g., 192.168.1.0/24) |
| Attack Surface | Total number of points where an attacker can try to enter |
| Defense in Depth | Multiple security layers so one failure is not fatal |
