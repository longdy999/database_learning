# Day 124: MySQL Basics and Differences from PostgreSQL

## Topic
MySQL Basics and Differences from PostgreSQL - Understanding MySQL syntax, its storage engines, and key differences when writing queries for both systems.

## English Explanation
MySQL is another popular open-source relational database, owned by Oracle. It is widely used in web applications (WordPress, Facebook early days). Key differences from PostgreSQL include: MySQL uses AUTO_INCREMENT instead of SERIAL, uses backticks for identifier quoting instead of double quotes, has different default behaviors for GROUP BY, and uses InnoDB as its default storage engine (which supports transactions and foreign keys). While PostgreSQL focuses on standards compliance and advanced features, MySQL focuses on speed and simplicity. Many developers need to work with both systems, so understanding their syntax differences is a practical professional skill.

## SQL Example
```sql
-- MySQL Syntax vs PostgreSQL Syntax Comparison

-- ===== AUTO INCREMENT =====
-- PostgreSQL:
CREATE TABLE users_pg (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100)
);

-- MySQL:
CREATE TABLE users_mysql (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100)
) ENGINE=InnoDB;

-- ===== QUOTING IDENTIFIERS =====
-- PostgreSQL uses double quotes:
SELECT "column_name" FROM "table_name";

-- MySQL uses backticks:
SELECT `column_name` FROM `table_name`;

-- ===== LIMIT with OFFSET =====
-- Both support (same syntax):
SELECT * FROM users LIMIT 10 OFFSET 5;

-- ===== STRING CONCATENATION =====
-- PostgreSQL uses ||
SELECT first_name || ' ' || last_name AS full_name FROM users;

-- MySQL uses CONCAT()
SELECT CONCAT(first_name, ' ', last_name) AS full_name FROM users;
```

```sql
-- ===== BOOLEAN VALUES =====
-- PostgreSQL: TRUE / FALSE
SELECT * FROM users WHERE is_active = TRUE;

-- MySQL: 1 / 0 (or TRUE/FALSE as aliases)
SELECT * FROM users WHERE is_active = 1;

-- ===== CURRENT TIMESTAMP =====
-- PostgreSQL:
SELECT NOW(), CURRENT_TIMESTAMP;

-- MySQL:
SELECT NOW(), CURRENT_TIMESTAMP;  -- same!

-- ===== UPSERT (Insert or Update) =====
-- PostgreSQL:
INSERT INTO users (id, name) VALUES (1, 'Sokha')
ON CONFLICT (id) DO UPDATE SET name = EXCLUDED.name;

-- MySQL:
INSERT INTO users (id, name) VALUES (1, 'Sokha')
ON DUPLICATE KEY UPDATE name = VALUES(name);
```

## Speaking Practice
"The main differences between PostgreSQL and MySQL are in syntax details. PostgreSQL uses SERIAL for auto-increment while MySQL uses AUTO_INCREMENT. For string concatenation, PostgreSQL uses the double-pipe operator while MySQL uses the CONCAT function. Both are excellent databases, but PostgreSQL is more standards-compliant while MySQL is known for its speed in simple read-heavy workloads."
