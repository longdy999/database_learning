# Day 121: Introduction to PostgreSQL - Why It Matters

## Topic
Introduction to PostgreSQL - Understanding why PostgreSQL is the world's most advanced open-source relational database and how it differs from SQLite.

## English Explanation
PostgreSQL is a powerful, enterprise-grade relational database management system (RDBMS). Unlike SQLite which stores data in a single file, PostgreSQL runs as a client-server system. This means multiple users and applications can connect to it simultaneously. PostgreSQL supports advanced features like JSON data types, full-text search, custom functions, and complex transactions. It is used by companies like Instagram, Spotify, and Netflix for production workloads. The key differences from SQLite are: PostgreSQL requires installation and runs as a service, it handles concurrent users with MVCC (Multi-Version Concurrency Control), it enforces strict data types, and it scales to handle millions of rows efficiently.

## SQL Example
```sql
-- Connecting to PostgreSQL via psql command line
-- Step 1: Open terminal and connect
psql -U postgres -h localhost -p 5432

-- Step 2: List all databases
\l

-- Step 3: Create a new database
CREATE DATABASE my_first_pg_db;

-- Step 4: Connect to the new database
\c my_first_pg_db

-- Step 5: Create a table with PostgreSQL features
CREATE TABLE employees (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(255) UNIQUE,
    hire_date DATE DEFAULT CURRENT_DATE,
    salary NUMERIC(10, 2),
    is_active BOOLEAN DEFAULT TRUE,
    metadata JSONB
);

-- Step 6: Insert data
INSERT INTO employees (name, email, salary, metadata)
VALUES ('Sokha', 'sokha@company.com', 2500.00, '{"department": "IT"}');

-- Step 7: Query with PostgreSQL-specific casting
SELECT name, salary::TEXT || ' USD' AS salary_display
FROM employees
WHERE is_active = TRUE;
```
