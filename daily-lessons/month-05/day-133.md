# Day 133: Data Validation and Quality Checks

## Topic
Data Validation and Quality Checks — Checking data completeness, accuracy, and consistency using SQL validation queries.

## English Explanation
Data validation ensures that data in your database meets defined standards for quality. This includes checking completeness (are all required fields filled?), accuracy (are values within expected ranges?), and consistency (do related records agree with each other?). There are two approaches: constraint-based validation (using CHECK, NOT NULL, UNIQUE constraints to prevent bad data from entering) and query-based validation (writing queries to find existing problems). Good database professionals regularly run quality checks to catch issues before they affect reports or applications.

## SQL Example

```sql
-- Quality Check 1: Completeness - Find NULL required fields
SELECT COUNT(*) AS total_rows,
       COUNT(email) AS has_email,
       COUNT(*) - COUNT(email) AS missing_email
FROM customers;

-- Quality Check 2: COUNT(*) vs COUNT(column)
-- COUNT(*) counts all rows including NULLs
-- COUNT(column) counts only non-NULL values
SELECT
    COUNT(*) AS all_rows,
    COUNT(phone) AS rows_with_phone,
    COUNT(DISTINCT city) AS unique_cities
FROM customers;

-- Quality Check 3: Range validation
SELECT * FROM products
WHERE price < 0 OR price > 100000;

-- Quality Check 4: Format consistency
SELECT * FROM customers
WHERE email NOT LIKE '%@%.%';

-- Quality Check 5: Referential integrity check
SELECT o.id AS orphan_order_id
FROM orders o
LEFT JOIN customers c ON o.customer_id = c.id
WHERE c.id IS NULL;
```

## Hands-on Practice
Write 10 data quality check queries for a sample database:


```sql
-- Check 1: Overall table completeness
SELECT
    COUNT(*) AS total,
    COUNT(name) AS has_name,
    COUNT(email) AS has_email,
    COUNT(phone) AS has_phone,
    COUNT(address) AS has_address
FROM customers;

-- Check 2: Duplicate detection
SELECT email, COUNT(*) FROM customers
GROUP BY email HAVING COUNT(*) > 1;

-- Check 3: Date range validation
SELECT * FROM orders
WHERE order_date > CURRENT_DATE OR order_date < '2020-01-01';

-- Check 4: Numeric range check
SELECT * FROM products WHERE price <= 0 OR stock_qty < 0;

-- Check 5: String length validation
SELECT * FROM customers WHERE LENGTH(phone) < 9 OR LENGTH(phone) > 15;

-- Check 6: Cross-table consistency
SELECT p.id FROM payments p
LEFT JOIN orders o ON p.order_id = o.id
WHERE o.id IS NULL;

-- Check 7: Status field validation
SELECT * FROM orders
WHERE status NOT IN ('pending', 'shipped', 'delivered', 'cancelled');

-- Check 8: Freshness check - stale records
SELECT * FROM customers
WHERE last_login < CURRENT_DATE - INTERVAL '365 days';

-- Check 9: Aggregate sanity check
SELECT customer_id, SUM(amount) AS total_spent
FROM orders GROUP BY customer_id
HAVING SUM(amount) > 1000000;

-- Check 10: Constraint vs reality comparison
SELECT * FROM employees WHERE salary < 0 OR hire_date > CURRENT_DATE;
```
