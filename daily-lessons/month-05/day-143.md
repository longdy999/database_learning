# Day 143: Dashboard Data Queries

## Topic
Writing queries that power real-time dashboards, using materialized views for performance, and understanding data refresh strategies.

## English Explanation
Dashboards display live business data: sales today, active users, system health. The queries behind dashboards must be FAST because they run frequently. Materialized views pre-compute expensive queries and store results physically. You choose between scheduled refresh (update every 5 minutes) or on-demand refresh (update when data changes). Good dashboard queries are optimized with indexes, use materialized views for complex aggregations, and return data formatted for charts.

## SQL Example
```sql
-- Sales dashboard: Today's summary (must be fast)
SELECT
    COUNT(*) AS orders_today,
    COALESCE(SUM(total_amount), 0) AS revenue_today,
    COALESCE(AVG(total_amount), 0) AS avg_order_today,
    COUNT(DISTINCT customer_id) AS customers_today
FROM orders
WHERE order_date >= CURRENT_DATE;

-- Materialized view for heavy aggregation
CREATE MATERIALIZED VIEW mv_daily_sales AS
SELECT
    order_date::DATE AS sale_date,
    COUNT(*) AS order_count,
    SUM(total_amount) AS daily_revenue,
    AVG(total_amount) AS avg_order_value,
    COUNT(DISTINCT customer_id) AS unique_customers
FROM orders
GROUP BY order_date::DATE
ORDER BY sale_date DESC;

-- Create index on materialized view
CREATE UNIQUE INDEX idx_mv_daily_sales_date ON mv_daily_sales(sale_date);

-- Refresh materialized view (scheduled via cron or application)
REFRESH MATERIALIZED VIEW CONCURRENTLY mv_daily_sales;

-- Weekly trend query from materialized view (very fast)
SELECT sale_date, daily_revenue,
    AVG(daily_revenue) OVER (ORDER BY sale_date ROWS BETWEEN 6 PRECEDING AND CURRENT ROW)
        AS rolling_7day_avg
FROM mv_daily_sales
WHERE sale_date >= CURRENT_DATE - INTERVAL '30 days';
```

## Technical Vocabulary

| English Term | Khmer Translation | Definition |
|---|---|---|
| Dashboard | ផ្ទាំងគ្រប់គ្រង | Visual display of key metrics updated in real-time |
| Materialized View | ទិដ្ឋភាពរក្សាទុក | Pre-computed query results stored physically |
| Refresh | ធ្វើបច្ចុប្បន្នភាព | Update materialized view with latest data |
| CONCURRENTLY | ក្នុងពេលដំណាលគ្នា | Refresh without locking reads |
| Rolling Average | មធ្យមរំកិល | Average calculated over a moving window |
| Real-time | ពេលវេលាពិត | Data that updates immediately or near-immediately |
| Scheduled | កំណត់កាលវិភាគ | Running at fixed time intervals |
| On-demand | តាមតម្រូវការ | Running only when explicitly triggered |

## Speaking Practice
"Our dashboard uses materialized views for the heavy aggregations like monthly trends. The daily sales view refreshes every 5 minutes using a scheduled job. For today's live metrics, we query the orders table directly with an index on order_date. This gives us a balance between real-time data and query performance."
