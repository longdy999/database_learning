## SQL Example

```sql
-- CAPSTONE PROJECT: Online Learning Platform Database

-- 1. Schema with proper PostgreSQL types
CREATE TABLE students (
  id SERIAL PRIMARY KEY,
  full_name VARCHAR(100) NOT NULL,
  email VARCHAR(150) UNIQUE NOT NULL,
  profile JSONB DEFAULT '{}',
  enrolled_at TIMESTAMP DEFAULT NOW(),
  is_active BOOLEAN DEFAULT TRUE
);

CREATE TABLE courses (
  id SERIAL PRIMARY KEY,
  title VARCHAR(200) NOT NULL,
  description TEXT,
  difficulty_level VARCHAR(20) CHECK (difficulty_level IN ('beginner','intermediate','advanced')),
  price NUMERIC(8,2) NOT NULL DEFAULT 0,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE enrollments (
  id SERIAL PRIMARY KEY,
  student_id INT REFERENCES students(id) ON DELETE CASCADE,
  course_id INT REFERENCES courses(id) ON DELETE RESTRICT,
  enrolled_at TIMESTAMP DEFAULT NOW(),
  completed_at TIMESTAMP,
  progress_pct SMALLINT DEFAULT 0 CHECK (progress_pct BETWEEN 0 AND 100),
  UNIQUE(student_id, course_id)
);
```


## Hands-on Practice
Complete the following capstone deliverables:

**Part 1: Schema Design**
- Create at least 5 tables with proper relationships, constraints, and indexes

**Part 2: User Roles and Permissions**
- Create roles: db_admin, app_backend, report_viewer, readonly_user
- Grant appropriate permissions to each role

**Part 3: Backup Strategy Document**
- Daily incremental backups at 2 AM
- Weekly full backup on Sunday at 1 AM
- Retention: 7 daily, 4 weekly, 3 monthly
- Document restore procedure steps

**Part 4: Dashboard Reporting Queries (write 5)**
- Total revenue this month vs last month
- Top 10 most popular courses by enrollment
- Student completion rate by course
- Daily new enrollments trend (last 30 days)
- Revenue forecast based on current growth

**Part 5: Security Checklist**
- [ ] All tables have proper constraints
- [ ] No superuser access for application
- [ ] Sensitive data columns identified
- [ ] Row-level security considered
- [ ] Connection limits configured

**Part 6: API-Ready Patterns**
- Paginated course listing with filters
- Student progress endpoint query
- Search with full-text or ILIKE patterns
