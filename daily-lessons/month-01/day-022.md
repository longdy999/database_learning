# Day 22: UPDATE - Changing Existing Data

## Topic
UPDATE statement - modifying existing records in a database table.

## English Explanation
The UPDATE statement changes data that already exists in your table. You must ALWAYS use a WHERE clause with UPDATE, otherwise you will change EVERY row in the table. This is one of the most dangerous mistakes in SQL. Always verify your WHERE condition with a SELECT first.

## Khmer Explanation

## SQL Example
```sql
-- Basic UPDATE syntax
UPDATE products SET price = 29.99 WHERE id = 5;

-- Update multiple columns
UPDATE customers
SET email = 'new@email.com', phone = '012345678'
WHERE customer_id = 10;

-- DANGER: This changes ALL rows!
-- UPDATE products SET price = 0;  -- NEVER do this!

-- Safe pattern: SELECT first, then UPDATE
SELECT * FROM products WHERE category = 'Old';
UPDATE products SET category = 'Archive' WHERE category = 'Old';
```

## Speaking Practice
Explain to a client:
"UPDATE lets us change existing data in the database. For safety, I always check which rows will be affected using SELECT first, then run the UPDATE with a WHERE clause to target only the correct rows."

## Reading Practice
Read and understand this dangerous vs safe pattern:
- DANGEROUS: `UPDATE products SET price = 0;` (changes ALL rows)
- SAFE: `UPDATE products SET price = 0 WHERE id = 5;` (changes only one row)
- SAFEST: First run `SELECT * FROM products WHERE id = 5;` then UPDATE

## Review Task
Write 5 UPDATE statements from memory. For each one, write the corresponding SELECT statement you would run first to verify the WHERE condition.

## Expected Result
By end of today, you can safely update records using UPDATE with WHERE, and you understand why UPDATE without WHERE is dangerous.
