# Day 5: Setting Up SQLite - Your First Database

## Topic
Installing SQLite and creating your very first database file.

## English Explanation
**SQLite** is the simplest database system to start with. Unlike PostgreSQL or MySQL, it doesn't need a server — your entire database is just ONE file on your computer.

SQLite is perfect for learning because:
- No installation hassle (often already on your computer)
- No server to configure
- Your database is just a .db file
- Same SQL syntax you'll use in bigger databases later

## SQL Example
```sql
-- In terminal, create a new database:
-- $ sqlite3 my_first_database.db

-- You are now inside SQLite! Try these:

-- Create your first table
CREATE TABLE greetings (
    id INTEGER PRIMARY KEY,
    message TEXT,
    language TEXT
);

-- Insert some data
INSERT INTO greetings (message, language) VALUES ('Hello', 'English');
INSERT INTO greetings (message, language) VALUES ('សួស្តី', 'Khmer');
INSERT INTO greetings (message, language) VALUES ('こんにちは', 'Japanese');

-- Read the data
SELECT * FROM greetings;

-- Exit SQLite: type .quit
```


## Hands-on Practice (10-15 minutes)
1. Open your terminal
2. Type: `sqlite3 practice.db`
3. Create a table: `CREATE TABLE notes (id INTEGER PRIMARY KEY, title TEXT, content TEXT);`
4. Insert 3 rows of data
5. Run: `SELECT * FROM notes;`
6. Type `.tables` to see your tables
7. Type `.quit` to exit
8. Notice the `practice.db` file was created!

**If SQLite is not installed:**
- Mac: Already installed
- Linux: `sudo apt install sqlite3`
- Windows: Download from sqlite.org

## Speaking Practice

"For this project, we'll start with SQLite because it's the simplest to set up. Your entire database is just one file. Later, when we need more power — like multiple users accessing the database at the same time — we'll upgrade to PostgreSQL. But the SQL you learn now works in both systems."
