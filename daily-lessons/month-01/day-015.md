# Day 15: INSERT INTO - Adding Your First Records

## Topic
INSERT INTO: The SQL command for adding new data records into a table. Learn full syntax, single-row inserts, and named-column inserts.

## English Explanation
**INSERT INTO** is how you add new rows (records) to a table. There are two main forms:

1. **Full insert** (all columns in order):
```
INSERT INTO table_name VALUES (value1, value2, value3);
```

2. **Named columns** (specify which columns, in any order):
```
INSERT INTO table_name (col1, col3) VALUES (value1, value3);
```

The named-column form is preferred in professional work because:
- It is clearer to read
- Column order does not matter
- You can skip columns with DEFAULT values
- It still works if new columns are added to the table later

## SQL Example
```sql
-- First, create a products table to practice with
CREATE TABLE products (
    product_id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    category TEXT NOT NULL DEFAULT 'General',
    price REAL NOT NULL CHECK(price > 0),
    stock INTEGER NOT NULL DEFAULT 0,
    is_available INTEGER NOT NULL DEFAULT 1
);

-- Method 1: Named columns (RECOMMENDED - professional style)
INSERT INTO products (name, category, price, stock)
VALUES ('Wireless Mouse', 'Electronics', 25.99, 50);

INSERT INTO products (name, category, price, stock)
VALUES ('USB Keyboard', 'Electronics', 35.00, 30);

-- Method 2: Relying on DEFAULT values (only specify required fields)
INSERT INTO products (name, price)
VALUES ('Mystery Item', 9.99);
-- Result: category='General', stock=0, is_available=1

-- Method 3: Multiple considerations
INSERT INTO products (name, category, price, stock, is_available)
VALUES ('Old Laptop', 'Electronics', 299.99, 2, 0);

-- Verify all inserts
SELECT * FROM products;
```

## Technical Vocabulary

| Term | Meaning | Khmer |

## Speaking Practice
"INSERT INTO is how we add new data to a table. I prefer using named columns because it makes the code clearer and easier to maintain. When I write INSERT INTO products with column names in parentheses, anyone reading the code knows exactly which values go into which columns. If a column has a DEFAULT value, I can skip it and the database fills it in automatically."

## Reading Practice
Read each INSERT statement in the practice section above. For each one:
1. Which columns are specified?
2. Which columns are relying on DEFAULT values?
3. What will the complete row look like after insertion?
4. Would the INSERT succeed or fail if you removed a required column?

Practice reading INSERT statements as sentences: "Insert into shop_products, setting name to Notebook, category to Stationery, price to 2.50..."
