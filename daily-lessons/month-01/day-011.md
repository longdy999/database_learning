# Day 11: CREATE TABLE вҖ” Multiple Columns and Constraints (NOT NULL, UNIQUE, CHECK)

## Topic
CREATE TABLE with advanced constraints вҖ” NOT NULL, UNIQUE, CHECK вҖ” to enforce data integrity rules at the table level.

## English Explanation
**Constraints** are rules you attach to columns to prevent bad data from entering your database. Today we focus on three powerful constraints:
- **NOT NULL** вҖ” the column must always have a value (cannot be empty)
- **UNIQUE** вҖ” no two rows can have the same value in this column
- **CHECK** вҖ” a custom rule that validates data before allowing it in

Using multiple constraints together makes your tables strong and professional. A well-constrained table rejects invalid data automatically.

## Khmer Explanation (бһҖбһ¶бһҡбһ–бһ“бҹ’бһҷбһӣбҹӢбһҮбһ¶бһ—бһ¶бһҹбһ¶бһҒбҹ’бһҳбҹӮбһҡ)
Constraints (бһҖбҹҶбһ бһ·бһҸ) бһӮбһәбһҮбһ¶бһ…бҹ’бһ”бһ¶бһ”бҹӢбһҠбҹӮбһӣбһҖбһ¶бһҡбһ–бһ¶бһҡбһ‘бһ·бһ“бҹ’бһ“бһ“бҹҗбһҷбһҳбһ·бһ“бһҸбҹ’бһҡбһ№бһҳбһҸбҹ’бһҡбһјбһңбһ–бһёбһҖбһ¶бһҡбһ…бһјбһӣбһ‘бҹ…бһҖбҹ’бһ“бһ»бһ„ databaseбҹ” бһҗбҹ’бһ„бҹғбһ“бҹҒбҹҮбһҷбһҫбһ„бһҡбҹҖбһ“ бҹЈ constraints бһҹбҹҶбһҒбһ¶бһ“бҹӢбҹ–
- **NOT NULL** вҖ” бһҮбһҪбһҡбһҲбһҡбһ“бҹҒбҹҮбһҸбҹ’бһҡбһјбһңбһҸбҹӮбһҳбһ¶бһ“бһҸбһҳбҹ’бһӣбҹғбһҮбһ¶бһ“бһ·бһ…бҹ’бһ… бһҳбһ·бһ“бһўбһ¶бһ…бһ‘бһ‘бҹҒбһ”бһ¶бһ“
- **UNIQUE** вҖ” бһӮбҹ’бһҳбһ¶бһ“бһҮбһҪбһҡбһҠбҹҒбһҖбһ–бһёбһҡбһўбһ¶бһ…бһҳбһ¶бһ“бһҸбһҳбҹ’бһӣбҹғбһҠбһјбһ…бһӮбҹ’бһ“бһ¶бһҖбҹ’бһ“бһ»бһ„бһҮбһҪбһҡбһҲбһҡбһ“бҹҒбҹҮ
- **CHECK** вҖ” бһ…бҹ’бһ”бһ¶бһ”бҹӢбһ•бҹ’бһ‘бһ¶бһӣбҹӢбһҒбҹ’бһӣбһҪбһ“бһҠбҹӮбһӣбһ–бһ·бһ“бһ·бһҸбҹ’бһҷбһ‘бһ·бһ“бҹ’бһ“бһ“бҹҗбһҷбһҳбһ»бһ“бһ–бҹҒбһӣбһўбһ“бһ»бһүбҹ’бһүбһ¶бһҸбһұбҹ’бһҷбһ…бһјбһӣ

Constraints бһҮбһҪбһҷбһ’бҹ’бһңбһҫбһұбҹ’бһҷбһҸбһ¶бһҡбһ¶бһ„бһҡбһ”бһҹбҹӢбһўбҹ’бһ“бһҖбһҡбһ№бһ„бһҳбһ¶бҹҶ бһ“бһ·бһ„бһҳбһ¶бһ“бһңбһ·бһҮбҹ’бһҮбһ¶бһҮбһёбһңбҹҲбҹ”

## SQL Example
```sql
-- A professional table with multiple constraints
CREATE TABLE students (
    student_id INTEGER PRIMARY KEY AUTOINCREMENT,
    full_name TEXT NOT NULL,
    email TEXT UNIQUE NOT NULL,
    age INTEGER NOT NULL CHECK(age > 0 AND age < 120),
    phone TEXT UNIQUE,
    gpa REAL CHECK(gpa >= 0.0 AND gpa <= 4.0),
    enrollment_status TEXT NOT NULL DEFAULT 'active'
        CHECK(enrollment_status IN ('active', 'inactive', 'graduated'))
);


-- This INSERT works (valid data)
INSERT INTO students (full_name, email, age, phone, gpa)
VALUES ('Sovann Pich', 'sovann@school.edu', 20, '012-345-678', 3.5);

-- This INSERT fails: age is negative (CHECK violation)
INSERT INTO students (full_name, email, age)
VALUES ('Dara Sok', 'dara@school.edu', -5);
-- ERROR: CHECK constraint failed

-- This INSERT fails: duplicate email (UNIQUE violation)
INSERT INTO students (full_name, email, age)
VALUES ('Another Student', 'sovann@school.edu', 22);
-- ERROR: UNIQUE constraint failed

-- This INSERT fails: missing full_name (NOT NULL violation)
INSERT INTO students (email, age)
VALUES ('test@school.edu', 19);
-- ERROR: NOT NULL constraint failed
```

## Hands-on Practice (10-15 minutes)
Create this table and test each constraint:
```sql
-- Create a table with all 3 constraint types
CREATE TABLE employees (
    emp_id INTEGER PRIMARY KEY AUTOINCREMENT,
    full_name TEXT NOT NULL,
    email TEXT UNIQUE NOT NULL,
    age INTEGER NOT NULL CHECK(age > 0 AND age <= 65),
    salary REAL NOT NULL CHECK(salary >= 0),
    department TEXT NOT NULL,
    employee_code TEXT UNIQUE NOT NULL
);


-- Test 1: Insert valid data (should succeed)
INSERT INTO employees (full_name, email, age, salary, department, employee_code)
VALUES ('Sokha Meas', 'sokha@company.com', 28, 1500.00, 'Engineering', 'EMP001');

-- Test 2: Try age = 0 (should fail - CHECK)
INSERT INTO employees (full_name, email, age, salary, department, employee_code)
VALUES ('Test User', 'test@company.com', 0, 1000.00, 'HR', 'EMP002');

-- Test 3: Try duplicate email (should fail - UNIQUE)
INSERT INTO employees (full_name, email, age, salary, department, employee_code)
VALUES ('Dara Kim', 'sokha@company.com', 25, 1200.00, 'Sales', 'EMP003');

-- Test 4: Try NULL full_name (should fail - NOT NULL)
INSERT INTO employees (email, age, salary, department, employee_code)
VALUES ('dara@company.com', 30, 1100.00, 'Marketing', 'EMP004');
```

## Speaking Practice
"This table uses three types of constraints to protect data quality. NOT NULL ensures every employee has a name and email. UNIQUE prevents duplicate emails and employee codes. CHECK validates that age is between 1 and 65, and salary cannot be negative. If any rule is broken, the database rejects the data automatically."

## Reading Practice
Read the CREATE TABLE statement above carefully. For each column, identify:
1. Which constraints are applied?
2. What invalid data would each constraint reject?
3. Can you explain why each constraint makes sense for this column?

Practice reading error messages: "CHECK constraint failed" means your data broke a CHECK rule. "UNIQUE constraint failed" means a duplicate value was rejected.
