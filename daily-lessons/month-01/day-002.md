# Day 2: Tables, Rows, and Columns

## Topic
Understanding the three building blocks of every database table.

## English Explanation
Every database is made of **tables**. Each table stores one type of thing (customers, products, orders, etc.).

- **Table** = A collection of related data (like one spreadsheet tab)
- **Column** = A category of information (like a spreadsheet header: "name", "email", "age")
- **Row** = One complete record (like one line in a spreadsheet: "Sokha, sokha@email.com, 25")

**Important rule:** Each column holds only ONE type of data. The "age" column only holds numbers. The "name" column only holds text.

## Khmer Explanation (ការពន្យល់ជាភាសាខ្មែរ)
Database រាល់មួយត្រូវបានបង្កើតឡើងពី **tables (តារាង)**។ តារាងនីមួយៗរក្សាទុកប្រភេទរបស់មួយ (អតិថិជន ផលិតផល ការបញ្ជាទិញ។ល។)

- **Table (តារាង)** = ការប្រមូលផ្តុំទិន្នន័យដែលពាក់ព័ន្ធគ្នា
- **Column (ជួរឈរ)** = ប្រភេទព័ត៌មានមួយ (ដូចជា "ឈ្មោះ" "អ៊ីមែល" "អាយុ")
- **Row (ជួរដេក)** = កំណត់ត្រាពេញមួយ (ទិន្នន័យមួយបន្ទាត់)

## SQL Example
```sql
-- A "products" table might look like this:
-- Table name: products
--
-- COLUMNS:  id  |  product_name  |  price  |  category
-- ROW 1:    1   |  Laptop        |  899.99 |  Electronics
-- ROW 2:    2   |  Desk Chair    |  249.50 |  Furniture
-- ROW 3:    3   |  Notebook      |  4.99   |  Stationery

-- To see all columns and rows:
SELECT * FROM products;

-- To see only specific columns:
SELECT product_name, price FROM products;
```

## Technical Vocabulary

| Term | Meaning | Khmer |
|------|---------|-------|
| Table | Structured collection of related data | តារាង |
| Column | Vertical category (field) | ជួរឈរ |
| Row | Horizontal record (entry) | ជួរដេក |
| Field | Same as column - one piece of data | វាល |

## Speaking Practice
**Say this out loud as if explaining to a client:**

"Your customer data will be stored in a table called 'customers'. Each column represents one piece of information — like name, email, and phone number. Each row is one customer. So if you have 500 customers, you'll have 500 rows in that table."

## Reading Practice
Read and understand:

> "A well-designed table has clear column names that describe exactly what data they hold. Each row should represent one unique entity — one customer, one product, one order. Columns should not contain mixed types of data."
