# Day 1: What Is a Database?

## Topic
Understanding what a database is and why every business needs one.

## English Explanation
A **database** is an organized collection of data stored electronically. Think of it like a super-powered spreadsheet that can store millions of records and find any piece of information in milliseconds.

Before databases, people stored information on paper or in simple files. This was slow, error-prone, and impossible to search quickly. Databases solve this by organizing data into **tables** (like spreadsheets), with **rows** (individual records) and **columns** (categories of information).

**Example:** A school database stores student names, grades, classes, and attendance — all organized so any teacher can find any student's information instantly.

## SQL Example
```sql
-- This is what data looks like inside a database table:
-- Table: students
-- +----+----------+-----+-------+
-- | id | name     | age | grade |
-- +----+----------+-----+-------+
-- | 1  | Sokha    | 18  | A     |
-- | 2  | Dara     | 19  | B     |
-- | 3  | Chantha  | 17  | A     |
-- +----+----------+-----+-------+

-- We use SQL to talk to the database:
SELECT * FROM students;
```

## Speaking Practice
**Say this out loud as if explaining to a client:**

"A database is like a smart filing cabinet for your business data. Instead of searching through papers, the computer can find any information you need in less than a second. We organize your data into tables — think of them like spreadsheets — where each row is one record and each column is one type of information."
