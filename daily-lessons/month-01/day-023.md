# Day 23: DELETE - Removing Data Safely

## Topic
DELETE statement - removing records from a database table safely.

## English Explanation
The DELETE statement removes rows from a table. Like UPDATE, you must ALWAYS use a WHERE clause. DELETE without WHERE removes ALL rows from the table permanently. Always use SELECT first to verify which rows will be deleted. In professional work, many teams use "soft delete" (setting a status column to 'deleted') instead of actually removing data.

## SQL Example
```sql
-- Basic DELETE syntax
DELETE FROM products WHERE id = 5;

-- Delete with condition
DELETE FROM orders WHERE status = 'cancelled' AND order_date < '2023-01-01';

-- DANGER: This deletes ALL rows!
-- DELETE FROM products;  -- NEVER do this without WHERE!

-- Safe pattern: always SELECT first
SELECT * FROM products WHERE id = 5;
-- Verify the result, then:
DELETE FROM products WHERE id = 5;

-- Soft delete alternative (professional pattern)
UPDATE products SET status = 'deleted' WHERE id = 5;
```

## Speaking Practice
Explain to a client:
"DELETE removes data permanently. In our system, we use soft delete — we mark records as deleted but keep the data for safety. If you need to truly remove data, we always verify the selection first and create a backup."

## Reading Practice
Read and understand when to use each approach:
- Hard DELETE: Test data, temporary records, legal requirements (GDPR)
- Soft DELETE: Customer data, orders, anything you might need to recover
- TRUNCATE TABLE: Remove ALL rows quickly (even more dangerous than DELETE)

## Review Task

## Expected Result
