# Day 3: Real-World Database Examples

## Topic
Seeing how databases are used in real businesses and applications you use every day.

## English Explanation
Databases are everywhere. Every app on your phone, every website you visit, every business you interact with uses at least one database.

**Real examples:**
- **Facebook/Instagram:** Users table, posts table, comments table, likes table
- **Online shopping (Amazon):** Products table, customers table, orders table, payments table
- **Hospital:** Patients table, doctors table, appointments table, medicines table
- **Bank:** Accounts table, transactions table, customers table
- **School:** Students table, teachers table, classes table, grades table

Each of these systems has MULTIPLE tables that work together. This is called a **relational database** — tables are related to each other.

## SQL Example
```sql
-- A simple school database has multiple tables:

-- Table: students
-- id | name    | age | class_id
-- 1  | Sokha   | 16  | 101
-- 2  | Dara    | 17  | 102

-- Table: classes  
-- id  | class_name     | teacher_name
-- 101 | Math Grade 10  | Mr. Vann
-- 102 | Science Grade 11| Ms. Leap

-- The class_id in students CONNECTS to id in classes
-- This is how tables relate to each other!

SELECT * FROM students;
SELECT * FROM classes;
```

## Speaking Practice
**Say this out loud as if explaining to a client:**

"For your coffee shop, we would create separate tables for your menu items, your customers, and your orders. Each table stores one type of information. They connect to each other — for example, each order links to a specific customer. This way, you can quickly see all orders from one customer, or find which items sell the most."

## Reading Practice
Read and understand:

> "A relational database management system (RDBMS) allows you to store data in multiple related tables. The relationships between tables are defined by keys — special columns that link one table to another. Popular RDBMS systems include PostgreSQL, MySQL, SQLite, and Microsoft SQL Server."
