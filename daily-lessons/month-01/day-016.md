# Day 16: INSERT Multiple Rows at Once

## Topic
INSERT multiple rows in a single statement: batch insertion for efficiency and cleaner code.

## English Explanation
Instead of writing separate INSERT statements for each row, SQL allows you to **insert multiple rows in one statement**. This is faster, cleaner, and more efficient.

**Single-statement multi-row syntax:**
```
INSERT INTO table_name (col1, col2, col3)
VALUES
    (val1a, val2a, val3a),
    (val1b, val2b, val3b),
    (val1c, val2c, val3c);
```

**Why use multi-row INSERT?**
- Faster execution (one command instead of many)
- Fewer round-trips to the database
- Easier to read for bulk data entry
- Common in data migration and seeding scripts

**Sequential inserts** (one at a time) are still valid but slower:
```
INSERT INTO table_name (col1) VALUES (val1);
INSERT INTO table_name (col1) VALUES (val2);
INSERT INTO table_name (col1) VALUES (val3);
```

## SQL Example
```sql
-- Create employees table
CREATE TABLE employees (
    emp_id INTEGER PRIMARY KEY AUTOINCREMENT,
    first_name TEXT NOT NULL,
    last_name TEXT NOT NULL,
    department TEXT NOT NULL,
    salary REAL NOT NULL CHECK(salary > 0),
    hire_date TEXT NOT NULL DEFAULT (date('now'))
);


-- Method 1: Multi-row INSERT (RECOMMENDED for bulk data)
INSERT INTO employees (first_name, last_name, department, salary)
VALUES
    ('Sokha', 'Chem', 'Engineering', 1200.00),
    ('Dara', 'Phan', 'Marketing', 950.00),
    ('Bopha', 'Kim', 'Engineering', 1350.00),
    ('Vanna', 'Sok', 'Sales', 1000.00),
    ('Channa', 'Ly', 'HR', 1100.00);

-- Method 2: Sequential inserts (one at a time - slower)
INSERT INTO employees (first_name, last_name, department, salary)
VALUES ('Maly', 'Heng', 'Marketing', 900.00);

INSERT INTO employees (first_name, last_name, department, salary)
VALUES ('Pich', 'Oun', 'Sales', 1050.00);

-- Verify all records
SELECT * FROM employees;
```

## Technical Vocabulary

| Term | Meaning | Khmer |
|------|---------|-------|
| Multi-row INSERT | Inserting many rows in one statement | បញ្ចូលច្រើនជួរដេកក្នុងមួយពាក្យបញ្ជា |

| Bulk operation | Processing many items at once | ប្រតិបត្តិការទិន្នន័យច្រើនក្នុងពេលតែមួយ |

## Speaking Practice
"When I need to add many records to a database, I use multi-row INSERT instead of writing separate statements for each row. This is much faster because the database only processes one command instead of many. In real projects, we often use multi-row INSERT for data seeding — filling tables with initial data when setting up a new system."

## Reading Practice
