# Day 25: WHERE with IN and BETWEEN

## Topic
Using IN for matching lists of values and BETWEEN for ranges.

## English Explanation
IN lets you check if a value matches any item in a list — it replaces multiple OR conditions. BETWEEN lets you filter a range of values (inclusive of both ends). Both make your SQL cleaner and easier to read compared to writing many OR or AND conditions.

## SQL Example
```sql
-- IN: match any value in a list
SELECT * FROM products
WHERE category IN ('Electronics', 'Books', 'Toys');

-- Same as writing multiple OR:
-- WHERE category = 'Electronics' OR category = 'Books' OR category = 'Toys'

-- NOT IN: exclude values in a list
SELECT * FROM products
WHERE category NOT IN ('Archived', 'Deleted');

-- BETWEEN: inclusive range
SELECT * FROM products
WHERE price BETWEEN 10 AND 50;

-- Same as:
-- WHERE price >= 10 AND price <= 50

-- BETWEEN with dates
SELECT * FROM orders
WHERE order_date BETWEEN '2024-01-01' AND '2024-03-31';

-- Combining IN and BETWEEN
SELECT * FROM products
WHERE category IN ('Electronics', 'Books')
AND price BETWEEN 20 AND 100;
```

## Speaking Practice
Explain to a client:
"Instead of writing many OR conditions, we use IN to check a list of values at once. For number or date ranges, we use BETWEEN. For example, to find all orders from Q1 2024, we write BETWEEN January 1 and March 31. Both make our queries shorter and clearer."

## Reading Practice
Read and simplify this query:
```sql
-- Before (verbose):
SELECT * FROM employees
WHERE department = 'Sales' OR department = 'Marketing'
OR department = 'Support';
-- After (clean):
SELECT * FROM employees
WHERE department IN ('Sales', 'Marketing', 'Support');
```
