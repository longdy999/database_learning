# Day 8: Data Types - TEXT, INTEGER, REAL, BOOLEAN

## Topic
Understanding what types of data each column can hold.

## English Explanation
Every column in a table must have a **data type** — this tells the database what kind of information to expect.

**Main data types in SQLite:**
- **INTEGER** — Whole numbers (1, 42, -7, 1000)
- **REAL** — Decimal numbers (3.14, 99.99, -0.5)
- **TEXT** — Any text/string ("hello", "Sokha", "2023-01-15")
- **BLOB** — Binary data (images, files — rarely used directly)
- **NULL** — No value / empty

**Why data types matter:**
- They prevent wrong data (you can't put "hello" in an INTEGER column)
- They save storage space
- They allow proper sorting and calculations

## SQL Example
```sql
CREATE TABLE products (
    id INTEGER PRIMARY KEY,     -- whole number, unique
    name TEXT NOT NULL,          -- text, cannot be empty
    price REAL NOT NULL,         -- decimal number
    quantity INTEGER DEFAULT 0,  -- whole number, default is 0
    description TEXT,            -- text, can be empty (NULL)
    is_active INTEGER DEFAULT 1  -- 1=true, 0=false (SQLite boolean)
);

-- SQLite does not have a true BOOLEAN type
-- We use INTEGER: 1 = true, 0 = false

INSERT INTO products VALUES (1, 'Laptop', 899.99, 10, 'Gaming laptop', 1);
INSERT INTO products VALUES (2, 'Mouse', 29.50, 50, NULL, 1);
INSERT INTO products VALUES (3, 'Old Keyboard', 15.00, 0, 'Discontinued', 0);
```
