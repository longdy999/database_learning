# Day 13: Practice Day - Create 3 Real-World Tables (Library System)

## Topic
Practice Day: Design and create 3 related tables for a library management system (books, members, borrowings).

## English Explanation
Today is a **practice day**. No new concepts. You apply everything from Days 1-12 to build a real system. A library needs three tables:
- **books** - catalog of all books with ISBN, title, author, year
- **members** - registered library members with contact info
- **borrowings** - records of who borrowed which book and when

This shows how real databases use multiple tables together.

## SQL Example
```sql
-- Table 1: Books
CREATE TABLE books (
    book_id INTEGER PRIMARY KEY AUTOINCREMENT,
    isbn TEXT UNIQUE NOT NULL,
    title TEXT NOT NULL,
    author TEXT NOT NULL,
    category TEXT NOT NULL DEFAULT 'General',
    published_year INTEGER CHECK(published_year > 1800),
    copies_available INTEGER NOT NULL DEFAULT 1 CHECK(copies_available >= 0),
    is_active INTEGER NOT NULL DEFAULT 1
);

-- Table 2: Members
CREATE TABLE members (
    member_id INTEGER PRIMARY KEY AUTOINCREMENT,
    full_name TEXT NOT NULL,
    email TEXT UNIQUE NOT NULL,
    phone TEXT UNIQUE,
    membership_date TEXT NOT NULL DEFAULT (date('now')),
    membership_type TEXT NOT NULL DEFAULT 'standard'
        CHECK(membership_type IN ('standard', 'premium', 'student')),
    is_active INTEGER NOT NULL DEFAULT 1
);

-- Table 3: Borrowings
CREATE TABLE borrowings (
    borrowing_id INTEGER PRIMARY KEY AUTOINCREMENT,
    book_id INTEGER NOT NULL,
    member_id INTEGER NOT NULL,
    borrow_date TEXT NOT NULL DEFAULT (date('now')),
    due_date TEXT NOT NULL,
    return_date TEXT,
    status TEXT NOT NULL DEFAULT 'borrowed'
        CHECK(status IN ('borrowed', 'returned', 'overdue'))
);
```

## Reading Practice
Review all 3 CREATE TABLE statements. For each table ask:

2. Which columns are NOT NULL?
3. Which columns have DEFAULT values?
4. Which columns have CHECK constraints?
5. How would these tables connect to each other in a real system?

Read the INSERT statements and verify: does each one respect all constraints defined in the CREATE TABLE?
