# Day 6: Your First SQL Command - SELECT

## Topic
Learning SELECT — the most important and most used SQL command.

## English Explanation
**SELECT** is the command you use to READ data from a database. You will use SELECT more than any other SQL command in your career. It asks the database: "Show me this data."

Basic patterns:
- `SELECT * FROM table_name;` — Get ALL columns
- `SELECT column1, column2 FROM table_name;` — Get specific columns
- `SELECT * FROM table_name WHERE condition;` — Get filtered rows

The asterisk `*` means "all columns." In professional work, you should name specific columns instead of using `*`.

## SQL Example
```sql
-- First, let's create a table with sample data:
CREATE TABLE employees (
    id INTEGER PRIMARY KEY,
    name TEXT NOT NULL,
    department TEXT,
    salary REAL,
    hire_date TEXT
);

INSERT INTO employees VALUES (1, 'Sokha', 'Engineering', 1200, '2023-01-15');
INSERT INTO employees VALUES (2, 'Dara', 'Marketing', 950, '2023-03-20');
INSERT INTO employees VALUES (3, 'Chantha', 'Engineering', 1100, '2022-11-01');
INSERT INTO employees VALUES (4, 'Vanna', 'Sales', 1000, '2023-06-10');
INSERT INTO employees VALUES (5, 'Bopha', 'Marketing', 980, '2023-02-28');

-- SELECT all columns, all rows:
SELECT * FROM employees;

-- SELECT specific columns only:
SELECT name, department FROM employees;

-- SELECT with alias (rename column in output):
SELECT name, salary AS monthly_pay FROM employees;
```


## Hands-on Practice (10-15 minutes)
1. Open SQLite: `sqlite3 practice.db`
2. Create the employees table above (copy and run it)
3. Run these queries and observe the output:
   - `SELECT * FROM employees;`
   - `SELECT name FROM employees;`
   - `SELECT name, salary FROM employees;`
   - `SELECT department, name FROM employees;`
4. Try using an alias: `SELECT name AS employee_name FROM employees;`
5. Count rows: `SELECT COUNT(*) FROM employees;`

## Speaking Practice
**Say this out loud:**

"The SELECT statement is how we retrieve data from the database. If you want to see all customer records, I would write SELECT star FROM customers. If you only need names and emails, I would write SELECT name, email FROM customers. This gives us exactly the data we need without loading everything."

## Reading Practice
> "The SELECT statement is the foundation of SQL querying. In production systems, avoid using SELECT * because it retrieves all columns, which can be slow on large tables. Instead, specify only the columns you actually need."
