# Day 17: SELECT * - Reading All Data from a Table (Review and Deeper)

## Topic
SELECT * explained in depth: understanding result sets, row ordering, performance implications, and when to use (or avoid) SELECT *.

## English Explanation
**SELECT *** retrieves every column from a table. Today we go deeper:

1. **Basic syntax:** `SELECT * FROM table_name;`
2. **Result set:** The data returned is called a "result set" — a temporary table of rows and columns
3. **Row order:** Without ORDER BY, the database returns rows in **no guaranteed order**
4. **Performance:** SELECT * reads ALL columns — in large tables with many columns, this can be slow
5. **When to use SELECT *:**
   - Quick exploration or debugging
   - Small tables during development
   - When you truly need all columns
6. **When to AVOID SELECT *:**
   - Production code (always name specific columns)
   - Large tables with many columns
   - When you only need 2-3 columns out of 20+

**Professional habit:** Use SELECT * for exploration, but SELECT specific columns for real applications.

## SQL Example
```sql
-- Setup: Create and populate a sample table
CREATE TABLE students (
    student_id INTEGER PRIMARY KEY AUTOINCREMENT,
    first_name TEXT NOT NULL,
    last_name TEXT NOT NULL,
    grade_level INTEGER NOT NULL CHECK(grade_level BETWEEN 1 AND 12),
    gpa REAL DEFAULT 0.0,
    enrolled_date TEXT NOT NULL DEFAULT (date('now'))
);


INSERT INTO students (first_name, last_name, grade_level, gpa)
VALUES
    ('Sokha', 'Chem', 10, 3.5),
    ('Dara', 'Phan', 11, 3.8),
    ('Bopha', 'Kim', 9, 3.2),
    ('Vanna', 'Sok', 12, 3.9),
    ('Channa', 'Ly', 10, 2.8);

-- SELECT * returns ALL columns and ALL rows
SELECT * FROM students;

-- Result set has 5 rows and 6 columns (including auto-generated id and date)

-- The order of rows is NOT guaranteed without ORDER BY
-- These two queries may return rows in different order:
SELECT * FROM students;
SELECT * FROM students;

-- Using SELECT * on multiple tables for exploration
CREATE TABLE courses (
    course_id INTEGER PRIMARY KEY AUTOINCREMENT,
    course_name TEXT NOT NULL,
    teacher TEXT NOT NULL,
    credits INTEGER NOT NULL DEFAULT 3
);

INSERT INTO courses (course_name, teacher, credits)
VALUES
    ('Database Fundamentals', 'Mr. Chen', 3),
    ('Web Development', 'Ms. Sokha', 4),
    ('Data Structures', 'Mr. Rith', 3);

-- Explore each table
SELECT * FROM students;
SELECT * FROM courses;
```

## Hands-on Practice (10-15 minutes)
```sql
-- Create 3 different tables and query them all with SELECT *
CREATE TABLE products (
    id INTEGER PRIMARY KEY AUTOINCREMENT,

    order_id INTEGER PRIMARY KEY AUTOINCREMENT,
```

## Speaking Practice
"SELECT * is useful for quickly exploring a table to see what data is inside. However, in production applications, I always select specific columns instead of using the asterisk. This is because SELECT * can be slow on large tables and it makes the code less clear — other developers cannot immediately see which columns the query actually needs."
