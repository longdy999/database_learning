# Day 9: Primary Keys

## Topic
Primary Keys — what they are and why every table needs one.

## English Explanation
A **primary key** is a column (or set of columns) that uniquely identifies each row in a table. No two rows can have the same primary key value, and it can never be NULL. Think of it like a national ID number — every person has a unique one.

**Rules of primary keys:**
- Must be unique for every row
- Cannot be NULL (empty)
- Each table should have exactly one primary key
- Often an auto-incrementing integer

## SQL Example
```sql
-- Table with INTEGER PRIMARY KEY and AUTOINCREMENT
CREATE TABLE customers (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    email TEXT UNIQUE NOT NULL,
    phone TEXT
);

INSERT INTO customers (name, email, phone)
VALUES ('Sokha', 'sokha@email.com', '012345678');

INSERT INTO customers (name, email, phone)
VALUES ('Dara', 'dara@email.com', '098765432');

-- id is assigned automatically: 1, 2, 3...
SELECT * FROM customers;
```


## Hands-on Practice (10-15 minutes)
1. Create a `books` table with `id INTEGER PRIMARY KEY AUTOINCREMENT`, `title TEXT NOT NULL`, `author TEXT`, `year INTEGER`:
```sql
CREATE TABLE books (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT NOT NULL,
    author TEXT,
    year INTEGER
);
INSERT INTO books (title, author, year) VALUES ('SQL Basics', 'John', 2020);
INSERT INTO books (title, author, year) VALUES ('Data Design', 'Maria', 2022);
```
2. Create an `orders` table with a proper primary key:
```sql
CREATE TABLE orders (
    order_id INTEGER PRIMARY KEY AUTOINCREMENT,
    customer_name TEXT NOT NULL,
    product TEXT NOT NULL,
    amount REAL NOT NULL
);
INSERT INTO orders (customer_name, product, amount) VALUES ('Sokha', 'Laptop', 899.99);
```

## Technical Vocabulary

| Term | Meaning | Khmer |
|------|---------|-------|
| Primary Key | Column that uniquely identifies each row | សោចម្បង |
| Unique | No duplicate values allowed | តែមួយគត់ |
| AUTOINCREMENT | Automatically assigns next number | បង្កើនដោយស្វ័យប្រវត្តិ |
| Identity | Auto-generated unique value | អត្តសញ្ញាណ |
| Constraint | Rule enforced on a column | កំហិត/ច្បាប់ |

## Speaking Practice
"Every table in our database has a primary key. This ensures that each record can be uniquely identified. For example, in our customers table, the id column is the primary key — it automatically increments and guarantees no duplicates."
