## SQL Example

```markdown
# Online Bookstore Database

## Project Overview
A relational database for managing an online bookstore including
customers, books, orders, and reviews.

## Database Schema
- **customers** - Store customer accounts and contact info
- **categories** - Book classification hierarchy
- **books** - Product catalog with pricing and inventory
- **orders** - Purchase transactions with status tracking
- **order_items** - Individual items within each order
- **reviews** - Customer ratings and feedback

## Design Decisions
1. Used SERIAL for auto-incrementing primary keys
2. Enforced referential integrity with foreign keys
3. Added CHECK constraints for business rules (positive prices, valid ratings)
4. Used ON DELETE CASCADE for order_items (items deleted with orders)
5. Created composite UNIQUE on reviews (one review per customer per book)

## Setup Instructions
1. Install PostgreSQL 14+
2. Create database: `createdb bookstore`
3. Run schema: `psql bookstore < schema.sql`
4. Load data: `psql bookstore < seed_data.sql`
5. Verify: `psql bookstore < verify.sql`

## Key Queries Included
| Query | Purpose |
|-------|---------|
| Top sellers | Books ranked by units sold |
| Revenue report | Monthly income breakdown |
| Customer value | Lifetime spending per customer |
| Stock alerts | Low inventory warnings |
| Category performance | Revenue by book category |
```

## Technical Vocabulary
- **README**: The first file people read in any project repository
- **ERD**: Entity Relationship Diagram showing table connections
- **DDL**: Data Definition Language (CREATE, ALTER, DROP)
- **DML**: Data Manipulation Language (INSERT, UPDATE, DELETE)
- **DQL**: Data Query Language (SELECT)
