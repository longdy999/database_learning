# Day 153: Exam Review — JOINs and Relationships

## 1. Topic
Review of INNER JOIN, LEFT JOIN, foreign keys, one-to-many (1:N), and many-to-many (M:N) relationships.

## 2. English Explanation
JOINs combine data from two or more tables based on a related column. INNER JOIN returns only matching rows from both tables. LEFT JOIN returns all rows from the left table and matching rows from the right (NULL if no match). Foreign keys create relationships between tables — they reference the primary key of another table. One-to-many (1:N) means one record in table A relates to many in table B (e.g., one customer has many orders). Many-to-many (M:N) requires a junction table (e.g., students and courses).

## 4. SQL Example

```sql
-- INNER JOIN: Get orders with customer names
SELECT c.first_name, c.last_name, o.order_date, o.total_amount
FROM customers c
INNER JOIN orders o ON c.customer_id = o.customer_id;

-- LEFT JOIN: All customers, even without orders
SELECT c.first_name, c.last_name, o.order_id
FROM customers c
LEFT JOIN orders o ON c.customer_id = o.customer_id;

-- Many-to-many with junction table
SELECT s.student_name, co.course_name
FROM students s
INNER JOIN student_courses sc ON s.student_id = sc.student_id
INNER JOIN courses co ON sc.course_id = co.course_id;

-- Foreign key definition
CREATE TABLE orders (
    order_id SERIAL PRIMARY KEY,
    customer_id INTEGER NOT NULL REFERENCES customers(customer_id),
    order_date DATE DEFAULT CURRENT_DATE,
    total_amount DECIMAL(10,2)
);
```

## 6. Technical Vocabulary

| Term | Meaning |
|------|---------|
| INNER JOIN | Returns only matching rows from both tables |
| LEFT JOIN | All rows from left table + matches from right |
| Foreign Key | Column referencing another table's primary key |
| One-to-Many (1:N) | One record relates to many in another table |
| Many-to-Many (M:N) | Records in both tables relate to many in the other |
| Junction Table | Bridge table connecting M:N relationships |
| ON clause | Specifies the join condition |
| Table Alias | Short name for a table (e.g., c for customers) |
| Referential Integrity | Ensuring foreign keys point to valid records |
| CASCADE | Automatically propagate changes to related rows |
