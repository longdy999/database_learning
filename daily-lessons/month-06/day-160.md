## Section 5: Hands-On Practice (30 minutes)
**Task 1:** Draw your ERD with at least 5 tables.
```sql
CREATE TABLE customers (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE,
    created_at TIMESTAMP DEFAULT NOW()
);
CREATE TABLE orders (
    id SERIAL PRIMARY KEY,
    customer_id INT REFERENCES customers(id),
    order_date TIMESTAMP DEFAULT NOW(),
    status VARCHAR(20) DEFAULT 'pending',
    total DECIMAL(10,2)
);
CREATE TABLE order_items (
    id SERIAL PRIMARY KEY,
    order_id INT REFERENCES orders(id) ON DELETE CASCADE,
    product_id INT NOT NULL,
    quantity INT DEFAULT 1,
    subtotal DECIMAL(10,2)
);
```
**Task 2:** Document all relationships (1:1, 1:M, M:M).
**Task 3:** Write 3 INSERT statements to test your schema.


## Section 6: Technical Vocabulary
| Term | Definition |
|------|-----------|
| ERD | Visual diagram of tables and relationships |
| Schema | Complete database structure |
| Cardinality | Relationship type (1:1, 1:M, M:M) |
| Junction table | Resolves many-to-many relationships |
| CASCADE | Auto-action on related rows |
| SERIAL | Auto-incrementing integer (PostgreSQL) |
| REFERENCES | Creates a foreign key constraint |

## Section 8: Reading Practice
Study crow's foot ERD notation: fork = many, line = one. Review your design — every table needs a clear PK and appropriate FKs.
