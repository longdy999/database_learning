## Section 4: SQL/Database Example
```sql
-- Check your schema quality:
-- 1. Tables without primary keys:
SELECT table_name FROM information_schema.tables
WHERE table_schema = 'public'
AND table_name NOT IN (
    SELECT table_name FROM information_schema.table_constraints
    WHERE constraint_type = 'PRIMARY KEY');

-- 2. Columns that should have NOT NULL:
SELECT table_name, column_name
FROM information_schema.columns
WHERE table_schema = 'public'
AND column_name IN ('email', 'name', 'created_at')
AND is_nullable = 'YES';
```

## Section 5: Hands-On Practice (10-15 minutes)
**Portfolio Project Feedback Checklist (Score each 1-5):**

### Schema Quality:
- [ ] All tables have clear, descriptive names (snake_case)
- [ ] Primary keys exist on every table
- [ ] Foreign keys properly defined with ON DELETE actions
- [ ] NOT NULL constraints on required fields
- [ ] Indexes on frequently queried columns

### SQL Quality:
- [ ] Queries are formatted and readable
- [ ] No SELECT * in production queries
- [ ] JOINs use explicit syntax (not implicit)

### Documentation Quality:
- [ ] README explains the project clearly
- [ ] Schema diagram (ERD) is included
- [ ] Setup instructions work from scratch
