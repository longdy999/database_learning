# Day 157: Exam Review - Indexes, Transactions, Performance

## Section 1: Topic
**EXPLAIN, ACID, Stored Procedures** — Review with practice problems.

## Section 2: English Explanation
Indexes create sorted lookups for fast retrieval. Transactions group operations to succeed or fail together (ACID). EXPLAIN shows the execution plan to find bottlenecks. Stored procedures encapsulate reusable server-side logic.

## Section 4: SQL Examples
```sql
-- Indexes
CREATE INDEX idx_emp_dept ON employees(department_id);
-- EXPLAIN
EXPLAIN ANALYZE SELECT name FROM employees WHERE salary > 50000;
-- Transaction
BEGIN;
UPDATE accounts SET balance = balance - 500 WHERE id = 1;
UPDATE accounts SET balance = balance + 500 WHERE id = 2;
COMMIT;
-- Stored procedure (PostgreSQL)
CREATE OR REPLACE FUNCTION get_dept_count(dept_id INT)
RETURNS INT AS $$
BEGIN
  RETURN (SELECT COUNT(*) FROM employees WHERE department_id = dept_id);
END;
$$ LANGUAGE plpgsql;
```

## Section 5: Practice Problems (30 minutes)
1. Create composite index on orders(customer_id, order_date)
2. Write a transaction transferring $200 between accounts
3. Use EXPLAIN ANALYZE — identify Seq Scan vs Index Scan
4. Write a function calculating monthly revenue
5. Identify which columns to index for a given slow query
6. Write a transaction with SAVEPOINT and partial ROLLBACK
7. Create a partial index (WHERE status = 'active')
8. Interpret EXPLAIN output: cost, rows, actual time
9. Write a procedure that raises an exception on invalid input
10. Compare query time before and after adding an index

**Key insight:** Always create indexes on columns in WHERE, JOIN, and ORDER BY clauses.


## Section 6: Technical Vocabulary
| Term | Definition |
|------|-----------|
| Index | Structure speeding up row lookups |
| B-tree | Default index type in most databases |
| EXPLAIN | Shows query execution plan |
| Seq Scan | Full table scan (slow on large tables) |
| Index Scan | Fast lookup using an index |
| ACID | Atomicity, Consistency, Isolation, Durability |
| Stored procedure | Reusable SQL logic on the server |
