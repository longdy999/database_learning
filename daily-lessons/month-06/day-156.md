# Day 156: Exam Review - Advanced SQL

## Section 1: Topic
**Subqueries, Views, and CTEs** — Review with 10 practice problems.

## Section 2: English Explanation
Subqueries are queries inside another query (in SELECT, FROM, or WHERE). Views are saved queries acting as virtual tables. CTEs use WITH clauses to name temporary result sets for clarity.

## Section 4: SQL Examples
```sql
-- Subquery in WHERE: employees above average salary
SELECT name, salary FROM employees
WHERE salary > (SELECT AVG(salary) FROM employees);

-- Correlated subquery: each customer's latest order
SELECT c.name,
  (SELECT MAX(order_date) FROM orders o WHERE o.customer_id = c.id) AS last_order
FROM customers c;

-- View: reusable virtual table
CREATE VIEW high_earners AS
SELECT name, salary, department_id FROM employees WHERE salary > 80000;

-- CTE: readable temporary result set
WITH dept_avg AS (
  SELECT department_id, AVG(salary) AS avg_sal
  FROM employees GROUP BY department_id
)
SELECT e.name, e.salary, da.avg_sal FROM employees e
JOIN dept_avg da ON e.department_id = da.department_id
WHERE e.salary > da.avg_sal;
```

## Section 5: 10 Practice Problems
1. Find products priced above average (subquery in WHERE)
2. Find each customer's most recent order (correlated subquery)
3. Create a view showing total sales per customer
4. Rank employees by salary per department (CTE + window)
5. Recursive CTE: show management hierarchy
6. Subquery in FROM: top department by headcount
7. EXISTS vs IN: find customers with orders
8. View with JOIN: employee + department name
9. CTE to calculate running totals
10. Nested subquery: products in most popular category


## Section 6: Technical Vocabulary
| Term | Definition |
|------|-----------|
| Subquery | Query nested inside another query |
| Correlated subquery | References outer query for each row |
| View | Stored SELECT acting as virtual table |
| CTE | Common Table Expression (WITH clause) |
| Recursive CTE | CTE referencing itself for hierarchies |
