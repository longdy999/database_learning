# Day 152: Exam Review — SQL SELECT and Filtering

## 1. Topic
Review of SELECT statements, WHERE clauses, ORDER BY, LIMIT, LIKE, IN, and BETWEEN operators.

## 2. English Explanation
The SELECT statement is the most-used SQL command. It retrieves data from tables. WHERE filters rows based on conditions. ORDER BY sorts results (ASC or DESC). LIMIT restricts how many rows are returned. LIKE allows pattern matching with wildcards (% for any characters, _ for one character). IN checks if a value matches any in a list. BETWEEN checks if a value falls within a range. Mastering these lets you answer any data question.

## 4. SQL Example

```sql
-- Basic SELECT
SELECT first_name, last_name, email FROM customers;

-- WHERE with comparison
SELECT * FROM products WHERE price > 50.00;

-- ORDER BY
SELECT * FROM products ORDER BY price DESC;

-- LIMIT
SELECT * FROM customers ORDER BY created_at DESC LIMIT 10;

-- LIKE pattern matching
SELECT * FROM customers WHERE email LIKE '%@gmail.com';
SELECT * FROM products WHERE name LIKE 'Smart%';

-- IN operator
SELECT * FROM products WHERE category IN ('Electronics', 'Books', 'Clothing');

-- BETWEEN
SELECT * FROM orders WHERE order_date BETWEEN '2024-01-01' AND '2024-06-30';

-- Combined filters
SELECT first_name, last_name, email
FROM customers
WHERE created_at BETWEEN '2024-01-01' AND '2024-12-31'
  AND email LIKE '%@gmail.com'
ORDER BY last_name ASC
LIMIT 20;
```

## 6. Technical Vocabulary

| Term | Meaning |
|------|---------|
| SELECT | Retrieve data from a table |
| WHERE | Filter rows based on conditions |
| ORDER BY | Sort results ascending or descending |
| LIMIT | Restrict number of returned rows |
| LIKE | Pattern matching with wildcards |
| IN | Match against a list of values |
| BETWEEN | Match within a range of values |
| ASC/DESC | Ascending/Descending sort order |
| Wildcard (%) | Matches any sequence of characters |
| IS NULL | Check for empty/missing values |
