# Day 164: Portfolio Project - Views and Indexes

## Topic
Add views for reports and indexes for performance optimization.

## English Explanation
Views simplify complex queries into reusable named objects. Indexes speed up data retrieval on frequently queried columns. Together they demonstrate that you think beyond basic queries — you consider maintainability and performance, which separates junior from professional developers.

## SQL Example

```sql
-- View 1: Customer Order Summary
CREATE VIEW customer_order_summary AS
SELECT c.customer_id, c.first_name, c.last_name, c.email,
       COUNT(o.order_id) AS total_orders,
       COALESCE(SUM(o.total_amount), 0) AS total_spent,
       MAX(o.order_date) AS last_order_date
FROM customers c
LEFT JOIN orders o ON c.customer_id = o.customer_id
    AND o.status != 'cancelled'
GROUP BY c.customer_id, c.first_name, c.last_name, c.email;

-- View 2: Book Sales Performance
CREATE VIEW book_sales_report AS
SELECT b.book_id, b.title, b.author, cat.category_name,
       b.price, b.stock_quantity,
       COALESCE(SUM(oi.quantity), 0) AS units_sold,
       COALESCE(SUM(oi.quantity * oi.unit_price), 0) AS revenue,
       ROUND(COALESCE(AVG(r.rating), 0), 2) AS avg_rating
FROM books b
LEFT JOIN categories cat ON b.category_id = cat.category_id
LEFT JOIN order_items oi ON b.book_id = oi.book_id
LEFT JOIN reviews r ON b.book_id = r.book_id
GROUP BY b.book_id, b.title, b.author, cat.category_name,
         b.price, b.stock_quantity;

-- View 3: Monthly Revenue Dashboard
CREATE VIEW monthly_revenue AS
SELECT DATE_TRUNC('month', order_date) AS month,
       COUNT(*) AS orders_count,
       SUM(total_amount) AS total_revenue,
       ROUND(AVG(total_amount), 2) AS avg_order_value
FROM orders
WHERE status NOT IN ('cancelled')
GROUP BY DATE_TRUNC('month', order_date);
```


```sql
-- Index 1: Speed up customer email lookups
CREATE INDEX idx_customers_email ON customers(email);

-- Index 2: Speed up order filtering by status
CREATE INDEX idx_orders_status ON orders(status);

-- Index 3: Speed up order date range queries
CREATE INDEX idx_orders_date ON orders(order_date);

-- Index 4: Speed up book searches by category
CREATE INDEX idx_books_category ON books(category_id);

-- Index 5: Speed up order items lookup by order
CREATE INDEX idx_order_items_order ON order_items(order_id);

-- Index 6: Composite index for reviews by book
CREATE INDEX idx_reviews_book_rating ON reviews(book_id, rating);

-- Verify views work
SELECT * FROM customer_order_summary ORDER BY total_spent DESC;
SELECT * FROM book_sales_report ORDER BY revenue DESC;
SELECT * FROM monthly_revenue ORDER BY month;

-- Check index usage with EXPLAIN
EXPLAIN ANALYZE SELECT * FROM orders WHERE status = 'delivered';
EXPLAIN ANALYZE SELECT * FROM books WHERE category_id = 3;
```

## Hands-on Practice

## Technical Vocabulary
- **View**: A saved SELECT query accessible like a virtual table
- **Index**: A data structure that speeds up row retrieval on specific columns
- **Composite index**: An index on multiple columns for multi-condition queries
- **COALESCE**: Returns the first non-NULL value from a list of arguments
- **EXPLAIN ANALYZE**: Shows actual query execution plan with timing
