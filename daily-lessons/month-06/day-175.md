## Section 4: Complete Topic List with Key Points

### Area 1: Database Fundamentals (Month 1)
| Topic | Key Point | Common Mistake |
|-------|-----------|----------------|
| Tables | Store data in rows and columns | Storing unrelated data in one table |
| Primary Key | Unique identifier for each row | Using business data as PK |
| Foreign Key | Links tables via references | Not enforcing referential integrity |
| Data Types | INT, VARCHAR, DATE, BOOLEAN | Using wrong type (VARCHAR for numbers) |
| NULL | Represents unknown/missing | Comparing with = instead of IS NULL |
| Constraints | Rules enforcing data integrity | Forgetting NOT NULL on required fields |

### Area 2: SQL Querying (Month 2)
| Topic | Key Point | Common Mistake |
|-------|-----------|----------------|
| SELECT | Retrieves data from tables | Using SELECT * in production |
| WHERE | Filters rows before grouping | Confusing with HAVING |
| JOIN | Combines rows from multiple tables | Forgetting join condition (cartesian) |
| GROUP BY | Groups rows for aggregation | Selecting non-aggregated columns |
| ORDER BY | Sorts result set | Sorting large datasets without LIMIT |
| Subquery | Query nested inside another query | Using correlated subquery when JOIN works |

### Area 3: Database Design (Month 3)
| Topic | Key Point | Common Mistake |
|-------|-----------|----------------|
| Normalization | Reduces data redundancy | Over-normalizing simple data |
| 1NF/2NF/3NF | Atomic values, no partial/transitive deps | Storing comma-separated values |
| ERD | Visual diagram of database structure | Missing cardinality notation |
| Relationships | 1:1, 1:Many, Many:Many | Not using junction tables for M:M |


### Area 4: Advanced SQL & Performance (Month 4)
| Topic | Key Point | Common Mistake |
|-------|-----------|----------------|
| Index | B-tree structure for fast lookups | Indexing every column (hurts writes) |
| View | Virtual table from stored query | Assuming views improve performance |
| Transaction | ACID-compliant unit of work | Not handling rollback scenarios |
| Window Function | Calculation across row sets | Confusing with GROUP BY |
| EXPLAIN | Shows query execution plan | Ignoring sequential scans on large tables |
| CTE | Named temporary result set (WITH) | Overusing when simple JOIN works |

### Area 5: Real-World Database Work (Month 5)
| Topic | Key Point | Common Mistake |
|-------|-----------|----------------|
| Backup | Regular copies for disaster recovery | Not testing restore procedures |
| Security | Least privilege access control | Using root/admin for applications |
| PostgreSQL | Powerful open-source RDBMS | Not using EXPLAIN ANALYZE |
| Data Cleaning | Ensuring data quality | Cleaning without backup first |
| Reporting | Aggregated queries for insights | Not optimizing report queries |
