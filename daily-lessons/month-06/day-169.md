# Day 169: Technical Writing - Database Documentation

## Section 1: Main Topic
**How to write schema docs, API docs, and runbooks. Practice writing a 1-page
technical summary.** Good documentation separates professionals from hobbyists.

## Section 2: Simple English Explanation
Documentation helps others (or future you) understand your work. Schema docs explain
tables. API docs explain how to interact with data. Runbooks explain what to do
when something breaks. Write as if someone must maintain your system tomorrow.

## Section 4: SQL/Database Example
```sql
-- Documenting tables with comments (PostgreSQL):
COMMENT ON TABLE customers IS 'Stores registered customer accounts.';
COMMENT ON COLUMN customers.id IS 'Auto-incrementing PK.';
COMMENT ON COLUMN customers.email IS 'Unique email. NOT NULL.';
COMMENT ON COLUMN customers.status IS 'active, suspended, or deleted.';
```

## Section 5: Hands-On Practice (10-15 minutes)
**Practice writing 3 types of documentation:**

### Type 1: Schema Documentation
```markdown
## Table: orders
**Purpose:** Stores customer purchase orders.
| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| id | SERIAL | PK | Unique order identifier |
| customer_id | INT | FK, NOT NULL | Who placed the order |
| total_amount | DECIMAL(10,2) | NOT NULL | Order total in USD |
| status | VARCHAR(20) | DEFAULT 'pending' | pending/paid/shipped |
| created_at | TIMESTAMP | DEFAULT NOW() | When order was placed |
```

### Type 2: API Documentation
```markdown
## GET /api/customers/:id/orders
**Description:** Retrieve all orders for a customer.
**Auth:** Required (Bearer token)
**Response:** JSON array of order objects
**Errors:** 404 if not found, 401 if unauthorized
```


### Type 3: Runbook
```markdown
## Runbook: Database Backup Failure
**Severity:** High | **Response time:** 15 minutes
1. Check backup service: `systemctl status pg_backup`
2. Check disk space: `df -h /backup`
3. If disk full: remove backups older than 30 days
4. Re-run manually: `pg_dump dbname > backup.sql`
5. Verify backup file size > 0
6. Notify team in #database-alerts
```

## Section 6: Technical Vocabulary
| Term | Meaning |
|------|---------|
| Schema documentation | Written description of database structure |
| API documentation | Guide for how to use data endpoints |
| Runbook | Step-by-step guide for operational tasks |
| COMMENT ON | PostgreSQL command to add metadata to objects |
| Data dictionary | Complete reference of all tables and columns |

## Section 7: Speaking Practice
