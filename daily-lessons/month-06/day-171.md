## Section 4: Top 20 Interview Questions with Model Answers

| # | Question | Model Answer |
|---|----------|-------------|
| 1 | What is a relational database? | Stores data in tables with rows/columns, connected by keys |
| 2 | WHERE vs HAVING? | WHERE filters rows before grouping; HAVING filters after aggregation |
| 3 | Explain normalization | Reduces redundancy: 1NF (atomic), 2NF (no partial dep), 3NF (no transitive dep) |
| 4 | What is an index? | Data structure speeding up retrieval, like a book index |
| 5 | ACID properties? | Atomicity, Consistency, Isolation, Durability |
| 6 | INNER JOIN vs LEFT JOIN? | INNER: only matches. LEFT: all left rows + right matches |
| 7 | What is a primary key? | Unique non-null identifier for each row |
| 8 | What is a foreign key? | Column referencing another table's primary key |
| 9 | What is a stored procedure? | Precompiled SQL statements stored in the database |
| 10 | DELETE vs TRUNCATE vs DROP? | DELETE: rows (logged). TRUNCATE: all rows (fast). DROP: table gone |
| 11 | What is a view? | Virtual table from a stored SELECT query |
| 12 | What is a transaction? | Unit of work: all succeed (commit) or all fail (rollback) |
| 13 | What is denormalization? | Adding redundancy intentionally to improve read performance |
| 14 | Clustered vs non-clustered index? | Clustered: physical order (1/table). Non-clustered: separate structure (many) |
| 15 | What is a deadlock? | Circular lock dependency between two transactions |
| 16 | What is referential integrity? | FK values must reference existing PK values |
| 17 | Aggregate functions? | COUNT, SUM, AVG, MIN, MAX - operate on row groups |
| 18 | Query optimization? | Improve speed via indexes, rewriting queries, execution plans |
| 19 | UNION vs UNION ALL? | UNION removes duplicates; UNION ALL keeps all (faster) |
| 20 | What is sharding? | Splitting database across multiple servers by shard key |

## Section 5: Practice - Answer 5 Questions Out Loud

## Section 6: Technical Vocabulary
| Term | Definition |
|------|-----------|
| Referential integrity | Consistency between related tables |
| Deadlock | Circular lock dependency between transactions |
| Denormalization | Adding redundancy for performance |
| Sharding | Distributing data across multiple servers |
| Execution plan | Database engine's strategy for running a query |
