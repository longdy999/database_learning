# Day 155: Exam Review — Database Design and Normalization

## 1. Topic
Review of 1NF, 2NF, 3NF, ERD diagrams, constraints, and naming conventions.

## 2. English Explanation
Database design is about organizing data efficiently. Normalization removes redundancy and prevents update anomalies. **1NF**: Each cell has one value (atomic), each row is unique. **2NF**: 1NF + every non-key column depends on the entire primary key (no partial dependencies). **3NF**: 2NF + no non-key column depends on another non-key column (no transitive dependencies). ERD (Entity-Relationship Diagram) visually shows tables, columns, and relationships. Constraints enforce data rules: NOT NULL, UNIQUE, CHECK, DEFAULT, FOREIGN KEY. Good naming uses snake_case, plural table names, and descriptive column names.

## 4. SQL Example

```sql
-- BAD: Unnormalized table (violates 1NF)
-- customer_name | phone_numbers       | orders
-- 'Sokha'       | '012345, 067890'    | 'Laptop, Mouse'

-- GOOD: Normalized to 3NF
CREATE TABLE customers (
    customer_id SERIAL PRIMARY KEY,
    first_name VARCHAR(50) NOT NULL,
    last_name VARCHAR(50) NOT NULL
);

CREATE TABLE phone_numbers (
    phone_id SERIAL PRIMARY KEY,
    customer_id INTEGER REFERENCES customers(customer_id),
    phone_number VARCHAR(20) NOT NULL
);

CREATE TABLE orders (
    order_id SERIAL PRIMARY KEY,
    customer_id INTEGER REFERENCES customers(customer_id),
    order_date DATE DEFAULT CURRENT_DATE
);

CREATE TABLE order_items (
    item_id SERIAL PRIMARY KEY,
    order_id INTEGER REFERENCES orders(order_id),
    product_id INTEGER REFERENCES products(product_id),
    quantity INTEGER NOT NULL CHECK (quantity > 0),
    unit_price DECIMAL(10,2) NOT NULL
);
```


## 5. Hands-on Practice (10–15 minutes)
Normalize this table step by step:

**Unnormalized Table: student_records**
| student_id | name | courses | teachers | grades |

| 1 | Sokha | Math, Science | Mr. Lee, Ms. Kim | A, B |
| 2 | Dara | Math, English | Mr. Lee, Ms. Chan | B, A |

Steps:
1. Identify 1NF violations and fix them.
2. Identify the primary key and partial dependencies → create 2NF.
3. Identify transitive dependencies → create 3NF.
4. Draw the final ERD showing all tables and relationships.
5. Write CREATE TABLE statements for all final tables.

**Expected final tables:** students, courses, teachers, enrollments (junction), course_teachers

## 6. Technical Vocabulary

| Term | Meaning |
|------|---------|
| Normalization | Process of organizing data to reduce redundancy |
| 1NF | Atomic values, unique rows |
| 2NF | No partial dependencies on composite keys |
| 3NF | No transitive dependencies |
| ERD | Entity-Relationship Diagram |
| Constraint | Rule enforced on data (NOT NULL, UNIQUE, CHECK) |
| Atomic | Single indivisible value in a cell |
| Partial Dependency | Non-key column depends on part of composite key |
| Transitive Dependency | Non-key column depends on another non-key column |
| snake_case | Naming style using underscores (e.g., first_name) |
