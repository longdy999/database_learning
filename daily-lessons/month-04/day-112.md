# Day 112: Introduction to Transactions - BEGIN, COMMIT, ROLLBACK

## Topic
Understanding database transactions: grouping multiple SQL statements so they all succeed together or all fail together.

## English Explanation
A transaction is a group of SQL statements that must all complete successfully, or none of them take effect. Think of a bank transfer: you must debit one account AND credit another. If the debit succeeds but the credit fails, you lose money. Transactions prevent this. BEGIN starts a transaction, COMMIT saves all changes permanently, and ROLLBACK undoes everything back to the beginning.

## SQL Example
```sql
-- Basic transaction structure
BEGIN;
  UPDATE accounts SET balance = balance - 500 WHERE account_id = 1;
  UPDATE accounts SET balance = balance + 500 WHERE account_id = 2;
COMMIT;

-- If something goes wrong, rollback
BEGIN;
  UPDATE accounts SET balance = balance - 500 WHERE account_id = 1;
  -- Oops, account 2 does not exist! Undo everything:
ROLLBACK;

-- Setup table for practice
CREATE TABLE accounts (
    account_id SERIAL PRIMARY KEY,
    owner_name VARCHAR(100) NOT NULL,
    balance DECIMAL(10,2) NOT NULL DEFAULT 0,
    CHECK (balance >= 0)
);

INSERT INTO accounts (owner_name, balance) VALUES
('Sokha', 10000.00),
('Dara', 5000.00),
('Vanna', 8000.00);
```

## Technical Vocabulary

| Term | Definition |
|------|-----------|
| Transaction | Group of statements that succeed or fail as one unit |
| BEGIN | Starts a new transaction block |
| COMMIT | Saves all changes in the transaction permanently |
| ROLLBACK | Undoes all changes in the current transaction |
| Atomic | All-or-nothing execution of grouped statements |
| Implicit Transaction | Single statements auto-committed by database |
| Explicit Transaction | Manually started with BEGIN statement |
| Constraint Violation | Error when data breaks a rule (triggers rollback) |

## Speaking Practice
Explain to a client: "A transaction ensures that related database changes happen together safely. For example, in a payment system, we debit your account and credit the merchant as one atomic operation. If anything fails mid-way, everything is rolled back and no money is lost."
