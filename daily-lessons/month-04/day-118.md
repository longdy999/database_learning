# Day 118: Week 16 Review — Transactions and Stored Procedures

## Topic
Comprehensive review of transactions (BEGIN/COMMIT/ROLLBACK, ACID, isolation levels, SAVEPOINT) and stored procedures (syntax, parameters, control flow).

## English Explanation
This review combines two critical database topics. Transactions ensure data integrity by grouping operations into atomic units — either all succeed or all fail. Stored procedures encapsulate reusable logic in the database. Together, they form the foundation of reliable, maintainable database applications. A transaction-safe stored procedure guarantees that complex multi-step operations remain consistent.

## SQL Example

```sql
-- Transaction-safe stored procedure
CREATE OR REPLACE FUNCTION process_order(
    p_customer_id INT,
    p_product_id INT,
    p_quantity INT
)
RETURNS TEXT AS $$
DECLARE
    v_stock INT;
    v_price NUMERIC;
    v_order_id INT;
BEGIN
    -- Check available stock
    SELECT stock_quantity, unit_price INTO v_stock, v_price
    FROM products WHERE product_id = p_product_id;

    IF v_stock < p_quantity THEN
        RAISE EXCEPTION 'Insufficient stock: % available, % requested',
            v_stock, p_quantity;
    END IF;

    -- Create savepoint before critical operations
    -- Deduct inventory
    UPDATE products SET stock_quantity = stock_quantity - p_quantity
    WHERE product_id = p_product_id;


    -- Insert order record
    INSERT INTO orders (customer_id, product_id, quantity, total_price, order_date)
    VALUES (p_customer_id, p_product_id, p_quantity, v_price * p_quantity, CURRENT_DATE)
    RETURNING order_id INTO v_order_id;

    -- Insert payment record
    INSERT INTO payments (order_id, amount, status)
    VALUES (v_order_id, v_price * p_quantity, 'pending');

    RETURN 'Order ' || v_order_id || ' created successfully';

EXCEPTION
    WHEN OTHERS THEN
        RAISE NOTICE 'Error: % - %', SQLSTATE, SQLERRM;
        RETURN 'Order failed: ' || SQLERRM;
END;
$$ LANGUAGE plpgsql;

-- Using transactions with the procedure
BEGIN;
    SELECT process_order(1, 101, 5);
    -- If everything looks correct:
COMMIT;
-- If something went wrong:
-- ROLLBACK;
```

## Technical Vocabulary

| Term | Definition |
|------|-----------|
| ACID | Atomicity, Consistency, Isolation, Durability |
| BEGIN/COMMIT | Start and confirm a transaction |
| ROLLBACK | Cancel all changes in a transaction |
| SAVEPOINT | Bookmark inside a transaction for partial rollback |
| Isolation Level | Controls visibility between concurrent transactions |
| EXCEPTION | Error handling block in plpgsql procedures |
| SQLSTATE | Standard error code returned on failure |
| SQLERRM | Error message text returned on failure |
| RAISE EXCEPTION | Immediately stops procedure and triggers error |
| Atomic operation | All-or-nothing execution guarantee |
