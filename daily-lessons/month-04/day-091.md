# Day 91: Subqueries Introduction - A Query Inside a Query

## Topic
Understanding subqueries — how to nest one query inside another query to solve complex problems.

## English Explanation
A **subquery** (also called an inner query or nested query) is a SQL query placed inside another SQL query. The outer query uses the result of the inner query to complete its operation. Subqueries can return a single value (scalar subquery), a single column of values, or a full table. Today we focus on scalar subqueries in the WHERE clause — these return exactly one value that the outer query can compare against.

## SQL Example

```sql
-- Find employees earning above the average salary
-- The subquery calculates the average first
SELECT first_name, last_name, salary
FROM employees
WHERE salary > (SELECT AVG(salary) FROM employees);

-- Find products priced above average
SELECT product_name, price
FROM products
WHERE price > (SELECT AVG(price) FROM products);

-- Find the most recent order
SELECT order_id, customer_id, order_date
FROM orders
WHERE order_date = (SELECT MAX(order_date) FROM orders);
```

## Speaking Practice
"A subquery is a query inside another query. The inner query runs first and gives a result. The outer query then uses that result. For example, we can find employees who earn more than average by using a subquery to calculate the average salary first."

## Reading Practice
Read this query and explain what it does:
```sql
SELECT name, total_sales
FROM salespeople
WHERE total_sales > (SELECT AVG(total_sales) FROM salespeople);
```
Answer: It finds salespeople whose total sales exceed the average total sales of all salespeople.
