# Day 94: Correlated Subqueries

## Topic
Understanding correlated subqueries Рђћ subqueries that reference the outer query and execute once for each row of the outer query.

## English Explanation
A **correlated subquery** is different from a regular subquery because it references a column from the outer query. This means the subquery cannot run independently Рђћ it must execute once for every row the outer query processes. While this makes them powerful for row-by-row comparisons (like comparing each employee to their department average), they can be slower on large datasets because they run many times.

## SQL Example

```sql
-- Find employees earning more than their department average
SELECT e.first_name, e.last_name, e.salary, e.department_id
FROM employees e
WHERE e.salary > (
    SELECT AVG(e2.salary)
    FROM employees e2
    WHERE e2.department_id = e.department_id
);

-- Find products priced above their category average
SELECT p.product_name, p.price, p.category_id
FROM products p
WHERE p.price > (
    SELECT AVG(p2.price)
    FROM products p2
    WHERE p2.category_id = p.category_id
);

-- Non-correlated (runs once):
SELECT name FROM employees WHERE salary > (SELECT AVG(salary) FROM employees);
-- Correlated (runs once per row):
SELECT name FROM employees e WHERE salary > (SELECT AVG(salary) FROM employees e2 WHERE e2.dept_id = e.dept_id);
```

## Speaking Practice
"A correlated subquery runs once for each row in the outer query. It references the outer query's data, so it compares each row against its own group. For example, we can find employees earning above their specific department's average Рђћ not the company-wide average."

## Reading Practice
Read and explain the difference:
```sql
-- Query A (non-correlated):
SELECT name FROM products WHERE price > (SELECT AVG(price) FROM products);
-- Query B (correlated):
SELECT name FROM products p WHERE price > (SELECT AVG(price) FROM products p2 WHERE p2.category_id = p.category_id);
```
Answer: Query A compares each product against the overall average price. Query B compares each product against only its own category's average price.
