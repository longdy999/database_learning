# Day 106: Creating Indexes - Single and Composite

## Topic
Creating Indexes - Single and Composite

## English Explanation
An index is a data structure that improves the speed of data retrieval on a database table. Think of it like the index at the back of a textbook — instead of reading every page, you look up the topic and go directly to the right page. A **single-column index** is created on one column, while a **composite index** (also called a multi-column index) covers two or more columns together. Composite indexes are useful when queries frequently filter or sort by multiple columns at the same time. The order of columns in a composite index matters — the index is most effective when queries use columns from left to right in the index definition.

## SQL Example

```sql
-- Create a sample orders table
CREATE TABLE orders (
    order_id SERIAL PRIMARY KEY,
    customer_id INT NOT NULL,
    order_date DATE NOT NULL,
    status VARCHAR(20) DEFAULT 'pending',
    total_amount DECIMAL(10, 2),
    shipping_city VARCHAR(100)
);

-- Single-column index on customer_id
CREATE INDEX idx_orders_customer ON orders(customer_id);

-- Single-column index on order_date
CREATE INDEX idx_orders_date ON orders(order_date);

-- Composite index on customer_id and order_date together
CREATE INDEX idx_orders_customer_date ON orders(customer_id, order_date);

-- Composite index on status and shipping_city
CREATE INDEX idx_orders_status_city ON orders(status, shipping_city);
```

## Technical Vocabulary

| Term | Definition |
|------|-----------|
| Index | A database structure that speeds up data retrieval |
| Single-column index | An index on one column only |
| Composite index | An index on two or more columns together |
| CREATE INDEX | SQL statement to create an index |
| Index scan | When the database uses an index to find rows |
| Leftmost prefix | The rule that composite indexes work left to right |
| B-tree | The default index structure in most databases |
| Index overhead | Extra storage and write cost from maintaining indexes |

## Speaking Practice
"We created both single and composite indexes on the orders table. A single index on customer_id helps when we search by customer. The composite index on customer_id and order_date helps queries that filter by customer within a date range. The column order matters — the database can use the composite index for customer_id alone, but not for order_date alone."

## Reading Practice
Read the PostgreSQL documentation on CREATE INDEX:
- Focus on the syntax: `CREATE INDEX index_name ON table_name (column1, column2);`
- Understand that indexes speed up SELECT but slow down INSERT/UPDATE/DELETE slightly.
- Note that composite indexes follow the leftmost prefix rule.
