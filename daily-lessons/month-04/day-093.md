# Day 93: Subqueries in WHERE with EXISTS

## Topic
Using the EXISTS operator with subqueries to check whether related rows exist in another table.

## English Explanation
**EXISTS** is a Boolean operator that returns TRUE if the subquery returns at least one row, and FALSE if it returns no rows. It does not care about the actual values returned — only whether rows exist. EXISTS is often more efficient than IN for large datasets because it stops searching as soon as it finds the first matching row.

## SQL Example

```sql
-- Find customers who have placed at least one order
SELECT c.customer_name, c.email
FROM customers c
WHERE EXISTS (
    SELECT 1 FROM orders o
    WHERE o.customer_id = c.customer_id
);

-- Find departments that have employees
SELECT d.department_name
FROM departments d
WHERE EXISTS (
    SELECT 1 FROM employees e
    WHERE e.department_id = d.id
);

-- Find customers who have NOT placed any orders
SELECT c.customer_name
FROM customers c
WHERE NOT EXISTS (
    SELECT 1 FROM orders o
    WHERE o.customer_id = c.customer_id
);
```

## Speaking Practice
"EXISTS checks if any matching rows exist. Unlike IN which compares actual values, EXISTS just asks: does at least one row match? This makes it faster for large tables because it stops as soon as it finds one match."

## Reading Practice
Read and explain:
```sql
SELECT s.store_name FROM stores s
WHERE NOT EXISTS (SELECT 1 FROM inventory i WHERE i.store_id = s.id AND i.quantity > 0);
```
Answer: It finds stores that have no items with quantity greater than zero in inventory — essentially stores with empty stock.
