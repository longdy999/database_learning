# Day 99: Views - Updating, Dropping, and Limitations
## Topic
ALTER VIEW, DROP VIEW, Updatable Views, and View Limitations
## English Explanation
Views can be modified with ALTER VIEW (or CREATE OR REPLACE VIEW), removed with DROP VIEW, and in some cases used for INSERT/UPDATE/DELETE operations. However, not all views are updatable. A view is generally updatable only if it maps directly to a single base table without aggregations, DISTINCT, GROUP BY, or joins. The WITH CHECK OPTION clause prevents inserts/updates through a view that would make the row disappear from the view. Materialized views (a separate concept) actually store data for performance.

## SQL Example
```sql
-- Modify a view: CREATE OR REPLACE VIEW (preferred over ALTER VIEW)
CREATE OR REPLACE VIEW active_customers AS
SELECT customer_id, customer_name, email, city, phone
FROM customers
WHERE status = 'active';

-- DROP VIEW: Remove a view completely
DROP VIEW IF EXISTS old_report_view;

-- Updatable view example (simple, single table, no aggregation)
CREATE VIEW california_customers AS
SELECT customer_id, customer_name, email, city
FROM customers
WHERE city = 'Los Angeles';

-- INSERT through the view (works because it's a simple single-table view)
INSERT INTO california_customers (customer_name, email, city)
VALUES ('New Client', 'client@email.com', 'Los Angeles');

-- WITH CHECK OPTION: Prevent rows that don't match the view's WHERE
CREATE VIEW phnom_penh_customers AS
SELECT customer_id, customer_name, email, city
FROM customers
WHERE city = 'Phnom Penh'
WITH CHECK OPTION;
```

## Hands-on Practice
1. **Modify a view:** Add a column to your `active_customers` view
```sql
CREATE OR REPLACE VIEW active_customers AS
SELECT customer_id, customer_name, email, city, signup_date
FROM customers WHERE status = 'active';
```

2. **Drop a view:** Remove an old view safely
```sql
DROP VIEW IF EXISTS monthly_sales_summary;
```

3. **Test updatable view:** Create a simple view and INSERT through it
```sql
CREATE VIEW junior_employees AS
SELECT employee_id, first_name, last_name, hire_date
FROM employees WHERE salary < 40000
WITH CHECK OPTION;
```

## Technical Vocabulary

| Term | Definition | Khmer |
|------|-----------|-------|
| ALTER VIEW | Modify an existing view definition | កែប្រែនិយមន័យ view ដែលមានស្រាប់ |
| DROP VIEW | Remove a view from the database | ដកចេញ view ពីមូលដ្ឋានទិន្នន័យ |
| Updatable view | A view that allows INSERT/UPDATE/DELETE | View ដែលអនុញ្ញាត INSERT/UPDATE/DELETE |
| Non-updatable view | A view that is read-only | View ដែលអានបានតែប៉ុណ្ណោះ |
| WITH CHECK OPTION | Prevents changes that violate view's WHERE | ការពារការផ្លាស់ប្តូរដែលបំពាន WHERE |
| Materialized view | A view that stores computed data physically | View ដែលរក្សាទុកទិន្នន័យគណនាផ្ទាល់ |
| CREATE OR REPLACE | Create new or overwrite existing view | បង្កើតថ្មីឬសរសេរជំនួស view មានស្រាប់ |
| Base table | The actual table a view is built from | តារាងពិតប្រាកដដែល view ត្រូវបានបង្កើតពី |

## Speaking Practice
Explain to a client: "Not all views can be used to change data. Simple views based on one table usually allow updates, but views with calculations, joins, or grouping are read-only. We can also add a safety check called WITH CHECK OPTION that prevents users from inserting data through a view that shouldn't belong there."
## Reading Practice
Read about the differences between regular views and materialized views. In PostgreSQL, materialized views store the result physically and must be refreshed manually with REFRESH MATERIALIZED VIEW. MySQL does not have built-in materialized views. Understand when each approach is useful.
