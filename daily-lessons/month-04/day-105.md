# Day 105: Introduction to Indexes - How They Work

## Topic
Understanding what indexes are, how B-tree structures work at a conceptual level, why indexes speed up reads but slow down writes, and when to consider using them.

## English Explanation
An index in a database is like the index at the back of a textbook. Instead of reading every page to find a topic, you look in the index which tells you the exact page number. Similarly, a database index is a separate data structure that stores pointers to row locations based on column values. Most indexes use a B-tree (balanced tree) structure that allows the database to find any value in logarithmic time - meaning even with millions of rows, it only needs a few steps to find your data. The tradeoff is that every INSERT, UPDATE, or DELETE must also update the index, making writes slightly slower.

## SQL Example

### Conceptual: How a Query Works Without an Index
```sql
-- Without index: database must scan ALL rows (full table scan)
-- Table has 1,000,000 rows. Finding one email checks every row.
SELECT * FROM users WHERE email = 'dara@example.com';
-- Time: O(n) - checks up to 1,000,000 rows

-- With index on email column: database uses B-tree lookup
-- Time: O(log n) - checks about 20 steps for 1,000,000 rows!
```

### Creating an Index
```sql
-- Basic index creation syntax
CREATE INDEX idx_users_email ON users(email);

-- Now this query uses the index automatically
SELECT * FROM users WHERE email = 'dara@example.com';
-- Database reads index вҶ’ finds pointer вҶ’ reads exact row
```


### B-tree Concept (Simplified)
```
         [M]                    вҶҗ Root node
        /   \
     [D,H]   [R,V]             вҶҗ Branch nodes
    / | \    / | \
  [A-C][E-G][I-L] [N-Q][S-U][W-Z]  вҶҗ Leaf nodes (point to rows)

-- To find "K": RootвҶ’Right of D, Left of H? NoвҶ’ Right pathвҶ’ I-L leaf вҶ’ Found!
-- Only 3 steps instead of scanning 26 entries
```

### The Write Tradeoff
```sql
-- Every INSERT must also update the index
INSERT INTO users (name, email) VALUES ('Sokha', 'sokha@example.com');
-- Database does: 1) Insert row into table  2) Update B-tree index for email

-- More indexes = slower writes
-- Rule of thumb: index columns used in WHERE, JOIN, ORDER BY
-- Do NOT index every column
```

## Technical Vocabulary

| Term | Definition |
|------|-----------|
| Index | A data structure that speeds up data retrieval |
| B-tree | Balanced tree structure used by most database indexes |
| Full table scan | Reading every row to find matching data (slow) |
| Pointer | A reference to the physical location of a row |
| Logarithmic time | O(log n) - grows very slowly as data increases |
| Read performance | How fast SELECT queries execute |
| Write overhead | Extra time needed for INSERT/UPDATE/DELETE with indexes |
| CREATE INDEX | SQL command to build an index on a column |

## Speaking Practice
"An index works like a book's index - it helps the database find data without reading every row. We use B-tree structures that let us find any value in just a few steps, even in tables with millions of rows. The tradeoff is that writes become slightly slower because the index must also be updated."
