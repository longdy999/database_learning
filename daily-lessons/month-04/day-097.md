## SQL Example
```sql
-- Practice Problem 1: Scalar - Find the most expensive product name
SELECT product_name FROM products
WHERE price = (SELECT MAX(price) FROM products);

-- Practice Problem 2: IN - Find employees in departments located in Phnom Penh
SELECT name FROM employees
WHERE dept_id IN (SELECT dept_id FROM departments WHERE city = 'Phnom Penh');

-- Practice Problem 3: EXISTS - Find customers who have at least one order
SELECT c.customer_name FROM customers c
WHERE EXISTS (SELECT 1 FROM orders o WHERE o.customer_id = c.customer_id);

-- Practice Problem 4: Correlated - Each product vs category average price
SELECT p.product_name, p.price,
  (SELECT AVG(p2.price) FROM products p2
   WHERE p2.category_id = p.category_id) AS category_avg
FROM products p;

-- Practice Problem 5: Derived table - Department with most employees
SELECT dept_name, emp_count
FROM (SELECT d.department_name AS dept_name, COUNT(*) AS emp_count
      FROM employees e JOIN departments d ON e.dept_id = d.dept_id
      GROUP BY d.department_name) AS dept_counts
ORDER BY emp_count DESC LIMIT 1;
```

## Speaking Practice
Practice explaining: "When I need to check if data exists without caring about the actual values, I use EXISTS because it stops searching as soon as it finds one match. This makes it faster than IN for large datasets. But when I need specific values from another table to compare, IN is more readable and appropriate."
