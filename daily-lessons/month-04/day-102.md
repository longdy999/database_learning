# Day 102: Recursive CTEs - Hierarchical Data

## Topic
Using WITH RECURSIVE to query tree structures like organizational charts, category hierarchies, and nested comment threads.

## English Explanation
A recursive CTE has two parts: the base case (anchor member) and the recursive case (recursive member) joined by UNION ALL. The base case selects the starting rows (like the CEO in an org chart). The recursive case references the CTE itself to find the next level (direct reports). The database engine repeats the recursive part until no new rows are found. This is perfect for hierarchical data where rows have parent-child relationships stored with a parent_id column.

## SQL Example

### Employee Hierarchy (Org Chart)
```sql
-- Table structure
CREATE TABLE employees (
    employee_id INT PRIMARY KEY,
    name VARCHAR(100),
    manager_id INT REFERENCES employees(employee_id),
    title VARCHAR(100)
);

-- Sample data
INSERT INTO employees VALUES
(1, 'Sophea', NULL, 'CEO'),
(2, 'Dara', 1, 'VP Engineering'),
(3, 'Channary', 1, 'VP Sales'),
(4, 'Virak', 2, 'Senior Developer'),
(5, 'Bopha', 2, 'Junior Developer'),
(6, 'Kosal', 3, 'Sales Manager');

-- Recursive CTE to show full hierarchy
WITH RECURSIVE org_chart AS (
    -- Base case: start with CEO (no manager)
    SELECT employee_id, name, title, manager_id, 1 AS level
    FROM employees
    WHERE manager_id IS NULL
    
    UNION ALL
    
    -- Recursive case: find direct reports
    SELECT e.employee_id, e.name, e.title, e.manager_id, oc.level + 1
    FROM employees e
    JOIN org_chart oc ON e.manager_id = oc.employee_id
)
SELECT REPEAT('  ', level - 1) || name AS hierarchy, title, level
FROM org_chart
ORDER BY level, name;
```


### Category Tree Example
```sql
WITH RECURSIVE category_tree AS (
    SELECT category_id, name, parent_id, 1 AS depth
    FROM categories
    WHERE parent_id IS NULL

    UNION ALL

    SELECT c.category_id, c.name, c.parent_id, ct.depth + 1
    FROM categories c
    JOIN category_tree ct ON c.parent_id = ct.category_id
)
SELECT REPEAT('-- ', depth - 1) || name AS category_path, depth
FROM category_tree
ORDER BY depth, name;
```

## Technical Vocabulary

| Term | Definition |
|------|-----------|
| Recursive CTE | A CTE that references itself to traverse hierarchies |
| Base case | The starting point (anchor) that does not recurse |
| Recursive case | The part that references the CTE to find next level |
| UNION ALL | Combines base case and recursive case results |
| Hierarchy | A tree structure with parent-child relationships |
| Self-referencing | A table with a foreign key pointing to its own primary key |
| Depth/Level | How many steps from the root node |
| Anchor member | Another name for the base case in recursive CTE |
