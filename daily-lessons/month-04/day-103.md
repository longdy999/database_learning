# Day 103: Temporary Tables

## Topic
Using CREATE TEMPORARY TABLE for session-scoped intermediate results in complex multi-step operations.

## English Explanation
A temporary table is a real table that exists only for the duration of your database session (connection). When you disconnect, it disappears automatically. Temporary tables are useful when you need to break a complex operation into multiple steps, store intermediate results, and query them several times. Unlike CTEs which exist only for one statement, temp tables persist across multiple queries in the same session. You can also add indexes to temp tables for better performance on large intermediate datasets.

## SQL Example

### Creating and Using Temporary Tables
```sql
-- Step 1: Create temp table with monthly sales summary
CREATE TEMPORARY TABLE monthly_sales AS
SELECT 
    DATE_TRUNC('month', order_date) AS month,
    product_id,
    SUM(quantity) AS total_qty,
    SUM(amount) AS total_revenue
FROM orders
WHERE order_date >= '2024-01-01'
GROUP BY DATE_TRUNC('month', order_date), product_id;

-- Step 2: Add an index for faster lookups
CREATE INDEX idx_temp_product ON monthly_sales(product_id);

-- Step 3: Use the temp table in multiple queries
SELECT month, SUM(total_revenue) AS monthly_total
FROM monthly_sales
GROUP BY month
ORDER BY month;

-- Step 4: Join temp table with products for detailed report
SELECT p.product_name, ms.month, ms.total_qty, ms.total_revenue
FROM monthly_sales ms
JOIN products p ON ms.product_id = p.product_id
WHERE ms.total_revenue > 1000
ORDER BY ms.total_revenue DESC;

-- Step 5: Clean up (optional - auto-dropped on disconnect)
DROP TABLE IF EXISTS monthly_sales;
```


### Comparison: CTE vs Temp Table
```sql
-- CTE: exists only for this one statement
WITH sales_summary AS (
    SELECT product_id, SUM(amount) AS total
    FROM orders GROUP BY product_id
)
SELECT * FROM sales_summary WHERE total > 500;
-- Cannot use sales_summary in another query!

-- Temp Table: exists for entire session
CREATE TEMPORARY TABLE sales_summary AS
SELECT product_id, SUM(amount) AS total
FROM orders GROUP BY product_id;

-- Query 1
SELECT * FROM sales_summary WHERE total > 500;
-- Query 2 (reuse same data)
SELECT AVG(total) FROM sales_summary;
-- Query 3 (join with other tables)
SELECT p.name, s.total FROM sales_summary s JOIN products p ON s.product_id = p.product_id;
```

## Technical Vocabulary

| Term | Definition |
|------|-----------|
| Temporary table | A table that exists only for the current session |
| Session-scoped | Lasts from connect to disconnect |
| Intermediate results | Data stored between processing steps |
| CREATE TEMPORARY TABLE | SQL command to make a temp table |
| DROP TABLE | Removes a table (temp tables auto-drop on disconnect) |
| Indexable | Can have indexes added for performance |
| Multi-step operation | A task broken into sequential queries |
| Persist | To remain available across multiple statements |
