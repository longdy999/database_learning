# Day 95: Subqueries in SELECT and FROM Clauses

## Topic
Using subqueries in the SELECT clause (computed columns) and FROM clause (derived tables/inline views).

## English Explanation
Subqueries are not limited to WHERE clauses. In the **SELECT clause**, a subquery can add a computed column to each row (like counting related records). In the **FROM clause**, a subquery creates a temporary result set called a **derived table** or **inline view** that the outer query treats like a regular table. Derived tables are useful for breaking complex queries into simpler steps.

## SQL Example

```sql
-- Subquery in SELECT: Add order count for each customer
SELECT
    c.customer_name,
    c.email,
    (SELECT COUNT(*) FROM orders o WHERE o.customer_id = c.customer_id) AS order_count
FROM customers c;

-- Subquery in SELECT: Add total spent per customer
SELECT
    c.customer_name,
    (SELECT SUM(o.amount) FROM orders o WHERE o.customer_id = c.customer_id) AS total_spent
FROM customers c;

-- Subquery in FROM: Derived table for department statistics
SELECT dept_stats.department_name, dept_stats.avg_salary
FROM (
    SELECT d.department_name, AVG(e.salary) AS avg_salary
    FROM departments d
    JOIN employees e ON d.id = e.department_id
    GROUP BY d.department_name
) AS dept_stats
WHERE dept_stats.avg_salary > 50000;
```


## Hands-on Practice (10-15 minutes)

**Task 1:** Add an order_count column to a customer list using a subquery in SELECT.

```sql
SELECT
    c.customer_id,
    c.customer_name,
    (SELECT COUNT(*) FROM orders o WHERE o.customer_id = c.customer_id) AS order_count
FROM customers c
ORDER BY order_count DESC;
```

**Task 2:** Use a derived table to find categories with average price above $100.

```sql
SELECT cat_summary.category_name, cat_summary.avg_price
FROM (
    SELECT c.category_name, AVG(p.price) AS avg_price
    FROM categories c
    JOIN products p ON c.id = p.category_id
    GROUP BY c.category_name
) AS cat_summary
WHERE cat_summary.avg_price > 100;
```

## Technical Vocabulary

| Term | Definition | Khmer |
|------|-----------|-------|
| Derived table | A subquery in the FROM clause used as a table | Subquery ក្នុង FROM ដែលប្រើជាតារាង |
| Inline view | Another name for derived table | ឈ្មោះមួយទៀតសម្រាប់ derived table |
| Computed column | A column calculated by a subquery in SELECT | ជួរឈរដែលគណនាដោយ subquery |
| Alias (AS) | Required name given to derived tables | ឈ្មោះដែលត្រូវដាក់ឱ្យ derived table |
| Result set | The temporary output of a query | សំណុំលទ្ធផលបណ្តោះអាសន្ន |

## Speaking Practice
"You can use subqueries in the SELECT clause to add calculated columns, like counting how many orders each customer has. You can also put a subquery in the FROM clause to create a derived table — a temporary table built from a query that you can then filter or join like any regular table."

## Reading Practice
Read and explain:
```sql
SELECT e.name, e.salary,
    (SELECT AVG(salary) FROM employees e2 WHERE e2.department_id = e.department_id) AS dept_avg
FROM employees e;
```
Answer: It shows each employee's name and salary alongside their department's average salary as an additional column, allowing easy visual comparison.
