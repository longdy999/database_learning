## SQL Example
```sql
-- Review: Creating an index
CREATE INDEX idx_orders_customer ON orders(customer_id);

-- Review: Composite index
CREATE INDEX idx_orders_date_status ON orders(order_date, status);

-- Review: Using EXPLAIN to analyze
EXPLAIN SELECT o.order_id, c.name
FROM orders o
JOIN customers c ON o.customer_id = c.customer_id
WHERE o.order_date > '2024-01-01'
ORDER BY o.order_date DESC;

-- Review: Checking if index is used
EXPLAIN SELECT * FROM products WHERE category_id = 5;
```

## Hands-on Practice
Complete this self-assessment checklist by writing and running queries:

**Task: Optimize these queries from scratch (no hints)**

```sql
-- Query 1: Find all orders from the last 30 days for a specific customer
SELECT * FROM orders
WHERE customer_id = 42 AND order_date > CURRENT_DATE - INTERVAL '30 days';

-- Query 2: Get top 10 products by total revenue
SELECT p.product_name, SUM(oi.quantity * oi.unit_price) AS revenue
FROM products p
JOIN order_items oi ON p.product_id = oi.product_id
GROUP BY p.product_name
ORDER BY revenue DESC LIMIT 10;


-- Query 3: Count orders per status for a date range
SELECT status, COUNT(*) FROM orders
WHERE order_date BETWEEN '2024-01-01' AND '2024-06-30'
GROUP BY status;
```

For each query: run EXPLAIN, identify missing indexes, create them, run EXPLAIN again.

## Technical Vocabulary

| Term | Definition |
|------|-----------|
| B-tree Index | Default balanced tree structure for fast lookups |
| Composite Index | Index on multiple columns together |
| Index Scan | Query uses index to find rows efficiently |
| Seq Scan | Full table scan without using index |
| EXPLAIN | Shows the query execution plan |
| Query Planner | Database component that chooses execution strategy |
| Covering Index | Index that contains all needed columns |
| Index Selectivity | How unique the indexed values are |
