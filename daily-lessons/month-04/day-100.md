# Day 100: Common Table Expressions (CTEs) - WITH Clause

## Topic
WITH clause - Writing Readable Queries Using Common Table Expressions

## English Explanation
A Common Table Expression (CTE) is a temporary named result set defined with the WITH keyword. CTEs make complex queries more readable by breaking them into logical steps. Instead of nesting subqueries inside subqueries, you define each part separately with a name, then reference it in the final SELECT. You can define multiple CTEs in one query. CTEs exist only for the duration of that single query.

## Khmer Explanation
CTE is a temporary named result set using WITH keyword. It makes complex queries readable by breaking them into named steps instead of nesting subqueries. You can use multiple CTEs in one query separated by commas. They only exist for that single statement.

## SQL Example
```sql
-- Basic CTE syntax
WITH high_earners AS (
  SELECT employee_id, first_name, salary, dept_id
  FROM employees WHERE salary > 60000
)
SELECT * FROM high_earners ORDER BY salary DESC;

-- CTE version replaces nested subquery for clarity
WITH dept_averages AS (
  SELECT d.department_name AS dept_name, AVG(e.salary) AS avg_sal
  FROM employees e JOIN departments d ON e.dept_id = d.dept_id
  GROUP BY d.department_name
)
SELECT dept_name, avg_sal FROM dept_averages WHERE avg_sal > 50000;

-- Multiple CTEs in one query
WITH
  active_orders AS (
    SELECT order_id, customer_id, total_amount
    FROM orders WHERE status = 'completed'
  ),
  customer_totals AS (
    SELECT customer_id, SUM(total_amount) AS lifetime_value
    FROM active_orders GROUP BY customer_id
  )
SELECT c.customer_name, ct.lifetime_value
FROM customer_totals ct
JOIN customers c ON ct.customer_id = c.customer_id
WHERE ct.lifetime_value > 1000 ORDER BY ct.lifetime_value DESC;
```

## Hands-on Practice
Rewrite these complex subqueries as CTEs:

1. **Single CTE:** Find products with above-average price
```sql
WITH avg_price AS (SELECT AVG(price) AS average FROM products)
SELECT product_name, price
FROM products, avg_price WHERE price > avg_price.average;
```

2. **Multiple CTEs:** Top customers by order count and spending
```sql
WITH
  order_counts AS (
    SELECT customer_id, COUNT(*) AS num_orders FROM orders GROUP BY customer_id
  ),
  order_totals AS (
    SELECT customer_id, SUM(total_amount) AS total_spent FROM orders GROUP BY customer_id
  )
SELECT c.customer_name, oc.num_orders, ot.total_spent
FROM customers c
JOIN order_counts oc ON c.customer_id = oc.customer_id
JOIN order_totals ot ON c.customer_id = ot.customer_id
ORDER BY ot.total_spent DESC LIMIT 5;
```

| Term | Definition |

| CTE | Common Table Expression - temporary named result set |

| Multiple CTEs | Using more than one CTE in a query |
| Logical steps | Breaking a problem into clear parts |
