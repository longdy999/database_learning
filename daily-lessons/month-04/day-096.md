## SQL Example
```sql
-- 1. Scalar subquery: Find employees earning above average
SELECT name, salary
FROM employees
WHERE salary > (SELECT AVG(salary) FROM employees);

-- 2. IN subquery: Find customers who placed orders
SELECT name FROM customers
WHERE customer_id IN (SELECT customer_id FROM orders);

-- 3. EXISTS subquery: Find departments with employees
SELECT d.department_name
FROM departments d
WHERE EXISTS (SELECT 1 FROM employees e WHERE e.dept_id = d.dept_id);

-- 4. Correlated subquery: Each employee vs their department average
SELECT name, salary,
  (SELECT AVG(salary) FROM employees e2
   WHERE e2.dept_id = e1.dept_id) AS dept_avg
FROM employees e1;

-- 5. Derived table: Top department by total salary
SELECT dept_name, total_salary
FROM (SELECT d.department_name AS dept_name, SUM(e.salary) AS total_salary
      FROM employees e JOIN departments d ON e.dept_id = d.dept_id
      GROUP BY d.department_name) AS dept_totals
ORDER BY total_salary DESC LIMIT 1;
```

## Speaking Practice
Explain to a client: "There are five main types of subqueries. I choose which one to use based on what data I need. For a single value, I use a scalar subquery. For checking a list, I use IN. For checking existence, EXISTS is most efficient. When each row needs its own calculation, I use correlated subqueries. And when I need a temporary result set, I use a derived table."
