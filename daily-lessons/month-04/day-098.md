# Day 98: Views - Creating Virtual Tables
## Topic
CREATE VIEW - Saving Queries as Virtual Tables
## English Explanation
A view is a saved SQL query that acts like a virtual table. It does not store data itself - every time you query a view, it runs the underlying SELECT statement. Views simplify complex queries by giving them a name, provide security by hiding certain columns from users, and ensure consistency so everyone uses the same query logic. Think of a view as a "window" into your data that shows only what you want others to see.

## SQL Example
```sql
-- Basic CREATE VIEW syntax
CREATE VIEW active_customers AS
SELECT customer_id, customer_name, email, city
FROM customers
WHERE status = 'active';

-- Now query it like a table
SELECT * FROM active_customers WHERE city = 'Phnom Penh';

-- View for reporting: monthly sales summary
CREATE VIEW monthly_sales_summary AS
SELECT
  DATE_FORMAT(order_date, '%Y-%m') AS month,
  COUNT(*) AS total_orders,
  SUM(total_amount) AS revenue,
  AVG(total_amount) AS avg_order_value
FROM orders
GROUP BY DATE_FORMAT(order_date, '%Y-%m');

-- View for security: employee info without salary
CREATE VIEW employee_directory AS
SELECT employee_id, first_name, last_name, email, department
FROM employees;
-- Users who query this view cannot see salary data
```
## Hands-on Practice
Create these 3 views for different reporting needs:
1. **Customer Report View:** Shows customer name, total orders, and total spent
```sql
CREATE VIEW customer_report AS
SELECT c.customer_name,
  COUNT(o.order_id) AS total_orders,
  COALESCE(SUM(o.total_amount), 0) AS total_spent
FROM customers c
LEFT JOIN orders o ON c.customer_id = o.customer_id
GROUP BY c.customer_id, c.customer_name;
```

2. **Product Inventory View:** Shows products with low stock (below 10 units)
```sql
CREATE VIEW low_stock_products AS
SELECT product_id, product_name, stock_quantity, category_name
FROM products p JOIN categories c ON p.category_id = c.category_id
WHERE stock_quantity < 10;
```

3. **Department Summary View:** Shows each department with employee count and avg salary
```sql
CREATE VIEW department_summary AS
SELECT d.department_name,
  COUNT(e.employee_id) AS employee_count,
  ROUND(AVG(e.salary), 2) AS avg_salary
FROM departments d
LEFT JOIN employees e ON d.dept_id = e.dept_id
GROUP BY d.dept_id, d.department_name;
```

## Speaking Practice
Explain to a client: "I created views for your reporting needs. A view is like a saved report template. Instead of writing complex queries every time, your team can simply query the view name. This ensures everyone sees the same data calculated the same way, and we can hide sensitive columns like salary from general users."
## Reading Practice
Read the PostgreSQL CREATE VIEW documentation. Note the syntax differences from MySQL. Look at how views appear in database management tools like pgAdmin or MySQL Workbench - they show up under a "Views" section separate from tables.
