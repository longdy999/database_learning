# Day 104: Week 14 Review and Checkpoint

## Topic
Review of views, CTEs, recursive CTEs, and temporary tables. Self-assessment and comparison of when to use each approach.

## English Explanation
This week covered four ways to organize and simplify complex queries: Views (permanent named queries stored in the database), CTEs (temporary named result sets for one statement), Recursive CTEs (self-referencing CTEs for hierarchical data), and Temporary Tables (session-scoped real tables for multi-step operations). Each has a specific use case. Today you will review all four, write them from memory, and understand the tradeoffs so you can choose the right tool for each situation.

## SQL Example

### Comparison Chart in SQL Comments
```sql
-- ============================================
-- VIEWS: Permanent, stored in database schema
-- ============================================
CREATE VIEW active_customers AS
SELECT customer_id, name, email
FROM customers WHERE status = 'active';
-- Use: SELECT * FROM active_customers; (anytime, any session)

-- ============================================
-- CTE: Temporary, one statement only
-- ============================================
WITH recent_orders AS (
    SELECT order_id, customer_id, amount
    FROM orders WHERE order_date > CURRENT_DATE - 30
)
SELECT c.name, SUM(ro.amount)
FROM customers c JOIN recent_orders ro ON c.customer_id = ro.customer_id
GROUP BY c.name;

-- ============================================
-- RECURSIVE CTE: For tree/hierarchy traversal
-- ============================================
WITH RECURSIVE dept_tree AS (
    SELECT dept_id, name, parent_id, 1 AS level
    FROM departments WHERE parent_id IS NULL
    UNION ALL
    SELECT d.dept_id, d.name, d.parent_id, dt.level + 1
    FROM departments d JOIN dept_tree dt ON d.parent_id = dt.dept_id
)
SELECT * FROM dept_tree;
```


```sql
-- ============================================
-- TEMPORARY TABLE: Session-scoped, multi-step
-- ============================================
CREATE TEMPORARY TABLE quarterly_report AS
SELECT product_id, SUM(revenue) AS q_revenue
FROM sales WHERE sale_date >= '2024-01-01'
GROUP BY product_id;
-- Reusable in multiple queries during this session
SELECT * FROM quarterly_report WHERE q_revenue > 5000;
SELECT AVG(q_revenue) FROM quarterly_report;
```

### When to Use Each

| Tool | Scope | Reusable | Indexable | Best For |
|------|-------|----------|-----------|----------|
| View | Permanent | Yes, all sessions | No (uses base indexes) | Repeated access patterns |
| CTE | One statement | No | No | Readable complex queries |
| Recursive CTE | One statement | No | No | Hierarchy traversal |
| Temp Table | One session | Yes, within session | Yes | Multi-step processing |

## Technical Vocabulary

| Term | Definition |
|------|-----------|
| View | A stored named query accessible like a table |
| CTE | A temporary named result set within one statement |
| Recursive CTE | A CTE that references itself for tree traversal |
| Temporary Table | A session-scoped table for intermediate storage |
| Scope | How long and where something is accessible |
| Tradeoff | Choosing between advantages and disadvantages |

## Speaking Practice
"There are four main ways to organize complex SQL: views for permanent reusable queries, CTEs for clean readable one-time logic, recursive CTEs for tree data, and temp tables when you need to reuse results across multiple steps in one session. The choice depends on whether you need persistence, reusability, or hierarchy support."
