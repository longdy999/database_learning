## SQL Example

```sql
-- Subquery: Find customers with above-average orders
SELECT customer_name FROM customers
WHERE customer_id IN (
    SELECT customer_id FROM orders
    GROUP BY customer_id
    HAVING SUM(total_amount) > (SELECT AVG(total_amount) FROM orders)
);

-- View: Simplified sales report
CREATE VIEW monthly_sales AS
SELECT DATE_TRUNC('month', order_date) AS month,
       COUNT(*) AS total_orders,
       SUM(total_amount) AS revenue
FROM orders GROUP BY DATE_TRUNC('month', order_date);

-- CTE: Customer ranking with readable structure
WITH customer_totals AS (
    SELECT customer_id, SUM(total_amount) AS lifetime_value
    FROM orders GROUP BY customer_id
),
ranked AS (
    SELECT *, RANK() OVER (ORDER BY lifetime_value DESC) AS rank
    FROM customer_totals
)
SELECT * FROM ranked WHERE rank <= 10;


-- Index: Speed up common lookups
CREATE INDEX idx_orders_customer ON orders(customer_id);
CREATE INDEX idx_orders_date ON orders(order_date);

-- EXPLAIN: Check query performance
EXPLAIN ANALYZE SELECT * FROM orders WHERE customer_id = 5;
```

## Technical Vocabulary

| Term | Definition |
|------|-----------|
| Subquery | A query nested inside another query |
| View | A virtual table based on a stored SELECT query |
| CTE | Common Table Expression using WITH clause |
| Index | Data structure that speeds up row lookups |
| EXPLAIN | Shows the query execution plan |
| Transaction | Atomic group of operations |
| ACID | Properties ensuring reliable transactions |
| Stored Procedure | Reusable block of SQL logic in database |
| Seq Scan | Full table scan (no index used) |
| Index Scan | Fast lookup using an index |
